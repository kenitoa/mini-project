# 힙 정렬

힙 정렬은 리스트를 `힙(heap)`이라는 구조로 만든 뒤, 가장 큰 값을 하나씩 꺼내 뒤쪽부터 채우는 방식으로 정렬한다.

여기서는 가장 큰 값이 위로 올라오는 `최대 힙(max heap)`을 기준으로 생각한다.

## 핵심 생각

예를 들어 이런 숫자들이 있다고 하자.

```text
[5, 3, 8, 4, 2, 7, 1, 10]
```

힙 정렬은 먼저 이 리스트를 최대 힙으로 만든다.

최대 힙은 부모 값이 자식 값보다 항상 크거나 같은 구조이다.

```text
부모 >= 자식
```

그래서 최대 힙의 맨 앞에는 항상 가장 큰 값이 온다.

```text
가장 큰 값 = 힙의 첫 번째 값
```

그다음 가장 큰 값을 리스트의 맨 뒤로 보낸다.

```text
[가장 큰 값] <-> [마지막 값]
```

이제 맨 뒤의 값은 정렬이 끝난 값으로 보고, 남은 부분만 다시 힙으로 만든다.

이 과정을 반복하면 큰 값부터 뒤쪽에 차례대로 고정된다.

## 힙 정렬 의사코드

```text
heap_sort(리스트):
    리스트 전체를 최대 힙으로 만든다

    마지막 위치를 리스트의 끝으로 잡는다

    마지막 위치가 1보다 클 동안 반복한다:
        힙의 첫 번째 값과 마지막 위치의 값을 바꾼다
        마지막 위치의 값은 정렬된 상태가 된다

        힙의 크기를 1 줄인다
        첫 번째 값을 아래로 내려 보내며 다시 최대 힙을 만든다

    리스트를 반환한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function heap_sort(arr):
    n = length(arr)

    for i from parent of last value down to 0:
        heapify(arr, n, i)

    for end from n - 1 down to 1:
        swap arr[0] and arr[end]
        heapify(arr, end, 0)

    return arr
```

`heapify`는 특정 위치의 값이 자식보다 작으면 아래로 내려 보내서 다시 힙 모양을 맞추는 함수이다.

```text
function heapify(arr, heap_size, root):
    largest = root
    left = root * 2 + 1
    right = root * 2 + 2

    if left < heap_size and arr[left] > arr[largest]:
        largest = left

    if right < heap_size and arr[right] > arr[largest]:
        largest = right

    if largest is not root:
        swap arr[root] and arr[largest]
        heapify(arr, heap_size, largest)
```

## 흐름을 쉽게 풀어보기

힙 정렬은 이렇게 생각하면 쉽다.

```text
1. 리스트를 최대 힙으로 만든다.
2. 가장 큰 값은 맨 앞에 있다.
3. 맨 앞 값과 맨 뒤 값을 바꾼다.
4. 맨 뒤 값은 정렬이 끝난 값으로 본다.
5. 남은 부분만 다시 최대 힙으로 만든다.
6. 이 과정을 반복한다.
```

즉, 처음부터 작은 값을 앞에 하나씩 넣는 방식이 아니다.

가장 큰 값을 계속 찾아서 뒤쪽부터 확정해 나가는 방식이다.

## 예시

```text
heap_sort([5, 3, 8, 4, 2])
```

먼저 리스트를 최대 힙으로 만든다.

```text
[8, 4, 5, 3, 2]
```

이제 가장 큰 값인 `8`은 맨 앞에 있다.

맨 앞 값과 맨 뒤 값을 바꾼다.

```text
[2, 4, 5, 3, 8]
```

`8`은 이제 정렬된 위치에 있으므로, 남은 부분만 다시 힙으로 만든다.

```text
[5, 4, 2, 3, 8]
```

다시 맨 앞 값 `5`와 남은 부분의 마지막 값을 바꾼다.

```text
[3, 4, 2, 5, 8]
```

`5`도 정렬된 위치에 있다.

남은 부분을 다시 힙으로 만든다.

```text
[4, 3, 2, 5, 8]
```

같은 과정을 반복하면 다음과 같이 된다.

```text
[2, 3, 4, 5, 8]
```

## 종료 조건

힙 정렬은 정렬되지 않은 힙의 크기를 계속 줄여 나간다.

힙의 크기가 1이 되면 더 이상 바꿀 값이 없다.

```text
while heap_size > 1:
    가장 큰 값을 뒤로 보낸다
```

리스트에 값이 없거나 하나만 있으면 더 이상 정렬할 필요가 없다.

예를 들어:

```text
[]      이미 정렬됨
[7]     이미 정렬됨
```

그래서 그대로 반환한다.

## 힙 구조 이해하기

힙은 보통 트리처럼 생각한다.

하지만 실제 코드에서는 리스트만으로도 표현할 수 있다.

어떤 값의 위치를 `i`라고 하면 자식 위치는 다음과 같다.

```text
왼쪽 자식  = i * 2 + 1
오른쪽 자식 = i * 2 + 2
```

예를 들어 다음 리스트가 있다고 하자.

```text
[8, 4, 5, 3, 2]
```

위치를 트리처럼 보면 다음과 같다.

```text
        8
      /   \
     4     5
    / \
   3   2
```

부모가 자식보다 크므로 최대 힙 조건을 만족한다.

## heapify 과정

`heapify`는 힙 조건이 깨졌을 때 다시 맞추는 과정이다.

예를 들어 다음처럼 맨 앞 값이 작아졌다고 하자.

```text
[2, 4, 5, 3]
```

맨 앞 값 `2`의 자식은 `4`와 `5`이다.

가장 큰 값은 `5`이므로 `2`와 `5`를 바꾼다.

```text
[5, 4, 2, 3]
```

이제 다시 최대 힙이 된다.

즉, `heapify`는 작은 값이 위에 있으면 알맞은 위치까지 아래로 내려 보내는 과정이다.

## 시간 복잡도

힙 정렬의 시간 복잡도는 다음과 같다.

```text
O(n log n)
```

가장 큰 값을 하나 꺼낼 때마다 힙을 다시 정리해야 하고, 이때 걸리는 시간이 `log n` 정도이기 때문이다.

이 작업을 값의 개수만큼 반복하므로 전체 시간 복잡도는 `O(n log n)`이 된다.

퀵 정렬은 피벗 선택에 따라 최악의 경우 느려질 수 있지만, 힙 정렬은 보통 다음 시간 복잡도를 유지한다.

```text
O(n log n)
```

## 장점

```text
추가 리스트를 거의 사용하지 않는다.
시간 복잡도가 안정적으로 O(n log n)이다.
가장 큰 값이나 가장 작은 값을 꺼내는 구조를 이해하기 좋다.
```

## 단점

```text
병합 정렬이나 퀵 정렬보다 과정이 직관적으로 덜 쉬울 수 있다.
같은 값의 기존 순서가 유지되지 않을 수 있다.
실제 실행 속도는 퀵 정렬보다 느린 경우가 많다.
```

## 한 문장 요약

힙 정렬은 리스트를 최대 힙으로 만든 뒤, 가장 큰 값을 뒤쪽부터 하나씩 고정해서 전체를 정렬하는 알고리즘이다.

## 알고리즘 흐름도

아래 흐름은 힙 정렬을 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
정렬할 리스트를 받는다
  |
  v
리스트 길이가 0 또는 1인가?
  |
  +-- 예 --> 리스트를 그대로 반환한다
  |
  +-- 아니오
        |
        v
      리스트 전체를 최대 힙으로 만든다
        |
        v
      힙의 마지막 위치를 잡는다
        |
        v
      힙의 크기가 1보다 큰가?
        |
        +-- 아니오 --> 리스트를 반환한다
        |
        +-- 예
              |
              v
            첫 번째 값과 마지막 값을 바꾼다
              |
              v
            마지막 값은 정렬된 위치로 확정한다
              |
              v
            힙의 크기를 1 줄인다
              |
              v
            첫 번째 값부터 heapify를 실행한다
              |
              v
            다시 힙의 크기를 확인한다
              |
              v
            끝
```

## 더 짧게 외우는 흐름

```text
1. 리스트를 받는다.
2. 길이가 1 이하이면 그대로 반환한다.
3. 리스트 전체를 최대 힙으로 만든다.
4. 맨 앞의 가장 큰 값과 힙의 마지막 값을 바꾼다.
5. 마지막 값은 정렬된 값으로 확정한다.
6. 남은 부분을 다시 힙으로 만든다.
7. 힙의 크기가 1이 될 때까지 반복한다.
```

## 머릿속 결정 순서

힙 정렬을 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
이 리스트는 이미 정렬된 상태인가?
  -> 길이가 0 또는 1이면 그대로 반환

어떻게 가장 큰 값을 빠르게 찾을 것인가?
  -> 리스트를 최대 힙으로 만들기

가장 큰 값은 어디에 있는가?
  -> 힙의 첫 번째 위치

가장 큰 값을 어디로 보내야 하는가?
  -> 현재 힙의 마지막 위치

남은 부분은 아직 힙 조건을 만족하는가?
  -> heapify로 다시 최대 힙 만들기

언제 끝나는가?
  -> 힙의 크기가 1이 되면 끝
```
