# 계수 정렬

계수 정렬은 값끼리 직접 비교하지 않고, 각 숫자가 몇 번 나왔는지 세어 둔 뒤 그 개수만큼 다시 꺼내는 방식으로 정렬한다.

이때 숫자의 등장 횟수를 저장하는 리스트를 `카운트 배열(count array)`이라고 생각하면 된다.

## 핵심 생각

예를 들어 이런 숫자들이 있다고 하자.

```text
[5, 3, 8, 4, 2, 3, 1, 5]
```

계수 정렬은 먼저 각 숫자가 몇 번 나왔는지 센다.

```text
1은 1번
2는 1번
3은 2번
4는 1번
5는 2번
8은 1번
```

그다음 작은 숫자부터 개수만큼 다시 적는다.

```text
1을 1번 적는다 -> [1]
2를 1번 적는다 -> [1, 2]
3을 2번 적는다 -> [1, 2, 3, 3]
4를 1번 적는다 -> [1, 2, 3, 3, 4]
5를 2번 적는다 -> [1, 2, 3, 3, 4, 5, 5]
8을 1번 적는다 -> [1, 2, 3, 3, 4, 5, 5, 8]
```

값을 서로 비교해서 위치를 바꾸는 것이 아니라, 값의 개수를 이용해서 정렬하는 방식이다.

## 계수 정렬 의사코드

```text
counting_sort(리스트):
    만약 리스트의 길이가 0 또는 1이면:
        이미 정렬된 상태이므로 그대로 반환한다

    리스트에서 가장 큰 값을 찾는다

    0부터 가장 큰 값까지 담을 수 있는 카운트 배열을 만든다

    리스트 안의 값을 하나씩 확인한다:
        해당 값의 개수를 1 늘린다

    결과 리스트를 만든다

    카운트 배열을 처음부터 끝까지 확인한다:
        현재 숫자가 나온 개수만큼 결과 리스트에 넣는다

    결과 리스트를 반환한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function counting_sort(arr):
    if length(arr) <= 1:
        return arr

    max_value = maximum value in arr
    count = array of 0 with length max_value + 1

    for value in arr:
        count[value] = count[value] + 1

    result = []

    for number from 0 to max_value:
        repeat count[number] times:
            add number to result

    return result
```

## 흐름을 쉽게 풀어보기

계수 정렬은 이렇게 생각하면 쉽다.

```text
1. 숫자가 몇 번 나왔는지 기록할 배열을 만든다.
2. 원래 리스트를 처음부터 끝까지 확인한다.
3. 숫자를 만날 때마다 해당 숫자의 개수를 1 늘린다.
4. 작은 숫자부터 차례대로 확인한다.
5. 나온 개수만큼 결과 리스트에 넣는다.
6. 결과 리스트를 반환한다.
```

즉, 두 값을 비교해서 더 작은 값을 앞으로 보내는 방식이 아니다.

각 숫자의 등장 횟수를 세고, 작은 숫자부터 다시 펼쳐서 정렬한다.

## 예시

```text
counting_sort([5, 3, 8, 4, 2])
```

가장 큰 값은 `8`이다.

그래서 `0`부터 `8`까지 개수를 저장할 수 있는 카운트 배열을 만든다.

```text
number: 0 1 2 3 4 5 6 7 8
count : 0 0 0 0 0 0 0 0 0
```

리스트의 값을 하나씩 세면 다음과 같다.

```text
5를 만난다 -> count[5] = 1
3을 만난다 -> count[3] = 1
8을 만난다 -> count[8] = 1
4를 만난다 -> count[4] = 1
2를 만난다 -> count[2] = 1
```

카운트 배열은 다음처럼 된다.

```text
number: 0 1 2 3 4 5 6 7 8
count : 0 0 1 1 1 1 0 0 1
```

이제 작은 숫자부터 개수만큼 꺼낸다.

```text
2가 1번 나왔으므로 2를 넣는다 -> [2]
3이 1번 나왔으므로 3을 넣는다 -> [2, 3]
4가 1번 나왔으므로 4를 넣는다 -> [2, 3, 4]
5가 1번 나왔으므로 5를 넣는다 -> [2, 3, 4, 5]
8이 1번 나왔으므로 8을 넣는다 -> [2, 3, 4, 5, 8]
```

정렬된 결과는 다음과 같다.

```text
[2, 3, 4, 5, 8]
```

## 종료 조건

리스트에 값이 없거나 하나만 있으면 더 이상 정렬할 필요가 없다.

```text
if length(arr) <= 1:
    return arr
```

예를 들어:

```text
[]      이미 정렬됨
[7]     이미 정렬됨
```

그래서 그대로 반환한다.

## 카운트 배열 이해하기

카운트 배열은 숫자의 값을 인덱스로 사용한다.

예를 들어 값 `3`이 나오면 `count[3]`을 1 늘린다.

```text
count[3] = count[3] + 1
```

다음 리스트가 있다고 하자.

```text
[3, 1, 3, 2, 5]
```

값을 모두 세면 다음과 같다.

```text
number: 0 1 2 3 4 5
count : 0 1 1 2 0 1
```

이 뜻은 다음과 같다.

```text
1은 1번 나왔다.
2는 1번 나왔다.
3은 2번 나왔다.
5는 1번 나왔다.
```

그래서 작은 숫자부터 다시 쓰면 다음처럼 정렬된다.

```text
[1, 2, 3, 3, 5]
```

## 음수가 있는 경우

기본적인 계수 정렬은 `0` 이상의 정수를 기준으로 설명하는 경우가 많다.

음수가 있으면 그대로 `count[-3]`처럼 쓰기 어렵기 때문에 값을 옮겨서 생각해야 한다.

예를 들어 가장 작은 값이 `-2`라면 모든 값에 `2`를 더해서 인덱스로 사용할 수 있다.

```text
원래 값: -2 -1  0  1
인덱스:  0  1  2  3
```

즉, 실제 코드에서는 다음처럼 보정값을 사용할 수 있다.

```text
index = value - min_value
```

## 시간 복잡도

계수 정렬의 시간 복잡도는 다음과 같다.

```text
O(n + k)
```

여기서 `n`은 정렬할 값의 개수이고, `k`는 값의 범위이다.

예를 들어 값이 `0`부터 `8`까지만 나오면 `k`는 9 정도로 볼 수 있다.

리스트를 한 번 돌면서 개수를 세고, 카운트 배열을 한 번 돌면서 결과를 만들기 때문에 `O(n + k)`가 된다.

하지만 값의 범위가 너무 크면 카운트 배열도 커진다.

예를 들어 값이 3개뿐이어도 다음처럼 범위가 크면 비효율적일 수 있다.

```text
[1, 1000000, 5000000]
```

## 장점

```text
값의 범위가 작으면 매우 빠르다.
값을 비교하지 않고 정렬할 수 있다.
같은 값이 많이 반복되는 데이터에서 이해하기 좋다.
```

## 단점

```text
값의 범위가 크면 카운트 배열이 너무 커질 수 있다.
정수처럼 개수를 셀 수 있는 값에 주로 사용한다.
음수가 있으면 인덱스를 보정해야 한다.
```

## 한 문장 요약

계수 정렬은 각 숫자가 몇 번 나왔는지 센 뒤, 작은 숫자부터 나온 개수만큼 다시 적어서 정렬하는 알고리즘이다.

## 알고리즘 흐름도

아래 흐름은 계수 정렬을 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
정렬할 리스트를 받는다
  |
  v
리스트 길이가 0 또는 1인가?
  |
  +-- 예 --> 리스트를 그대로 반환한다
  |
  +-- 아니오
        |
        v
      리스트에서 가장 큰 값을 찾는다
        |
        v
      카운트 배열을 만든다
        |
        v
      리스트의 값을 처음부터 끝까지 하나씩 확인한다
        |
        v
      현재 값에 해당하는 카운트를 1 늘린다
        |
        v
      모든 값을 확인했는가?
        |
        +-- 아니오 --> 다음 값을 확인한다
        |
        +-- 예
              |
              v
            작은 숫자부터 카운트 배열을 확인한다
              |
              v
            현재 숫자가 나온 개수만큼 결과 리스트에 넣는다
              |
              v
            카운트 배열을 끝까지 확인했는가?
              |
              +-- 아니오 --> 다음 숫자를 확인한다
              |
              +-- 예 --> 결과 리스트를 반환한다
                    |
                    v
                  끝
```

## 더 짧게 외우는 흐름

```text
1. 리스트를 받는다.
2. 길이가 1 이하이면 그대로 반환한다.
3. 가장 큰 값을 찾는다.
4. 숫자의 개수를 저장할 카운트 배열을 만든다.
5. 각 숫자가 몇 번 나왔는지 센다.
6. 작은 숫자부터 개수만큼 결과 리스트에 넣는다.
7. 결과 리스트를 반환한다.
```

## 머릿속 결정 순서

계수 정렬을 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
이 리스트는 이미 정렬된 상태인가?
  -> 길이가 0 또는 1이면 그대로 반환

숫자의 범위는 어디까지인가?
  -> 가장 큰 값 찾기

각 숫자는 몇 번 나왔는가?
  -> count[value]를 1씩 늘리기

어떤 순서로 결과를 만들 것인가?
  -> 작은 숫자부터 확인하기

현재 숫자를 몇 번 넣어야 하는가?
  -> count[number]만큼 result에 넣기

최종 결과는 어떻게 만드는가?
  -> result 반환
```
