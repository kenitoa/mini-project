# 병합 정렬

병합 정렬은 리스트를 계속 반으로 나눈 뒤, 나뉜 작은 리스트들을 정렬하면서 다시 합치는 방식으로 정렬한다.

이때 두 정렬된 리스트를 하나의 정렬된 리스트로 합치는 과정을 `병합(merge)`이라고 부른다.

## 핵심 생각

예를 들어 이런 숫자들이 있다고 하자.

```text
[5, 3, 8, 4, 2, 7, 1, 10]
```

병합 정렬은 먼저 리스트를 반으로 나눈다.

```text
[5, 3, 8, 4]    [2, 7, 1, 10]
```

나뉜 리스트도 다시 반으로 나눈다.

```text
[5, 3]  [8, 4]  [2, 7]  [1, 10]
```

더 나누면 값이 하나씩 남는다.

```text
[5] [3] [8] [4] [2] [7] [1] [10]
```

값이 하나만 있는 리스트는 이미 정렬된 상태라고 볼 수 있다.

이제 두 개씩 비교하면서 정렬된 상태로 합친다.

```text
[5] + [3]      -> [3, 5]
[8] + [4]      -> [4, 8]
[2] + [7]      -> [2, 7]
[1] + [10]     -> [1, 10]
```

다시 합친다.

```text
[3, 5] + [4, 8]      -> [3, 4, 5, 8]
[2, 7] + [1, 10]     -> [1, 2, 7, 10]
```

마지막으로 한 번 더 합치면 정렬된 결과가 된다.

```text
[3, 4, 5, 8] + [1, 2, 7, 10]
-> [1, 2, 3, 4, 5, 7, 8, 10]
```

## 병합 정렬 의사코드

```text
merge_sort(리스트):
    만약 리스트의 길이가 0 또는 1이면:
        이미 정렬된 상태이므로 그대로 반환한다

    리스트를 가운데 기준으로 왼쪽과 오른쪽으로 나눈다

    왼쪽 리스트를 merge_sort로 정렬한다
    오른쪽 리스트를 merge_sort로 정렬한다

    정렬된 왼쪽 리스트와 정렬된 오른쪽 리스트를 병합한다

    병합한 리스트를 반환한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function merge_sort(arr):
    if length(arr) <= 1:
        return arr

    mid = length(arr) / 2

    left = arr from start to mid
    right = arr from mid to end

    sorted_left = merge_sort(left)
    sorted_right = merge_sort(right)

    return merge(sorted_left, sorted_right)
```

병합하는 함수는 이렇게 생각할 수 있다.

```text
function merge(left, right):
    result = []

    while left and right both have values:
        if first value of left <= first value of right:
            move first value of left to result
        else:
            move first value of right to result

    move remaining values of left to result
    move remaining values of right to result

    return result
```

## 흐름을 쉽게 풀어보기

병합 정렬은 이렇게 생각하면 쉽다.

```text
1. 리스트를 반으로 나눈다.
2. 더 이상 나눌 수 없을 때까지 계속 나눈다.
3. 값이 하나씩 남으면 이미 정렬된 상태로 본다.
4. 두 리스트의 앞쪽 값부터 비교한다.
5. 더 작은 값을 먼저 꺼내 새 리스트에 넣는다.
6. 남은 값까지 모두 넣으면 병합이 끝난다.
```

즉, 처음부터 값을 하나하나 제자리에 넣으려는 것이 아니다.

먼저 아주 작은 단위까지 나누고, 정렬된 작은 묶음들을 차례대로 합치면서 전체를 정렬한다.

## 예시

```text
merge_sort([5, 3, 8, 4, 2])
```

가운데를 기준으로 나눈다.

```text
left = [5, 3]
right = [8, 4, 2]
```

왼쪽을 다시 나눈다.

```text
merge_sort([5, 3])

left = [5]
right = [3]
```

값이 하나씩 있으므로 이제 합친다.

```text
[5] + [3] = [3, 5]
```

오른쪽도 다시 나눈다.

```text
merge_sort([8, 4, 2])

left = [8]
right = [4, 2]
```

`[4, 2]`도 다시 나눈다.

```text
left = [4]
right = [2]
```

합치면 다음과 같다.

```text
[4] + [2] = [2, 4]
```

이제 `[8]`과 `[2, 4]`를 합친다.

```text
[8] + [2, 4] = [2, 4, 8]
```

처음 문제로 돌아가면:

```text
merge_sort([5, 3, 8, 4, 2])

= merge_sort([5, 3]) + merge_sort([8, 4, 2])
= [3, 5] + [2, 4, 8]
= [2, 3, 4, 5, 8]
```

여기서 `+`는 그냥 이어 붙인다는 뜻이 아니라, 작은 값부터 비교하면서 정렬된 상태로 병합한다는 뜻이다.

## 종료 조건

병합 정렬도 계속 자기 자신을 다시 호출한다.

이런 방식을 `재귀`라고 한다.

그런데 계속 나누기만 하면 끝나지 않으므로, 멈추는 조건이 필요하다.

```text
if length(arr) <= 1:
    return arr
```

리스트에 값이 없거나 하나만 있으면 더 이상 정렬할 필요가 없다.

예를 들어:

```text
[]      이미 정렬됨
[7]     이미 정렬됨
```

그래서 그대로 반환한다.

## 병합 과정

병합 정렬에서 가장 중요한 부분은 두 정렬된 리스트를 합치는 과정이다.

예를 들어 다음 두 리스트가 있다고 하자.

```text
left = [3, 5, 8]
right = [1, 4, 7]
```

두 리스트의 맨 앞 값을 비교한다.

```text
3과 1을 비교한다 -> 1이 더 작다
result = [1]
```

이제 `right`의 다음 값과 `left`의 맨 앞 값을 비교한다.

```text
3과 4를 비교한다 -> 3이 더 작다
result = [1, 3]
```

계속 반복한다.

```text
5와 4를 비교한다 -> 4가 더 작다
result = [1, 3, 4]

5와 7을 비교한다 -> 5가 더 작다
result = [1, 3, 4, 5]

8과 7을 비교한다 -> 7이 더 작다
result = [1, 3, 4, 5, 7]
```

한쪽 리스트가 비면 남은 값은 그대로 뒤에 붙인다.

```text
result = [1, 3, 4, 5, 7, 8]
```

## 시간 복잡도

병합 정렬의 시간 복잡도는 다음과 같다.

```text
O(n log n)
```

리스트를 반으로 나누는 단계가 `log n`번 정도 생기고, 각 단계마다 전체 값을 한 번씩 병합하기 때문이다.

퀵 정렬은 피벗 선택에 따라 최악의 경우 느려질 수 있지만, 병합 정렬은 보통 다음 시간 복잡도를 안정적으로 유지한다.

```text
O(n log n)
```

## 장점

```text
항상 안정적으로 빠른 편이다.
큰 데이터에서도 시간 복잡도가 일정하게 좋다.
분할 정복 방식을 이해하기 좋은 정렬이다.
```

## 단점

```text
병합할 때 추가 리스트가 필요하다.
작은 데이터에서는 단순한 정렬보다 복잡하게 느껴질 수 있다.
재귀를 사용하므로 호출이 많이 쌓일 수 있다.
```

## 한 문장 요약

병합 정렬은 리스트를 계속 반으로 나눈 뒤, 정렬된 작은 리스트들을 비교하면서 다시 합쳐 전체를 정렬하는 알고리즘이다.

## 알고리즘 흐름도

아래 흐름은 병합 정렬을 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
정렬할 리스트를 받는다
  |
  v
리스트 길이가 0 또는 1인가?
  |
  +-- 예 --> 리스트를 그대로 반환한다
  |
  +-- 아니오
        |
        v
      리스트의 가운데 위치를 찾는다
        |
        v
      리스트를 왼쪽과 오른쪽으로 나눈다
        |
        +--> left  : 앞쪽 절반
        +--> right : 뒤쪽 절반
        |
        v
      left를 merge_sort로 다시 정렬한다
        |
        v
      right를 merge_sort로 다시 정렬한다
        |
        v
      정렬된 left와 정렬된 right를 병합한다
        |
        v
      병합한 리스트를 반환한다
        |
        v
      끝
```

## 더 짧게 외우는 흐름

```text
1. 리스트를 받는다.
2. 길이가 1 이하이면 그대로 반환한다.
3. 가운데를 기준으로 왼쪽과 오른쪽으로 나눈다.
4. 왼쪽과 오른쪽을 각각 병합 정렬한다.
5. 정렬된 왼쪽과 오른쪽을 병합한다.
6. 병합한 리스트를 반환한다.
```

## 머릿속 결정 순서

병합 정렬을 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
이 리스트는 이미 정렬된 상태인가?
  -> 길이가 0 또는 1이면 그대로 반환

어디를 기준으로 나눌 것인가?
  -> 가운데 위치 선택

왼쪽과 오른쪽은 각각 정렬되어 있는가?
  -> merge_sort(left), merge_sort(right)

두 정렬된 리스트를 어떻게 합칠 것인가?
  -> 앞쪽 값끼리 비교해서 작은 값부터 result에 넣기

최종 결과는 어떻게 만드는가?
  -> merge(sorted_left, sorted_right)
```
