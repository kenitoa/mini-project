# 퀵 정렬

퀵 정렬은 기준값을 하나 정한 뒤, 그 기준값보다 작은 값은 왼쪽에, 큰 값은 오른쪽에 모으는 방식으로 정렬한다.

이때 기준값을 `피벗(pivot)`이라고 부른다.

## 핵심 생각

예를 들어 이런 숫자들이 있다고 하자.

```text
[5, 3, 8, 4, 2, 7, 1, 10]
```

여기서 피벗을 하나 고른다.

```text
pivot = 5
```

그러면 숫자들을 세 부분으로 나눌 수 있다.

```text
5보다 작은 값: [3, 4, 2, 1]
5와 같은 값: [5]
5보다 큰 값: [8, 7, 10]
```

작은 값 쪽과 큰 값 쪽도 다시 같은 방식으로 정렬한다.

마지막에 합치면 정렬된 결과가 된다.

```text
정렬된 작은 값 + 피벗 + 정렬된 큰 값
```

## 퀵 정렬 의사코드

```text
quick_sort(리스트):
    만약 리스트의 길이가 0 또는 1이면:
        이미 정렬된 상태이므로 그대로 반환한다

    피벗을 하나 고른다

    피벗보다 작은 값을 담을 리스트를 만든다
    피벗과 같은 값을 담을 리스트를 만든다
    피벗보다 큰 값을 담을 리스트를 만든다

    리스트 안의 값을 하나씩 확인한다:
        만약 값이 피벗보다 작으면:
            작은 값 리스트에 넣는다
        아니고, 값이 피벗보다 크면:
            큰 값 리스트에 넣는다
        아니면:
            같은 값 리스트에 넣는다

    작은 값 리스트를 quick_sort로 정렬한다
    큰 값 리스트를 quick_sort로 정렬한다

    정렬된 작은 값 리스트 + 같은 값 리스트 + 정렬된 큰 값 리스트를 반환한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function quick_sort(arr):
    if length(arr) <= 1:
        return arr

    pivot = arr[0]

    left = []
    middle = []
    right = []

    for value in arr:
        if value < pivot:
            add value to left
        else if value > pivot:
            add value to right
        else:
            add value to middle

    return quick_sort(left) + middle + quick_sort(right)
```

## 흐름을 쉽게 풀어보기

퀵 정렬은 이렇게 생각하면 쉽다.

```text
1. 기준 숫자 하나를 고른다.
2. 기준보다 작은 숫자는 왼쪽으로 보낸다.
3. 기준보다 큰 숫자는 오른쪽으로 보낸다.
4. 왼쪽 묶음과 오른쪽 묶음도 똑같이 정렬한다.
5. 왼쪽 + 기준 + 오른쪽 순서로 합친다.
```

즉, 한 번에 전체를 완벽하게 정렬하려는 것이 아니다.

먼저 피벗을 기준으로 대충 왼쪽과 오른쪽을 나누고, 나뉜 부분을 다시 같은 방법으로 정렬한다.

## 예시

```text
quick_sort([5, 3, 8, 4, 2])
```

피벗을 첫 번째 값인 `5`로 잡는다.

```text
pivot = 5
left = [3, 4, 2]
middle = [5]
right = [8]
```

이제 왼쪽도 다시 퀵 정렬한다.

```text
quick_sort([3, 4, 2])
```

피벗을 `3`으로 잡는다.

```text
pivot = 3
left = [2]
middle = [3]
right = [4]
```

합치면 다음과 같다.

```text
[2] + [3] + [4] = [2, 3, 4]
```

처음 문제로 돌아가면:

```text
quick_sort([5, 3, 8, 4, 2])

= quick_sort([3, 4, 2]) + [5] + quick_sort([8])
= [2, 3, 4] + [5] + [8]
= [2, 3, 4, 5, 8]
```

## 종료 조건

퀵 정렬은 계속 자기 자신을 다시 호출한다.

이런 방식을 `재귀`라고 한다.

그런데 계속 호출만 하면 끝나지 않으므로, 멈추는 조건이 필요하다.

```text
if length(arr) <= 1:
    return arr
```

리스트에 값이 없거나 하나만 있으면 더 이상 정렬할 필요가 없다.

예를 들어:

```text
[]      이미 정렬됨
[7]     이미 정렬됨
```

그래서 그대로 반환한다.

## 피벗을 고르는 방법

피벗은 여러 방식으로 고를 수 있다.

```text
첫 번째 값 고르기
마지막 값 고르기
가운데 값 고르기
랜덤으로 고르기
```

가장 이해하기 쉬운 방법은 첫 번째 값을 피벗으로 고르는 것이다.

```text
pivot = arr[0]
```

하지만 이미 거의 정렬된 리스트에서는 첫 번째 값이나 마지막 값을 피벗으로 고르면 성능이 나빠질 수 있다.

실제로는 가운데 값이나 랜덤 값을 피벗으로 고르기도 한다.

## 시간 복잡도

퀵 정렬의 평균 시간 복잡도는 다음과 같다.

```text
O(n log n)
```

하지만 피벗이 계속 한쪽으로 치우치게 선택되면 최악의 경우 다음처럼 느려질 수 있다.

```text
O(n^2)
```

예를 들어 이미 정렬된 리스트에서 매번 첫 번째 값을 피벗으로 고르면 나누기가 균형 있게 되지 않는다.

## 장점

```text
평균적으로 빠르다.
큰 데이터에서도 잘 동작한다.
분할 정복 방식을 이해하기 좋은 정렬이다.
```

## 단점

```text
피벗 선택이 나쁘면 느려질 수 있다.
재귀를 사용하므로 호출이 많이 쌓일 수 있다.
구현 방식에 따라 추가 리스트가 필요할 수 있다.
```

## 한 문장 요약

퀵 정렬은 피벗을 기준으로 작은 값과 큰 값을 나누고, 나뉜 부분을 다시 같은 방식으로 정렬한 뒤 합치는 정렬이다.

## 알고리즘 흐름도

아래 흐름은 퀵 정렬을 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
정렬할 리스트를 받는다
  |
  v
리스트 길이가 0 또는 1인가?
  |
  +-- 예 --> 리스트를 그대로 반환한다
  |
  +-- 아니오
        |
        v
      피벗을 하나 고른다
        |
        v
      빈 리스트 3개를 만든다
        |
        +--> left   : 피벗보다 작은 값
        +--> middle : 피벗과 같은 값
        +--> right  : 피벗보다 큰 값
        |
        v
      리스트의 값을 처음부터 끝까지 하나씩 확인한다
        |
        v
      현재 값이 피벗보다 작은가?
        |
        +-- 예 --> left에 넣는다
        |
        +-- 아니오
              |
              v
            현재 값이 피벗보다 큰가?
              |
              +-- 예 --> right에 넣는다
              |
              +-- 아니오 --> middle에 넣는다
        |
        v
      모든 값을 확인했는가?
        |
        +-- 아니오 --> 다음 값을 확인한다
        |
        +-- 예
              |
              v
            left를 quick_sort로 다시 정렬한다
              |
              v
            right를 quick_sort로 다시 정렬한다
              |
              v
            정렬된 left + middle + 정렬된 right를 합친다
              |
              v
            합친 리스트를 반환한다
              |
              v
            끝
```

## 더 짧게 외우는 흐름

```text
1. 리스트를 받는다.
2. 길이가 1 이하이면 그대로 반환한다.
3. 피벗을 고른다.
4. 피벗보다 작은 값, 같은 값, 큰 값으로 나눈다.
5. 작은 값 묶음과 큰 값 묶음을 다시 퀵 정렬한다.
6. 정렬된 작은 값 + 같은 값 + 정렬된 큰 값을 합쳐 반환한다.
```

## 머릿속 결정 순서

퀵 정렬을 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
이 리스트는 이미 정렬된 상태인가?
  -> 길이가 0 또는 1이면 그대로 반환

기준값은 무엇인가?
  -> 피벗 선택

각 값은 피벗보다 작은가, 같은가, 큰가?
  -> left, middle, right로 분리

나뉜 왼쪽과 오른쪽도 정렬되어 있는가?
  -> quick_sort(left), quick_sort(right)

최종 결과는 어떻게 만드는가?
  -> quick_sort(left) + middle + quick_sort(right)
```
