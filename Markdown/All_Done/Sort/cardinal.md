# 기수 정렬

기수 정렬은 숫자를 한 번에 직접 비교하지 않고, 각 자리수를 기준으로 여러 번 정렬하는 방식이다.

보통 1의 자리부터 시작해서 10의 자리, 100의 자리처럼 낮은 자리수에서 높은 자리수로 올라가며 정렬한다.

이때 각 자리수를 기준으로 정렬할 때는 보통 `계수 정렬(counting sort)`처럼 안정적인 정렬을 사용한다.

## 핵심 생각

예를 들어 이런 숫자들이 있다고 하자.

```text
[170, 45, 75, 90, 802, 24, 2, 66]
```

기수 정렬은 먼저 1의 자리만 보고 정렬한다.

```text
170 -> 0
45  -> 5
75  -> 5
90  -> 0
802 -> 2
24  -> 4
2   -> 2
66  -> 6
```

1의 자리 기준으로 정렬하면 다음처럼 된다.

```text
[170, 90, 802, 2, 24, 45, 75, 66]
```

그다음 10의 자리만 보고 다시 정렬한다.

```text
[802, 2, 24, 45, 66, 170, 75, 90]
```

마지막으로 100의 자리만 보고 정렬한다.

```text
[2, 24, 45, 66, 75, 90, 170, 802]
```

즉, 기수 정렬은 숫자 전체를 바로 비교하는 것이 아니라 자리수를 하나씩 기준으로 삼아 정렬을 반복한다.

## 기수 정렬 의사코드

```text
radix_sort(리스트):
    만약 리스트의 길이가 0 또는 1이라면:
        이미 정렬된 상태이므로 그대로 반환한다

    리스트에서 가장 큰 값을 찾는다
    자리수를 1로 시작한다

    가장 큰 값 / 자리수가 0보다 큰 동안 반복한다:
        현재 자리수를 기준으로 리스트를 안정 정렬한다
        자리수를 10배 키운다

    리스트를 반환한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function radix_sort(arr):
    if length(arr) <= 1:
        return arr

    max_value = maximum value in arr
    digit = 1

    while max_value / digit > 0:
        arr = counting_sort_by_digit(arr, digit)
        digit = digit * 10

    return arr
```

자리수 기준 계수 정렬은 이렇게 생각할 수 있다.

```text
function counting_sort_by_digit(arr, digit):
    count = array of 0 with length 10
    result = array with same length as arr

    for value in arr:
        digit_value = (value / digit) % 10
        count[digit_value] = count[digit_value] + 1

    for i from 1 to 9:
        count[i] = count[i] + count[i - 1]

    for i from length(arr) - 1 down to 0:
        digit_value = (arr[i] / digit) % 10
        result[count[digit_value] - 1] = arr[i]
        count[digit_value] = count[digit_value] - 1

    return result
```

## 흐름을 쉽게 떠올려보기

기수 정렬은 이렇게 생각하면 쉽다.

```text
1. 가장 큰 숫자의 자리수를 확인한다.
2. 1의 자리부터 정렬한다.
3. 10의 자리로 이동해서 다시 정렬한다.
4. 100의 자리로 이동해서 다시 정렬한다.
5. 가장 큰 숫자의 마지막 자리까지 반복한다.
6. 최종 리스트를 반환한다.
```

즉, 처음부터 숫자 전체를 비교해서 위치를 바꾸는 것이 아니다.

낮은 자리수부터 차례대로 정렬하면서 전체 순서가 점점 맞춰지도록 만드는 방식이다.

## 예시

```text
radix_sort([170, 45, 75, 90, 802, 24, 2, 66])
```

가장 큰 값은 `802`이다.

그러므로 1의 자리, 10의 자리, 100의 자리까지 확인하면 된다.

먼저 1의 자리를 기준으로 정렬한다.

```text
170 -> 0
45  -> 5
75  -> 5
90  -> 0
802 -> 2
24  -> 4
2   -> 2
66  -> 6
```

1의 자리 기준으로 정렬하면 다음과 같다.

```text
[170, 90, 802, 2, 24, 45, 75, 66]
```

그다음 10의 자리를 기준으로 정렬한다.

```text
170 -> 7
90  -> 9
802 -> 0
2   -> 0
24  -> 2
45  -> 4
75  -> 7
66  -> 6
```

10의 자리 기준으로 정렬하면 다음과 같다.

```text
[802, 2, 24, 45, 66, 170, 75, 90]
```

마지막으로 100의 자리를 기준으로 정렬한다.

```text
802 -> 8
2   -> 0
24  -> 0
45  -> 0
66  -> 0
170 -> 1
75  -> 0
90  -> 0
```

100의 자리 기준으로 정렬하면 최종 결과는 다음과 같다.

```text
[2, 24, 45, 66, 75, 90, 170, 802]
```

## 종료 조건

리스트에 값이 없거나 하나만 있으면 더 이상 정렬할 필요가 없다.

```text
if length(arr) <= 1:
    return arr
```

예를 들어:

```text
[]      이미 정렬됨
[7]     이미 정렬됨
```

그래서 그대로 반환한다.

또한 가장 큰 숫자의 마지막 자리까지 정렬했다면 더 이상 확인할 자리수가 없다.

```text
while max_value / digit > 0:
    현재 자리수를 기준으로 정렬한다
```

`digit`이 1, 10, 100처럼 커지다가 가장 큰 값보다 커지면 반복을 멈춘다.

## 자리수 이해하기

기수 정렬에서 가장 중요한 부분은 현재 확인할 자리수를 구하는 것이다.

예를 들어 `170`이라는 값이 있다고 하자.

```text
1의 자리   = (170 / 1) % 10   = 0
10의 자리  = (170 / 10) % 10  = 7
100의 자리 = (170 / 100) % 10 = 1
```

즉, 현재 자리수를 `digit`이라고 하면 해당 자리의 값은 다음처럼 구할 수 있다.

```text
digit_value = (value / digit) % 10
```

`digit`이 `1`이면 1의 자리, `10`이면 10의 자리, `100`이면 100의 자리를 뜻한다.

## 안정 정렬이 필요한 이유

기수 정렬에서는 각 자리수를 기준으로 정렬할 때 안정 정렬을 사용해야 한다.

안정 정렬은 같은 값을 가진 원소들의 기존 순서를 유지하는 정렬이다.

예를 들어 1의 자리 기준으로 정렬한 뒤 10의 자리 기준으로 다시 정렬할 때, 같은 10의 자리 값을 가진 숫자들은 이전 단계에서 맞춰진 1의 자리 순서를 유지해야 한다.

그래야 낮은 자리수에서 맞춰 둔 순서가 높은 자리수 정렬 과정에서 깨지지 않는다.

그래서 기수 정렬에서는 자리수별 정렬에 계수 정렬을 자주 사용한다.

## 시간 복잡도

기수 정렬의 시간 복잡도는 다음과 같다.

```text
O(d * (n + k))
```

여기서 `n`은 정렬할 값의 개수이고, `d`는 가장 큰 숫자의 자리수 개수이다.

`k`는 한 자리에서 나올 수 있는 값의 개수이다.

10진수 숫자를 정렬한다면 각 자리에는 `0`부터 `9`까지만 나올 수 있으므로 `k`는 10이다.

그래서 보통은 다음처럼 생각할 수 있다.

```text
O(d * n)
```

숫자의 자리수가 아주 크지 않다면 빠르게 동작할 수 있다.

## 장점

```text
비교를 사용하지 않고 정렬할 수 있다.
숫자의 자리수가 크지 않으면 매우 빠르게 동작한다.
계수 정렬을 자리수별로 활용하는 흐름을 이해하기 좋다.
```

## 단점

```text
정렬할 값의 자리수를 확인해야 한다.
자리수별로 안정 정렬이 필요하다.
음수가 섞인 경우에는 별도의 처리가 필요하다.
문자열이나 특수한 데이터는 기준을 정해 주어야 한다.
```

## 한 문장 요약

기수 정렬은 숫자를 1의 자리, 10의 자리, 100의 자리처럼 낮은 자리수부터 차례대로 안정 정렬해서 전체를 정렬하는 알고리즘이다.

## 알고리즘 흐름도

아래 흐름은 기수 정렬을 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
정렬할 리스트를 받는다
  |
  v
리스트의 길이가 0 또는 1인가?
  |
  +-- 예 --> 리스트를 그대로 반환한다
  |
  +-- 아니오
        |
        v
      리스트에서 가장 큰 값을 찾는다
        |
        v
      digit을 1로 설정한다
        |
        v
      가장 큰 값 / digit이 0보다 큰가?
        |
        +-- 아니오 --> 리스트를 반환한다
        |
        +-- 예
              |
              v
            현재 digit 자리의 값을 기준으로 안정 정렬한다
              |
              v
            digit을 10배 키운다
              |
              v
            다시 반복 조건을 확인한다
              |
              v
            끝
```

## 더 짧게 외우는 흐름

```text
1. 리스트를 받는다.
2. 길이가 1 이하라면 그대로 반환한다.
3. 가장 큰 값을 찾는다.
4. 1의 자리부터 안정 정렬한다.
5. 10의 자리, 100의 자리처럼 자리수를 키운다.
6. 가장 큰 값의 마지막 자리까지 반복한다.
7. 리스트를 반환한다.
```

## 머릿속 결정 순서

기수 정렬을 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
이 리스트는 이미 정렬된 상태인가?
  -> 길이가 0 또는 1이면 그대로 반환

몇 자리까지 확인해야 하는가?
  -> 가장 큰 값 찾기

지금 어떤 자리수를 기준으로 정렬하는가?
  -> digit으로 1, 10, 100 순서로 확인

현재 자리의 숫자는 어떻게 구하는가?
  -> (value / digit) % 10

자리수 기준 정렬은 어떻게 해야 하는가?
  -> 안정 정렬 사용

언제 끝나는가?
  -> digit이 가장 큰 값보다 커지면 끝
```
