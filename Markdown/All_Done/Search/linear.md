# 선형 탐색

선형 탐색은 리스트의 첫 번째 값부터 마지막 값까지 차례대로 확인하면서 원하는 값을 찾는 탐색 방법이다.

값을 건너뛰거나 특별한 규칙으로 이동하지 않고, 앞에서부터 하나씩 직접 비교한다.

그래서 리스트가 정렬되어 있지 않아도 사용할 수 있다.

## 핵심 생각

예를 들어 이런 숫자들이 있다고 하자.

```text
[4, 9, 2, 7, 5]
```

여기서 `7`을 찾고 싶다면 선형 탐색은 앞에서부터 하나씩 확인한다.

```text
4 == 7 ? 아니오
9 == 7 ? 아니오
2 == 7 ? 아니오
7 == 7 ? 예
```

원하는 값인 `7`을 찾았으므로 그 위치를 반환한다.

이때 `7`은 인덱스 `3`에 있다.

즉, 선형 탐색은 값을 하나씩 비교하다가 찾는 값과 같아지는 순간 탐색을 멈춘다.

## 선형 탐색 의사코드

```text
linear_search(리스트, 찾는 값):
    리스트의 첫 번째 위치부터 마지막 위치까지 반복한다:
        현재 위치의 값이 찾는 값과 같다면:
            현재 위치를 반환한다

    끝까지 찾지 못했다면:
        찾지 못했다는 값을 반환한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function linear_search(arr, target):
    for i from 0 to length(arr) - 1:
        if arr[i] == target:
            return i

    return -1
```

여기서 `-1`은 원하는 값을 찾지 못했다는 뜻으로 자주 사용된다.

실제 코드에서는 상황에 따라 `false`, `null`, `None` 같은 값을 반환할 수도 있다.

## 흐름을 쉽게 떠올려보기

선형 탐색은 이렇게 생각하면 쉽다.

```text
1. 찾고 싶은 값을 정한다.
2. 리스트의 첫 번째 값부터 확인한다.
3. 현재 값이 찾는 값과 같은지 비교한다.
4. 같다면 현재 위치를 반환하고 끝낸다.
5. 다르다면 다음 값으로 이동한다.
6. 리스트의 끝까지 반복한다.
7. 끝까지 없으면 찾지 못했다고 반환한다.
```

즉, 선형 탐색은 한 칸씩 앞으로 이동하면서 값을 직접 확인하는 방식이다.

## 예시

```text
linear_search([10, 3, 6, 8, 2], 8)
```

찾는 값은 `8`이다.

리스트를 처음부터 하나씩 확인한다.

```text
index 0: 10 == 8 ? 아니오
index 1: 3  == 8 ? 아니오
index 2: 6  == 8 ? 아니오
index 3: 8  == 8 ? 예
```

`8`을 찾았으므로 인덱스 `3`을 반환한다.

```text
return 3
```

## 찾는 값이 없는 예시

```text
linear_search([10, 3, 6, 8, 2], 7)
```

찾는 값은 `7`이다.

```text
index 0: 10 == 7 ? 아니오
index 1: 3  == 7 ? 아니오
index 2: 6  == 7 ? 아니오
index 3: 8  == 7 ? 아니오
index 4: 2  == 7 ? 아니오
```

리스트의 끝까지 확인했지만 `7`이 없다.

그래서 찾지 못했다는 의미로 `-1`을 반환한다.

```text
return -1
```

## 종료 조건

선형 탐색이 끝나는 경우는 크게 두 가지다.

첫 번째는 원하는 값을 찾은 경우다.

```text
if arr[i] == target:
    return i
```

값을 찾는 순간 더 이상 뒤를 확인할 필요가 없으므로 바로 종료한다.

두 번째는 리스트의 끝까지 확인했지만 값을 찾지 못한 경우다.

```text
return -1
```

예를 들어:

```text
linear_search([1, 2, 3], 5)
```

`5`는 리스트 안에 없으므로 끝까지 확인한 뒤 `-1`을 반환한다.

## 빈 리스트인 경우

리스트에 값이 하나도 없다면 확인할 값도 없다.

```text
linear_search([], 3)
```

반복문이 한 번도 실행되지 않고 바로 찾지 못했다는 값을 반환한다.

```text
return -1
```

따라서 선형 탐색은 빈 리스트에서도 자연스럽게 동작한다.

## 같은 값이 여러 개 있는 경우

선형 탐색은 보통 가장 먼저 발견한 위치를 반환한다.

예를 들어:

```text
linear_search([5, 1, 5, 3, 5], 5)
```

앞에서부터 확인하면 첫 번째 값이 바로 `5`다.

```text
index 0: 5 == 5 ? 예
```

그래서 인덱스 `0`을 반환한다.

```text
return 0
```

뒤에 있는 다른 `5`들은 확인하지 않는다.

만약 같은 값을 가진 모든 위치를 찾고 싶다면, 찾았을 때 바로 종료하지 않고 끝까지 확인해야 한다.

```text
function linear_search_all(arr, target):
    result = []

    for i from 0 to length(arr) - 1:
        if arr[i] == target:
            add i to result

    return result
```

예를 들어:

```text
linear_search_all([5, 1, 5, 3, 5], 5)
```

결과는 다음과 같다.

```text
[0, 2, 4]
```

## 정렬 여부와의 관계

선형 탐색은 리스트가 정렬되어 있지 않아도 사용할 수 있다.

```text
[9, 1, 7, 3, 5]
```

이런 리스트에서도 앞에서부터 하나씩 확인하면 원하는 값을 찾을 수 있다.

정렬된 리스트에서도 사용할 수 있다.

```text
[1, 3, 5, 7, 9]
```

하지만 정렬된 리스트라면 이진 탐색처럼 더 빠른 방법을 사용할 수 있는 경우가 많다.

그래도 선형 탐색은 가장 단순하고, 데이터의 상태에 거의 영향을 받지 않는다는 장점이 있다.

## 시간 복잡도

선형 탐색의 시간 복잡도는 경우에 따라 다르게 생각할 수 있다.

찾는 값이 첫 번째에 있다면 한 번만 비교하면 된다.

```text
O(1)
```

예를 들어:

```text
linear_search([7, 2, 5, 9], 7)
```

첫 번째 값이 바로 `7`이므로 가장 빠르게 끝난다.

찾는 값이 마지막에 있거나 아예 없다면 리스트 전체를 확인해야 한다.

```text
O(n)
```

예를 들어:

```text
linear_search([7, 2, 5, 9], 9)
linear_search([7, 2, 5, 9], 1)
```

두 경우 모두 끝부분까지 확인해야 한다.

그래서 선형 탐색의 시간 복잡도는 보통 다음처럼 말한다.

```text
최선의 경우: O(1)
평균의 경우: O(n)
최악의 경우: O(n)
```

## 공간 복잡도

기본적인 선형 탐색은 추가 리스트를 만들지 않는다.

현재 확인 중인 위치 정도만 기억하면 된다.

그래서 공간 복잡도는 다음과 같다.

```text
O(1)
```

단, 모든 위치를 찾기 위해 결과 리스트를 따로 만든다면 그 리스트의 크기만큼 추가 공간이 필요할 수 있다.

## 장점

```text
구현이 매우 쉽다.
리스트가 정렬되어 있지 않아도 사용할 수 있다.
배열, 리스트, 문자열 등 여러 자료 구조에 적용하기 쉽다.
데이터의 크기가 작을 때는 충분히 실용적이다.
```

## 단점

```text
데이터가 많아지면 느려질 수 있다.
찾는 값이 뒤쪽에 있으면 많은 비교가 필요하다.
찾는 값이 없을 때는 전체를 끝까지 확인해야 한다.
정렬된 데이터에서는 이진 탐색보다 비효율적일 수 있다.
```

## 한 문장 요약

선형 탐색은 리스트의 처음부터 끝까지 값을 하나씩 비교하면서 원하는 값을 찾는 가장 단순한 탐색 알고리즘이다.

## 알고리즘 흐름도

아래 흐름은 선형 탐색을 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
리스트와 찾는 값을 받는다
  |
  v
현재 위치를 0으로 설정한다
  |
  v
현재 위치가 리스트의 길이보다 작은가?
  |
  +-- 아니오 --> 찾지 못했다는 값을 반환한다
  |
  +-- 예
        |
        v
      현재 위치의 값이 찾는 값과 같은가?
        |
        +-- 예 --> 현재 위치를 반환한다
        |
        +-- 아니오
              |
              v
            현재 위치를 1 증가시킨다
              |
              v
            다시 반복 조건을 확인한다
```

## 더 짧게 외우는 흐름

```text
1. 리스트를 받는다.
2. 찾는 값을 받는다.
3. 첫 번째 값부터 확인한다.
4. 현재 값이 찾는 값과 같으면 현재 위치를 반환한다.
5. 다르면 다음 값으로 이동한다.
6. 끝까지 없으면 -1을 반환한다.
```

## 머릿속 결정 순서

선형 탐색을 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
무엇을 찾고 싶은가?
  -> target 정하기

어디에서 찾을 것인가?
  -> 리스트 또는 배열 받기

어디부터 확인할 것인가?
  -> 인덱스 0부터 시작하기

현재 값이 찾는 값과 같은가?
  -> 같으면 현재 인덱스 반환

다르면 무엇을 할 것인가?
  -> 다음 인덱스로 이동

언제 끝낼 것인가?
  -> 값을 찾거나 리스트 끝까지 갔을 때

끝까지 없으면 어떻게 할 것인가?
  -> -1 같은 실패 값을 반환
```
