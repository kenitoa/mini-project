# DFS

DFS는 Depth First Search의 줄임말이고, 한국어로는 깊이 우선 탐색이라고 부른다.

DFS는 그래프나 트리에서 한 방향으로 갈 수 있는 만큼 깊게 들어간 뒤, 더 이상 갈 곳이 없으면 이전 위치로 되돌아와서 다른 길을 탐색하는 방법이다.

즉, 가까운 곳을 먼저 넓게 살피는 방식이 아니라, 한 길을 끝까지 따라가 보는 방식이다.

DFS는 보통 다음 두 가지 방식으로 구현한다.

```text
재귀 함수
스택
```

재귀 함수를 사용하면 함수 호출이 자연스럽게 스택처럼 동작한다.

직접 스택 자료 구조를 만들어서 DFS를 구현할 수도 있다.

## 핵심 생각

예를 들어 다음과 같은 그래프가 있다고 하자.

```text
A -- B -- D
|    |
C    E
|
F
```

이 그래프를 연결 관계로 나타내면 다음과 같다.

```text
A: B, C
B: A, D, E
C: A, F
D: B
E: B
F: C
```

`A`에서 시작해서 DFS를 한다면 먼저 한 방향으로 깊게 들어간다.

예를 들어 `A`의 이웃을 `B`, `C` 순서로 확인한다고 하자.

```text
A에서 시작
A -> B로 이동
B -> D로 이동
D에서 더 갈 곳이 없음
B로 되돌아감
B -> E로 이동
E에서 더 갈 곳이 없음
B로 되돌아감
A로 되돌아감
A -> C로 이동
C -> F로 이동
```

방문 순서는 다음과 같을 수 있다.

```text
A -> B -> D -> E -> C -> F
```

즉, DFS는 현재 위치에서 갈 수 있는 다음 위치로 계속 들어가다가, 막히면 되돌아오는 방식이다.

## 그래프에서 사용하는 말

DFS를 이해하려면 그래프에서 자주 사용하는 말을 알아두면 좋다.

```text
정점 또는 노드: 데이터가 있는 지점
간선: 정점과 정점을 연결하는 선
인접 정점: 현재 정점에서 바로 갈 수 있는 정점
방문: 어떤 정점에 도착해서 확인했다는 뜻
방문 체크: 이미 방문한 정점을 다시 방문하지 않도록 표시하는 것
```

예를 들어 다음 그래프에서:

```text
A -- B
|
C
```

`A`의 인접 정점은 `B`와 `C`이다.

DFS는 현재 정점의 인접 정점을 하나씩 확인하면서 깊게 이동한다.

## 왜 방문 체크가 필요한가

그래프에는 사이클이 있을 수 있다.

사이클은 다시 자기 자신 쪽으로 돌아올 수 있는 길이다.

예를 들어:

```text
A -- B
|    |
C -- D
```

`A`에서 `B`로 가고, `B`에서 `D`로 가고, `D`에서 `C`로 가고, `C`에서 다시 `A`로 갈 수 있다.

방문 체크를 하지 않으면 다음처럼 같은 정점을 계속 반복해서 방문할 수 있다.

```text
A -> B -> D -> C -> A -> B -> D -> C -> ...
```

그래서 DFS에서는 보통 `visited`라는 집합이나 배열을 사용한다.

```text
visited = 이미 방문한 정점들
```

어떤 정점을 방문하면 `visited`에 기록한다.

그리고 다음 정점으로 이동하기 전에 이미 방문했는지 확인한다.

```text
이미 방문했다면 다시 가지 않는다.
방문하지 않았다면 이동한다.
```

## DFS 의사코드

가장 기본적인 DFS는 현재 정점에서 시작해 갈 수 있는 정점을 모두 방문한다.

```text
dfs(그래프, 현재 정점, 방문 기록):
    현재 정점을 방문했다고 표시한다
    현재 정점을 처리한다

    현재 정점과 연결된 다음 정점들을 하나씩 확인한다:
        다음 정점을 아직 방문하지 않았다면:
            dfs(그래프, 다음 정점, 방문 기록)을 실행한다
```

여기서 현재 정점을 처리한다는 말은 문제에 따라 달라진다.

```text
정점 이름 출력하기
정점 개수 세기
목표 정점인지 확인하기
경로에 추가하기
```

## 조금 더 코드처럼 쓴 의사코드

```text
function dfs(graph, current, visited):
    add current to visited
    print current

    for each next in graph[current]:
        if next is not in visited:
            dfs(graph, next, visited)
```

이 코드는 `current`에서 시작해서 연결된 모든 정점을 깊이 우선으로 방문한다.

`visited`는 이미 방문한 정점을 저장한다.

## 목표 값을 찾는 DFS 의사코드

DFS는 단순히 모든 정점을 방문할 수도 있고, 특정 목표 정점을 찾는 데 사용할 수도 있다.

예를 들어 `target`을 찾는 DFS는 다음처럼 쓸 수 있다.

```text
function dfs_search(graph, current, target, visited):
    add current to visited

    if current == target:
        return current

    for each next in graph[current]:
        if next is not in visited:
            result = dfs_search(graph, next, target, visited)

            if result is not null:
                return result

    return null
```

현재 정점이 찾는 정점이라면 바로 반환한다.

그렇지 않다면 연결된 정점으로 더 깊게 들어가면서 찾는다.

끝까지 찾지 못하면 `null`을 반환한다.

## 흐름을 쉽게 떠올려보기

DFS는 이렇게 생각하면 쉽다.

```text
1. 시작 정점을 정한다.
2. 현재 정점을 방문했다고 표시한다.
3. 현재 정점이 원하는 정점인지 확인한다.
4. 현재 정점에서 갈 수 있는 정점들을 확인한다.
5. 아직 방문하지 않은 정점이 있으면 그 정점으로 깊게 들어간다.
6. 더 이상 갈 곳이 없으면 이전 정점으로 돌아온다.
7. 모든 길을 확인했는데도 없으면 찾지 못했다고 반환한다.
```

즉, DFS는 하나의 길을 끝까지 파고들고, 막히면 되돌아와 다른 길을 확인하는 방식이다.

## 예시

다음 그래프가 있다고 하자.

```text
A: B, C
B: A, D, E
C: A, F
D: B
E: B
F: C
```

`A`에서 시작해서 모든 정점을 DFS로 방문한다고 하자.

```text
dfs(graph, A)
```

이웃 정점을 적힌 순서대로 확인하면 흐름은 다음과 같다.

```text
A 방문
B 방문
D 방문
D에서 더 갈 곳 없음
B로 돌아감
E 방문
E에서 더 갈 곳 없음
B로 돌아감
A로 돌아감
C 방문
F 방문
F에서 더 갈 곳 없음
```

방문 순서는 다음과 같다.

```text
A -> B -> D -> E -> C -> F
```

## 목표 정점을 찾는 예시

같은 그래프에서 `E`를 찾는다고 하자.

```text
dfs_search(graph, A, E)
```

탐색 흐름은 다음과 같다.

```text
A == E ? 아니오
B == E ? 아니오
D == E ? 아니오
D에서 더 갈 곳 없음
B로 돌아감
E == E ? 예
```

`E`를 찾았으므로 탐색을 종료한다.

```text
return E
```

목표를 찾는 문제라면 모든 정점을 끝까지 방문하지 않아도 된다.

찾는 순간 바로 종료할 수 있다.

## 찾는 정점이 없는 예시

다음 그래프에서 `G`를 찾는다고 하자.

```text
A: B, C
B: A, D, E
C: A, F
D: B
E: B
F: C
```

```text
dfs_search(graph, A, G)
```

DFS는 `A`에서 갈 수 있는 모든 정점을 확인한다.

```text
A 방문
B 방문
D 방문
E 방문
C 방문
F 방문
```

하지만 `G`는 그래프 안에 없다.

따라서 끝까지 찾지 못했다는 값을 반환한다.

```text
return null
```

## 종료 조건

DFS가 끝나는 경우는 크게 두 가지다.

첫 번째는 원하는 정점을 찾은 경우다.

```text
if current == target:
    return current
```

목표를 찾았다면 더 이상 다른 길을 확인할 필요가 없으므로 바로 종료한다.

두 번째는 현재 정점에서 더 이상 갈 수 있는 방문하지 않은 정점이 없는 경우다.

```text
for each next in graph[current]:
    if next is not in visited:
        dfs(graph, next, visited)
```

반복문을 끝까지 돌았는데 더 갈 곳이 없다면 현재 함수가 끝난다.

재귀 DFS에서는 이때 이전 정점으로 되돌아간다.

## 빈 그래프인 경우

그래프에 정점이 하나도 없다면 DFS를 시작할 정점도 없다.

```text
graph = {}
```

이 경우에는 탐색을 실행할 수 없으므로 바로 찾지 못했다고 처리한다.

```text
return null
```

실제 코드에서는 DFS를 시작하기 전에 시작 정점이 그래프 안에 있는지 확인하는 것이 좋다.

```text
if start is not in graph:
    return null
```

## 시작 정점만 있는 경우

그래프에 시작 정점만 있고 연결된 정점이 없을 수도 있다.

```text
A: empty
```

이때 `A`에서 DFS를 시작하면 `A`만 방문한다.

```text
A 방문
```

찾는 정점이 `A`라면 성공이다.

```text
dfs_search(graph, A, A)
return A
```

찾는 정점이 다른 값이라면 더 갈 곳이 없으므로 실패한다.

```text
dfs_search(graph, A, B)
return null
```

## 연결되지 않은 그래프

그래프는 모든 정점이 하나로 연결되어 있지 않을 수도 있다.

예를 들어:

```text
A -- B

C -- D
```

이 그래프는 두 부분으로 나뉘어 있다.

```text
A: B
B: A
C: D
D: C
```

`A`에서 DFS를 시작하면 `A`와 `B`만 방문할 수 있다.

```text
A -> B
```

`C`와 `D`는 `A`에서 갈 수 없으므로 방문되지 않는다.

그래프 전체의 모든 정점을 방문하고 싶다면 모든 정점을 시작점 후보로 확인해야 한다.

```text
function dfs_all(graph):
    visited = empty set

    for each node in graph:
        if node is not in visited:
            dfs(graph, node, visited)
```

이 방식은 연결되지 않은 그래프에서도 모든 정점을 방문할 수 있다.

## 재귀로 구현하는 DFS

DFS는 재귀로 구현하면 가장 직관적이다.

```text
function dfs(graph, current, visited):
    add current to visited
    print current

    for each next in graph[current]:
        if next is not in visited:
            dfs(graph, next, visited)
```

재귀 DFS에서는 다음 정점으로 이동할 때 함수를 다시 호출한다.

더 이상 갈 곳이 없으면 함수가 종료되고, 이전 함수 호출 위치로 돌아간다.

이 되돌아가는 과정이 DFS의 핵심이다.

```text
A -> B -> D
```

`D`에서 더 갈 곳이 없으면 `B`로 돌아간다.

그리고 `B`에 남은 다른 이웃이 있는지 확인한다.

## 스택으로 구현하는 DFS

DFS는 스택으로도 구현할 수 있다.

스택은 나중에 넣은 값이 먼저 나오는 자료 구조이다.

```text
push: 스택에 넣기
pop : 스택에서 꺼내기
```

스택을 이용한 DFS 의사코드는 다음과 같다.

```text
function dfs_stack(graph, start):
    stack = [start]
    visited = empty set

    while stack is not empty:
        current = pop from stack

        if current is in visited:
            continue

        add current to visited
        print current

        for each next in graph[current]:
            if next is not in visited:
                push next to stack
```

스택 DFS에서는 방문할 정점을 스택에 넣어 두고, 가장 나중에 넣은 정점부터 꺼내서 방문한다.

그래서 한 방향으로 깊게 들어가는 효과가 생긴다.

## 재귀 DFS와 스택 DFS의 차이

재귀 DFS와 스택 DFS는 핵심 원리는 같다.

둘 다 깊게 들어갔다가 되돌아오는 방식이다.

차이는 스택을 누가 관리하느냐이다.

```text
재귀 DFS: 프로그램의 함수 호출 스택을 사용한다.
스택 DFS: 직접 만든 스택 자료 구조를 사용한다.
```

재귀 DFS는 코드가 짧고 이해하기 쉽다.

하지만 그래프가 너무 깊으면 함수 호출이 너무 많이 쌓일 수 있다.

스택 DFS는 직접 스택을 관리해야 하므로 코드가 조금 길지만, 재귀 깊이 제한을 피하는 데 도움이 된다.

## 방문 순서

DFS의 방문 순서는 인접 정점을 어떤 순서로 확인하느냐에 따라 달라질 수 있다.

예를 들어 다음 그래프가 있다고 하자.

```text
A: B, C
B: D
C: E
D: empty
E: empty
```

`A`의 이웃을 `B`, `C` 순서로 확인하면 방문 순서는 다음과 같다.

```text
A -> B -> D -> C -> E
```

하지만 `A`의 이웃을 `C`, `B` 순서로 확인하면 방문 순서가 달라진다.

```text
A -> C -> E -> B -> D
```

둘 다 올바른 DFS이다.

문제에서 방문 순서를 요구한다면 인접 정점을 정렬하거나, 주어진 순서를 정확히 지켜야 한다.

## 트리에서의 DFS

트리는 사이클이 없는 그래프라고 생각할 수 있다.

다음과 같은 트리가 있다고 하자.

```text
      A
     / \
    B   C
   / \   \
  D   E   F
```

왼쪽 자식을 먼저 방문하는 DFS를 하면 다음과 같은 순서가 된다.

```text
A -> B -> D -> E -> C -> F
```

트리에서는 부모에서 자식 방향으로만 내려간다면 같은 노드를 다시 만날 일이 거의 없다.

하지만 그래프처럼 부모와 자식을 양방향으로 저장했다면 방문 체크가 필요하다.

## 전위 순회, 중위 순회, 후위 순회

트리에서 DFS는 순회 순서에 따라 여러 이름으로 나뉜다.

대표적으로 이진 트리에서는 다음 세 가지가 자주 나온다.

```text
전위 순회: 현재 노드 -> 왼쪽 자식 -> 오른쪽 자식
중위 순회: 왼쪽 자식 -> 현재 노드 -> 오른쪽 자식
후위 순회: 왼쪽 자식 -> 오른쪽 자식 -> 현재 노드
```

예를 들어 다음 트리가 있다고 하자.

```text
    A
   / \
  B   C
```

전위 순회는 다음과 같다.

```text
A -> B -> C
```

중위 순회는 다음과 같다.

```text
B -> A -> C
```

후위 순회는 다음과 같다.

```text
B -> C -> A
```

이 세 순회는 모두 DFS의 한 종류라고 볼 수 있다.

## 경로를 저장하는 DFS

DFS를 사용할 때 단순히 방문만 하는 것이 아니라, 현재까지 지나온 경로를 저장해야 할 때도 있다.

예를 들어 `A`에서 `E`까지 가는 경로를 찾고 싶다고 하자.

```text
A: B, C
B: D, E
C: F
D: empty
E: empty
F: empty
```

경로를 저장하는 DFS는 다음처럼 생각할 수 있다.

```text
function dfs_path(graph, current, target, visited, path):
    add current to visited
    add current to path

    if current == target:
        return path

    for each next in graph[current]:
        if next is not in visited:
            result = dfs_path(graph, next, target, visited, path)

            if result is not null:
                return result

    remove current from path
    return null
```

핵심은 더 이상 갈 수 없거나 목표를 찾지 못한 길이라면 `path`에서 현재 정점을 빼고 되돌아가는 것이다.

이런 방식은 백트래킹에서도 자주 사용된다.

## 백트래킹과 DFS

백트래킹은 가능한 선택지를 하나씩 시도하다가, 조건에 맞지 않으면 되돌아가는 방법이다.

DFS는 백트래킹의 기본 구조로 자주 사용된다.

예를 들어 미로에서 출구를 찾는 문제를 생각해 보자.

```text
1. 현재 위치를 방문한다.
2. 위, 아래, 왼쪽, 오른쪽 중 갈 수 있는 곳으로 이동한다.
3. 막히면 이전 위치로 돌아간다.
4. 다른 방향을 다시 시도한다.
5. 출구를 찾으면 종료한다.
```

이 흐름은 DFS와 매우 비슷하다.

DFS가 그래프를 깊게 탐색하는 기본 방법이라면, 백트래킹은 그 과정에서 조건에 맞지 않는 선택을 버리는 응용이라고 볼 수 있다.

## 인접 리스트와 인접 행렬

DFS에서 그래프를 저장하는 대표적인 방법은 두 가지이다.

```text
인접 리스트
인접 행렬
```

인접 리스트는 각 정점마다 연결된 정점 목록을 저장한다.

```text
A: B, C
B: A, D
C: A
D: B
```

연결된 정점만 저장하므로 보통 공간을 적게 사용한다.

인접 행렬은 정점과 정점이 연결되어 있는지를 표처럼 저장한다.

```text
    A B C D
A   0 1 1 0
B   1 0 0 1
C   1 0 0 0
D   0 1 0 0
```

두 정점이 연결되어 있는지 빠르게 확인할 수 있지만, 정점이 많으면 공간을 많이 사용할 수 있다.

DFS에서는 보통 인접 리스트를 많이 사용한다.

## 시간 복잡도

DFS의 시간 복잡도는 그래프를 어떻게 저장했는지에 따라 다르게 볼 수 있다.

인접 리스트를 사용하면 각 정점과 각 간선을 한 번씩 확인한다.

```text
O(V + E)
```

여기서 `V`는 정점의 개수이고, `E`는 간선의 개수이다.

정점마다 방문 체크를 하고, 각 정점의 인접 정점을 확인하기 때문이다.

인접 행렬을 사용하면 각 정점에서 다른 모든 정점과 연결되어 있는지 확인해야 할 수 있다.

```text
O(V^2)
```

그래서 DFS의 시간 복잡도는 보통 다음처럼 말한다.

```text
인접 리스트: O(V + E)
인접 행렬: O(V^2)
```

트리에서는 간선의 수가 보통 정점 수보다 하나 적다.

그래서 트리 DFS는 보통 다음처럼 생각할 수 있다.

```text
O(n)
```

## 공간 복잡도

DFS는 방문 기록이 필요하다.

정점이 `V`개라면 방문 기록에 최대 `V`개의 정점이 저장될 수 있다.

```text
visited: O(V)
```

재귀 DFS는 함수 호출 스택도 사용한다.

그래프나 트리가 한쪽으로 길게 이어져 있다면 최대 `V`개만큼 호출이 쌓일 수 있다.

```text
재귀 호출 스택: O(V)
```

스택 DFS를 직접 구현해도 스택에 최대 `V`개의 정점이 들어갈 수 있다.

그래서 DFS의 공간 복잡도는 보통 다음과 같다.

```text
O(V)
```

트리에서 정점 수를 `n`이라고 하면 다음처럼 말할 수 있다.

```text
O(n)
```

## 장점

```text
구현이 비교적 간단하다.
재귀로 쓰면 코드가 짧고 직관적이다.
경로 탐색 문제에 잘 맞는다.
백트래킹 문제의 기본 구조로 사용하기 좋다.
그래프와 트리 모두에 사용할 수 있다.
현재 경로를 깊게 따라가야 하는 문제에 적합하다.
```

## 단점

```text
최단 경로를 보장하지 않는다.
그래프가 너무 깊으면 재귀 깊이 제한에 걸릴 수 있다.
방문 체크를 하지 않으면 사이클에서 무한 반복될 수 있다.
방문 순서가 인접 정점의 순서에 따라 달라질 수 있다.
목표가 가까운 곳에 있어도 먼 길로 먼저 들어갈 수 있다.
```

## BFS와 비교

BFS는 Breadth First Search의 줄임말이고, 너비 우선 탐색이라고 부른다.

DFS가 한 방향으로 깊게 들어가는 방식이라면, BFS는 가까운 정점부터 차례대로 넓게 확인하는 방식이다.

```text
DFS: 깊게 들어간다.
BFS: 가까운 곳부터 넓게 본다.
```

예를 들어 다음 그래프가 있다고 하자.

```text
A: B, C
B: D, E
C: F
```

DFS의 방문 순서는 다음과 같을 수 있다.

```text
A -> B -> D -> E -> C -> F
```

BFS의 방문 순서는 다음과 같다.

```text
A -> B -> C -> D -> E -> F
```

가중치가 없는 그래프에서 최단 거리를 찾는 문제는 보통 BFS가 더 적합하다.

반면 가능한 경로를 끝까지 탐색하거나, 모든 경우를 시도하는 문제는 DFS가 더 자연스럽게 쓰이는 경우가 많다.

```text
DFS가 좋은 경우: 경로 탐색, 백트래킹, 모든 경우 탐색
BFS가 좋은 경우: 가까운 정점부터 탐색, 최단 거리 탐색
```

## 언제 사용하면 좋은가

DFS는 다음과 같은 상황에서 사용하기 좋다.

```text
그래프의 모든 정점을 방문하고 싶을 때
두 정점이 연결되어 있는지 확인하고 싶을 때
트리나 그래프에서 특정 값을 찾고 싶을 때
미로에서 출구를 찾고 싶을 때
가능한 모든 경로를 탐색하고 싶을 때
백트래킹 문제를 풀 때
연결 요소의 개수를 구할 때
사이클이 있는지 확인할 때
위상 정렬 같은 그래프 알고리즘의 기초로 사용할 때
```

## 주의할 점

DFS를 사용할 때 가장 중요한 것은 방문 체크이다.

그래프에 사이클이 있으면 같은 정점을 계속 방문할 수 있기 때문이다.

```text
if next is not in visited:
    dfs(graph, next, visited)
```

또 재귀로 DFS를 구현할 때는 깊이가 너무 깊어질 수 있다는 점을 조심해야 한다.

정점이 아주 많이 한 줄로 연결되어 있다면 함수 호출이 많이 쌓인다.

이 경우에는 직접 스택을 사용하는 방식이 더 안전할 수 있다.

방문 순서가 중요한 문제라면 인접 정점의 순서도 확인해야 한다.

```text
A: B, C
A: C, B
```

두 경우 모두 같은 그래프처럼 보일 수 있지만 DFS 방문 순서는 달라진다.

## 한 문장 요약

DFS는 그래프나 트리에서 한 방향으로 갈 수 있는 만큼 깊게 들어갔다가, 막히면 되돌아오며 다른 길을 탐색하는 깊이 우선 탐색 알고리즘이다.

## 알고리즘 흐름도

아래 흐름은 DFS를 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
그래프와 시작 정점을 받는다
  |
  v
시작 정점을 방문했다고 표시한다
  |
  v
현재 정점을 처리한다
  |
  v
현재 정점과 연결된 다음 정점이 있는가?
  |
  +-- 아니오 --> 이전 정점으로 돌아간다
  |
  +-- 예
        |
        v
      다음 정점을 이미 방문했는가?
        |
        +-- 예 --> 다른 다음 정점을 확인한다
        |
        +-- 아니오
              |
              v
            다음 정점으로 이동한다
              |
              v
            그 정점에서 다시 DFS를 실행한다
```

## 더 짧게 외우는 흐름

```text
1. 시작 정점을 정한다.
2. 현재 정점을 방문한다.
3. 갈 수 있는 정점 중 아직 방문하지 않은 정점을 찾는다.
4. 있으면 그 정점으로 이동해서 다시 DFS를 한다.
5. 없으면 이전 정점으로 돌아간다.
6. 모든 길을 확인할 때까지 반복한다.
```

## 머릿속 결정 순서

DFS를 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
어디에서 시작할 것인가?
  -> start 정하기

그래프는 어떻게 저장되어 있는가?
  -> 인접 리스트 또는 인접 행렬 확인

현재 정점을 방문했는가?
  -> visited에 기록하기

현재 정점에서 어디로 갈 수 있는가?
  -> 인접 정점 확인하기

다음 정점을 이미 방문했는가?
  -> 방문했다면 건너뛰기

방문하지 않았다면 무엇을 할 것인가?
  -> 그 정점으로 깊게 들어가기

더 이상 갈 곳이 없으면 어떻게 할 것인가?
  -> 이전 정점으로 되돌아가기

목표를 찾으면 어떻게 할 것인가?
  -> 바로 반환하고 종료하기

끝까지 없으면 어떻게 할 것인가?
  -> null 같은 실패 값을 반환
```
