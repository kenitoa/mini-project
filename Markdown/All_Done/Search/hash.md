# 해시 탐색

해시 탐색은 데이터를 찾을 때 값을 처음부터 하나씩 비교하지 않고, 해시 함수를 이용해 찾을 위치를 바로 계산하는 탐색 방법이다.

영어로는 Hash Search 또는 Hash Table Search라고 부른다.

해시 탐색은 보통 해시 테이블이라는 자료 구조를 사용한다.

해시 테이블은 데이터를 `key`와 `value` 형태로 저장한다.

```text
key   -> 데이터를 찾기 위한 이름표
value -> key에 연결된 실제 데이터
```

예를 들어 학생 번호로 학생 이름을 찾는다면 학생 번호가 `key`, 학생 이름이 `value`가 된다.

```text
101 -> "Kim"
102 -> "Lee"
103 -> "Park"
```

해시 탐색의 핵심은 `key`를 해시 함수에 넣으면 데이터가 저장될 위치가 계산된다는 점이다.

## 핵심 생각

예를 들어 이런 데이터가 있다고 하자.

```text
101 -> "Kim"
102 -> "Lee"
103 -> "Park"
```

여기서 `102`번 학생을 찾고 싶다면 선형 탐색처럼 처음부터 하나씩 비교하지 않는다.

```text
101인가? 아니오
102인가? 예
```

이런 방식이 아니라 `102`라는 key를 해시 함수에 넣는다.

```text
hash(102) = 2
```

그러면 해시 테이블의 `2`번 위치를 바로 확인한다.

```text
table[2] = "Lee"
```

즉, 해시 탐색은 key를 이용해 위치를 계산하고, 계산된 위치에서 값을 찾는 방식이다.

## 해시 함수

해시 함수는 key를 받아서 해시 테이블의 위치를 계산하는 함수이다.

```text
hash(key) -> index
```

예를 들어 해시 테이블의 크기가 `10`이라면 다음처럼 만들 수 있다.

```text
hash(key) = key % 10
```

`%`는 나머지를 구하는 연산이다.

```text
hash(101) = 101 % 10 = 1
hash(102) = 102 % 10 = 2
hash(103) = 103 % 10 = 3
```

따라서 각각의 데이터는 다음 위치에 저장될 수 있다.

```text
table[1] = "Kim"
table[2] = "Lee"
table[3] = "Park"
```

실제 프로그래밍 언어에서는 문자열, 숫자, 객체 등 다양한 key를 해시 함수로 처리할 수 있다.

## 해시 탐색 의사코드

```text
hash_search(해시 테이블, 찾는 key):
    key를 해시 함수에 넣어 index를 구한다
    해시 테이블의 index 위치를 확인한다

    그 위치에 찾는 key가 있다면:
        그 key에 연결된 value를 반환한다

    찾는 key가 없다면:
        찾을 수 없다는 값을 반환한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function hash_search(table, key):
    index = hash(key)

    if table[index] has key:
        return table[index][key]

    return null
```

여기서 `null`은 원하는 key를 찾지 못했다는 뜻으로 자주 사용된다.

실제 코드에서는 상황에 따라 `-1`, `false`, `None`, `"not found"` 같은 값을 반환하기도 한다.

## key와 value

해시 탐색에서는 보통 데이터를 단순한 값 하나로만 저장하지 않는다.

대부분 다음처럼 key와 value를 함께 저장한다.

```text
key   : 데이터를 찾을 때 사용하는 값
value : key를 통해 얻고 싶은 실제 값
```

예를 들어 전화번호부를 생각해 보자.

```text
"민수" -> "010-1111-2222"
"지영" -> "010-3333-4444"
"현우" -> "010-5555-6666"
```

여기서 `"지영"`의 전화번호를 찾고 싶다면 `"지영"`이 key가 된다.

```text
hash("지영") = 4
```

그러면 해시 테이블의 `4`번 위치를 확인하고, 그곳에서 전화번호를 찾는다.

```text
return "010-3333-4444"
```

## 빠르게 떠올려보기

해시 탐색은 이렇게 생각하면 쉽다.

```text
1. 찾고 싶은 key를 정한다.
2. key를 해시 함수에 넣는다.
3. 해시 함수가 저장 위치를 계산한다.
4. 그 위치로 바로 이동한다.
5. key가 맞는지 확인한다.
6. 맞으면 value를 반환한다.
7. 없으면 찾지 못했다는 값을 반환한다.
```

즉, 해시 탐색은 key를 주소처럼 사용해서 값을 빠르게 찾는 방식이다.

## 예시

다음과 같은 해시 테이블이 있다고 하자.

```text
table[0] = empty
table[1] = 101 -> "Kim"
table[2] = 102 -> "Lee"
table[3] = 103 -> "Park"
table[4] = empty
```

여기서 key `102`를 찾는다고 하자.

```text
hash(102) = 102 % 5 = 2
```

계산된 위치는 `2`이다.

```text
table[2] = 102 -> "Lee"
```

저장된 key가 찾는 key인 `102`와 같으므로 연결된 값을 반환한다.

```text
return "Lee"
```

## 찾는 key가 없는 예시

```text
table[0] = empty
table[1] = 101 -> "Kim"
table[2] = 102 -> "Lee"
table[3] = 103 -> "Park"
table[4] = empty
```

여기서 key `104`를 찾는다고 하자.

```text
hash(104) = 104 % 5 = 4
```

계산된 위치는 `4`이다.

하지만 `table[4]`는 비어 있다.

```text
table[4] = empty
```

따라서 key `104`는 해시 테이블에 없다고 판단한다.

```text
return null
```

## 종료 조건

해시 탐색이 끝나는 경우는 크게 두 가지이다.

첫 번째는 원하는 key를 찾은 경우이다.

```text
if table[index] has key:
    return value
```

key를 찾았다면 더 이상 탐색할 필요가 없으므로 바로 종료한다.

두 번째는 계산된 위치에 key가 없거나, 충돌 처리를 확인했는데도 key를 찾지 못한 경우이다.

```text
return null
```

이 경우에는 찾는 데이터가 해시 테이블에 없다고 판단한다.

## 빈 해시 테이블인 경우

해시 테이블에 데이터가 하나도 없다면 어떤 key를 찾더라도 찾을 수 없다.

```text
hash_search({}, "apple")
```

테이블이 비어 있으므로 확인할 데이터가 없다.

따라서 바로 찾지 못했다는 값을 반환한다.

```text
return null
```

## 충돌

해시 탐색에서 중요한 문제가 하나 있다.

서로 다른 key가 같은 위치로 계산될 수 있다는 점이다.

이를 충돌이라고 한다.

예를 들어 해시 함수가 다음과 같다고 하자.

```text
hash(key) = key % 5
```

그렇다면 `101`과 `106`은 같은 위치가 된다.

```text
hash(101) = 101 % 5 = 1
hash(106) = 106 % 5 = 1
```

둘 다 `table[1]`에 저장되어야 하므로 충돌이 발생한다.

```text
101 -> "Kim"
106 -> "Choi"
```

해시 탐색을 제대로 사용하려면 이런 충돌을 처리해야 한다.

## 충돌 처리 방법 1: 체이닝

체이닝은 같은 위치에 여러 데이터를 리스트처럼 연결해서 저장하는 방법이다.

예를 들어 `101`과 `106`이 모두 `1`번 위치에 들어간다면 다음처럼 저장한다.

```text
table[1] = [(101, "Kim"), (106, "Choi")]
```

key `106`을 찾는다면 먼저 해시 함수를 이용해 위치를 계산한다.

```text
hash(106) = 1
```

그리고 `table[1]`에 연결된 데이터들 중에서 key가 `106`인 데이터를 찾는다.

```text
(101, "Kim")  -> key가 다름
(106, "Choi") -> key가 같음
```

따라서 `"Choi"`를 반환한다.

```text
return "Choi"
```

체이닝은 충돌이 일어난 위치 안에서만 다시 짧게 탐색한다고 생각하면 된다.

## 체이닝 의사코드

```text
function hash_search_chaining(table, key):
    index = hash(key)
    bucket = table[index]

    for each item in bucket:
        if item.key == key:
            return item.value

    return null
```

여기서 `bucket`은 같은 해시 위치에 모여 있는 데이터 묶음이다.

충돌이 적다면 bucket의 길이가 짧기 때문에 탐색이 매우 빠르다.

## 충돌 처리 방법 2: 개방 주소법

개방 주소법은 충돌이 발생했을 때 다른 빈 위치를 찾아 저장하는 방법이다.

예를 들어 `101`과 `106`이 모두 `1`번 위치로 계산된다고 하자.

```text
hash(101) = 1
hash(106) = 1
```

먼저 `101`은 `table[1]`에 저장된다.

```text
table[1] = 101 -> "Kim"
```

그 다음 `106`을 저장하려고 보니 `table[1]`이 이미 사용 중이다.

그러면 다음 위치를 확인한다.

```text
table[2]가 비어 있으면 table[2]에 저장
```

이처럼 빈 칸을 찾아 이동하면서 저장하는 방식이다.

탐색할 때도 같은 순서로 확인해야 한다.

## 개방 주소법 의사코드

```text
function hash_search_open_addressing(table, key):
    index = hash(key)

    while table[index] is not empty:
        if table[index].key == key:
            return table[index].value

        index = next_index(index)

    return null
```

여기서 `next_index`는 다음으로 확인할 위치를 정하는 함수이다.

가장 단순한 방법은 바로 다음 칸으로 이동하는 것이다.

```text
next_index(index) = index + 1
```

테이블의 끝까지 갔다면 다시 처음으로 돌아가도록 만들기도 한다.

## 같은 key를 다시 넣는 경우

해시 테이블에서는 보통 같은 key를 다시 넣으면 기존 value를 새 value로 바꾼다.

예를 들어 다음 데이터가 있다고 하자.

```text
"apple" -> 1000
```

여기에 같은 key로 다시 값을 넣는다.

```text
"apple" -> 1200
```

그러면 `"apple"`이라는 key는 하나만 유지되고 value만 바뀐다.

```text
"apple" -> 1200
```

따라서 해시 탐색에서 같은 key는 보통 하나의 value에만 연결된다.

만약 같은 key에 여러 값을 저장하고 싶다면 value를 리스트로 만들 수 있다.

```text
"apple" -> [1000, 1200, 900]
```

## 정렬 여부와 관계

해시 탐색은 데이터가 정렬되어 있을 필요가 없다.

선형 탐색처럼 처음부터 끝까지 확인하지도 않고, 이진 탐색처럼 가운데 값을 기준으로 범위를 줄이지도 않는다.

해시 탐색은 key를 해시 함수에 넣어 위치를 계산한다.

```text
hash(key) -> index
```

따라서 데이터가 오름차순인지 내림차순인지는 중요하지 않다.

하지만 해시 탐색은 순서가 중요한 작업에는 적합하지 않을 수 있다.

예를 들어 가장 작은 값, 가장 큰 값, 정렬된 순서대로 출력하기 같은 작업은 해시 테이블보다 정렬된 배열이나 트리가 더 알맞을 수 있다.

## 시간 복잡도

해시 탐색의 평균 시간 복잡도는 매우 빠르다.

충돌이 적고 해시 함수가 데이터를 고르게 분산한다면 key를 이용해 거의 바로 값을 찾을 수 있다.

```text
평균적인 경우: O(1)
```

`O(1)`은 데이터가 많아져도 탐색 시간이 거의 일정하다는 뜻이다.

하지만 충돌이 너무 많이 발생하면 한 위치에 데이터가 몰릴 수 있다.

예를 들어 체이닝에서 모든 데이터가 같은 bucket에 들어간다면 그 bucket 안에서 선형 탐색을 해야 한다.

```text
최악의 경우: O(n)
```

그래서 해시 탐색의 시간 복잡도는 보통 다음처럼 말한다.

```text
최선의 경우: O(1)
평균적인 경우: O(1)
최악의 경우: O(n)
```

## 공간 복잡도

해시 테이블은 데이터를 저장하기 위한 배열이나 bucket이 필요하다.

또 충돌을 줄이기 위해 실제 데이터 개수보다 더 큰 공간을 준비하는 경우도 많다.

따라서 공간 복잡도는 보통 다음과 같다.

```text
O(n)
```

여기서 `n`은 저장된 데이터의 개수이다.

체이닝을 사용하면 각 bucket 안에 리스트가 추가로 필요할 수 있다.

개방 주소법을 사용하면 빈 칸을 어느 정도 남겨 두어야 성능이 좋아진다.

## 장점

```text
탐색 속도가 매우 빠르다.
평균적으로 O(1)에 값을 찾을 수 있다.
데이터가 정렬되어 있지 않아도 사용할 수 있다.
key를 이용해 value를 찾는 문제에 매우 적합하다.
삽입, 삭제, 탐색을 모두 빠르게 처리할 수 있다.
```

## 단점

```text
충돌이 발생할 수 있다.
충돌 처리를 제대로 하지 않으면 성능이 나빠진다.
해시 함수가 좋지 않으면 데이터가 한쪽으로 몰릴 수 있다.
정렬된 순서로 데이터를 탐색하는 데는 적합하지 않다.
배열보다 추가 공간을 더 사용할 수 있다.
```

## 선형 탐색과 비교

선형 탐색은 데이터를 앞에서부터 하나씩 확인한다.

```text
[4, 9, 2, 7, 5]
```

여기서 `7`을 찾으려면 다음처럼 비교한다.

```text
4 == 7 ? 아니오
9 == 7 ? 아니오
2 == 7 ? 아니오
7 == 7 ? 예
```

반면 해시 탐색은 key를 이용해 위치를 계산한다.

```text
hash(7) = index
```

그리고 계산된 위치를 바로 확인한다.

```text
table[index]
```

그래서 많은 데이터를 key로 빠르게 찾아야 한다면 해시 탐색이 선형 탐색보다 훨씬 효율적이다.

```text
선형 탐색: O(n)
해시 탐색: 평균 O(1)
```

## 이진 탐색과 비교

이진 탐색은 정렬된 데이터에서만 사용할 수 있다.

```text
[1, 3, 5, 7, 9]
```

가운데 값을 기준으로 탐색 범위를 절반씩 줄인다.

반면 해시 탐색은 정렬이 필요 없다.

```text
정렬 여부와 관계없이 key로 위치를 계산
```

하지만 이진 탐색은 정렬된 순서를 유지하면서 범위 탐색을 할 때 유리하다.

예를 들어 `5 이상 10 이하의 값`을 찾는 문제는 해시 탐색보다 이진 탐색이 더 적합할 수 있다.

```text
이진 탐색: 정렬된 데이터에서 범위나 위치를 찾을 때 좋음
해시 탐색: 정확한 key 하나로 value를 찾을 때 좋음
```

## 언제 사용하면 좋은가

해시 탐색은 다음과 같은 상황에서 사용하기 좋다.

```text
key를 이용해 value를 빠르게 찾고 싶을 때
중복을 빠르게 확인하고 싶을 때
데이터 존재 여부를 빠르게 검사하고 싶을 때
전화번호부처럼 이름으로 정보를 찾고 싶을 때
학생 번호로 학생 정보를 찾고 싶을 때
단어가 몇 번 나왔는지 세고 싶을 때
```

예를 들어 어떤 단어가 이미 등장했는지 확인할 때 해시 탐색을 사용할 수 있다.

```text
seen["apple"] = true
```

다음에 `"apple"`이 다시 나오면 바로 확인할 수 있다.

```text
if seen["apple"] == true:
    이미 등장한 단어
```

## 주의할 점

해시 탐색은 빠르지만 항상 `O(1)`이라고만 생각하면 안 된다.

충돌이 많아지면 탐색 시간이 길어질 수 있다.

그래서 좋은 해시 함수와 적절한 테이블 크기가 중요하다.

또 해시 테이블이 너무 꽉 차면 충돌이 늘어나기 쉽다.

이때는 보통 테이블 크기를 늘리고 데이터를 다시 배치한다.

이 작업을 재해싱이라고 한다.

```text
기존 테이블보다 큰 테이블을 만든다.
기존 데이터를 새 테이블에 다시 넣는다.
충돌을 줄여 성능을 회복한다.
```

## 한 문장 요약

해시 탐색은 key를 해시 함수에 넣어 저장 위치를 계산하고, 그 위치에서 value를 빠르게 찾는 평균 O(1)의 탐색 알고리즘이다.

## 알고리즘 흐름도

아래 흐름은 해시 탐색을 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
해시 테이블과 찾는 key를 받는다
  |
  v
key를 해시 함수에 넣어 index를 구한다
  |
  v
table[index]를 확인한다
  |
  v
해당 위치가 비어 있는가?
  |
  +-- 예 --> 찾지 못했다는 값을 반환한다
  |
  +-- 아니오
        |
        v
      저장된 key가 찾는 key와 같은가?
        |
        +-- 예 --> 연결된 value를 반환한다
        |
        +-- 아니오
              |
              v
            충돌 처리 방식에 따라 다음 데이터를 확인한다
              |
              v
            key를 찾았는가?
              |
              +-- 예 --> value를 반환한다
              |
              +-- 아니오 --> 찾지 못했다는 값을 반환한다
```

## 머릿속 결정 순서

해시 탐색을 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
무엇으로 찾을 것인가?
  -> key 정하기

key로 어떤 위치를 볼 것인가?
  -> hash(key)로 index 구하기

그 위치에 데이터가 있는가?
  -> 비어 있으면 찾기 실패

그 위치의 key가 찾는 key와 같은가?
  -> 같으면 value 반환

다른 key가 있다면 어떻게 할 것인가?
  -> 충돌 처리 방식에 따라 확인

끝까지 찾지 못하면 어떻게 할 것인가?
  -> null 같은 실패 값을 반환
```
