# 레벤슈타인 거리

레벤슈타인 거리는 두 문자열이 얼마나 다른지 숫자로 나타내는 방법이다.

한 문자열을 다른 문자열로 바꾸기 위해 필요한 최소 편집 횟수를 의미한다.

여기서 편집은 보통 다음 세 가지를 말한다.

```text
삽입
삭제
교체
```

예를 들어 `cat`을 `cut`으로 바꾸려면 `a`를 `u`로 교체하면 된다.

```text
cat -> cut
편집 횟수 = 1
```

그래서 두 문자열의 레벤슈타인 거리는 `1`이다.

## 핵심 생각

예를 들어 `kitten`을 `sitting`으로 바꾼다고 하자.

가능한 편집 과정은 다음과 같다.

```text
kitten
sitten   k를 s로 교체
sittin   e를 i로 교체
sitting  g를 삽입
```

총 편집 횟수는 `3`이다.

```text
kitten -> sitting
거리 = 3
```

레벤슈타인 거리는 이런 최소 편집 횟수를 동적 계획법으로 구한다.

문자열의 앞부분끼리 비교한 결과를 표에 저장해 두고, 더 긴 부분 문자열의 답을 만든다.

## 문자열에서 사용하는 말

레벤슈타인 거리를 이해하려면 다음 말을 알아두면 좋다.

```text
삽입: 문자를 하나 추가하는 것
삭제: 문자를 하나 제거하는 것
교체: 문자를 하나 다른 문자로 바꾸는 것
부분 문자열: 문자열의 일부
동적 계획법: 작은 문제의 답을 저장해 큰 문제의 답을 구하는 방법
DP 테이블: 계산 결과를 저장하는 표
```

두 문자열을 다음처럼 두자.

```text
word1 = abc
word2 = adc
```

`abc`를 `adc`로 바꾸려면 `b`를 `d`로 교체하면 된다.

```text
거리 = 1
```

## DP 테이블의 의미

레벤슈타인 거리에서는 보통 2차원 DP 테이블을 사용한다.

```text
dp[i][j] = word1의 앞 i글자를 word2의 앞 j글자로 바꾸는 최소 편집 횟수
```

예를 들어:

```text
word1 = cat
word2 = cut
```

```text
dp[2][2] = "ca"를 "cu"로 바꾸는 최소 편집 횟수
```

`ca`를 `cu`로 바꾸려면 `a`를 `u`로 교체하면 된다.

```text
dp[2][2] = 1
```

## 기본값

빈 문자열을 어떤 문자열로 바꾸는 경우를 먼저 생각한다.

```text
"" -> "abc"
```

문자를 세 개 삽입해야 한다.

```text
거리 = 3
```

반대로 어떤 문자열을 빈 문자열로 바꾸려면 문자를 모두 삭제해야 한다.

```text
"abc" -> ""
거리 = 3
```

그래서 DP 테이블의 첫 줄과 첫 열은 다음처럼 채운다.

```text
dp[0][j] = j
dp[i][0] = i
```

## 점화식

`word1[i - 1]`와 `word2[j - 1]`를 비교한다.

두 문자가 같다면 새 편집이 필요 없다.

```text
if word1[i - 1] == word2[j - 1]:
    dp[i][j] = dp[i - 1][j - 1]
```

두 문자가 다르다면 세 가지 편집 중 하나를 선택한다.

```text
삭제: dp[i - 1][j] + 1
삽입: dp[i][j - 1] + 1
교체: dp[i - 1][j - 1] + 1
```

가장 작은 값을 고른다.

```text
dp[i][j] = min(
    dp[i - 1][j] + 1,
    dp[i][j - 1] + 1,
    dp[i - 1][j - 1] + 1
)
```

## 레벤슈타인 거리 의사코드

```text
levenshtein(word1, word2):
    word1 길이를 n이라고 한다
    word2 길이를 m이라고 한다
    크기가 (n + 1) x (m + 1)인 DP 테이블을 만든다

    dp[i][0]을 i로 채운다
    dp[0][j]를 j로 채운다

    i를 1부터 n까지 반복한다:
        j를 1부터 m까지 반복한다:
            두 문자가 같다면:
                dp[i][j] = dp[i - 1][j - 1]
            두 문자가 다르다면:
                삭제, 삽입, 교체 중 최소값 + 1을 넣는다

    dp[n][m]을 반환한다
```

마지막 칸 `dp[n][m]`이 두 전체 문자열 사이의 레벤슈타인 거리이다.

## 조금 더 코드처럼 쓴 의사코드

```text
function levenshtein(a, b):
    n = length(a)
    m = length(b)
    dp = 2D array of size (n + 1) x (m + 1)

    for i from 0 to n:
        dp[i][0] = i

    for j from 0 to m:
        dp[0][j] = j

    for i from 1 to n:
        for j from 1 to m:
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                delete_cost = dp[i - 1][j] + 1
                insert_cost = dp[i][j - 1] + 1
                replace_cost = dp[i - 1][j - 1] + 1
                dp[i][j] = min(delete_cost, insert_cost, replace_cost)

    return dp[n][m]
```

인덱스에서 `i - 1`, `j - 1`을 쓰는 이유는 DP 테이블에 빈 문자열을 위한 0번째 줄과 열이 있기 때문이다.

## 흐름을 쉽게 풀어보기

레벤슈타인 거리는 이렇게 생각하면 쉽다.

```text
1. 빈 문자열끼리의 거리는 0이다.
2. 빈 문자열에서 길이 j 문자열로 가려면 j번 삽입한다.
3. 길이 i 문자열에서 빈 문자열로 가려면 i번 삭제한다.
4. 마지막 문자가 같으면 이전 문자열의 답을 그대로 쓴다.
5. 마지막 문자가 다르면 삽입, 삭제, 교체 중 가장 싼 것을 고른다.
```

즉, 두 문자열 전체를 바로 비교하지 않고 앞부분부터 차근차근 답을 만든다.

## 예시

다음 두 문자열을 비교하자.

```text
word1 = cat
word2 = cut
```

`c`와 `c`는 같다.

```text
c -> c
편집 필요 없음
```

`a`와 `u`는 다르다.

```text
a -> u
교체 1번
```

`t`와 `t`는 같다.

```text
t -> t
편집 필요 없음
```

결과적으로 `cat`을 `cut`으로 바꾸는 최소 편집 횟수는 `1`이다.

```text
거리 = 1
```

## DP 테이블 예시

`cat`과 `cut`의 DP 테이블은 다음처럼 생각할 수 있다.

```text
      ""  c  u  t
""     0  1  2  3
c      1  0  1  2
a      2  1  1  2
t      3  2  2  1
```

마지막 칸이 답이다.

```text
dp[3][3] = 1
```

## 삽입, 삭제, 교체의 의미

두 문자열의 마지막 문자가 다를 때 선택지는 세 가지이다.

```text
삭제:
    word1의 마지막 문자를 지운다.

삽입:
    word1에 word2의 마지막 문자를 추가한다고 생각한다.

교체:
    word1의 마지막 문자를 word2의 마지막 문자로 바꾼다.
```

DP에서는 이 세 경우 중 가장 작은 비용을 고른다.

```text
min(삭제, 삽입, 교체)
```

## 실제 편집 과정 복원하기

거리뿐 아니라 어떤 편집을 했는지 알고 싶다면 DP 테이블을 거꾸로 따라가면 된다.

```text
현재 위치 = dp[n][m]
```

문자가 같으면 대각선으로 이동한다.

```text
i = i - 1
j = j - 1
```

문자가 다르면 현재 값이 어디서 왔는지 확인한다.

```text
dp[i - 1][j] + 1 이면 삭제
dp[i][j - 1] + 1 이면 삽입
dp[i - 1][j - 1] + 1 이면 교체
```

이 과정을 거꾸로 모으면 실제 편집 순서를 알 수 있다.

## 빈 문자열인 경우

한쪽 문자열이 비어 있으면 답은 다른 문자열의 길이이다.

```text
levenshtein("", "abc") = 3
levenshtein("abc", "") = 3
```

둘 다 비어 있으면 거리는 `0`이다.

```text
levenshtein("", "") = 0
```

## 같은 문자열인 경우

두 문자열이 완전히 같으면 편집이 필요 없다.

```text
levenshtein("hello", "hello") = 0
```

DP 테이블을 채워도 마지막 값은 `0`이 된다.

## 대소문자와 공백

레벤슈타인 거리는 문자를 그대로 비교한다.

그래서 대소문자나 공백도 다르면 편집이 필요하다고 본다.

```text
"Cat"과 "cat"은 C와 c가 다르다.
"hello"와 "hello "는 공백 하나가 다르다.
```

필요하다면 계산 전에 문자열을 같은 형식으로 바꿀 수 있다.

```text
소문자로 변환
앞뒤 공백 제거
특수문자 제거
```

## 공간 최적화

전체 DP 테이블을 저장하지 않아도 거리만 구할 수 있다.

현재 줄을 계산할 때 필요한 것은 이전 줄과 현재 줄뿐이다.

```text
previous row
current row
```

그래서 공간을 줄이면 다음처럼 만들 수 있다.

```text
공간 복잡도: O(min(n, m))
```

단, 실제 편집 과정을 복원하려면 전체 테이블이 필요할 수 있다.

## 해밍 거리와 비교

해밍 거리는 같은 길이의 문자열에서 서로 다른 위치의 개수를 센다.

```text
cat
cut
다른 위치: a와 u
해밍 거리 = 1
```

하지만 길이가 다르면 해밍 거리는 보통 정의하기 어렵다.

레벤슈타인 거리는 삽입과 삭제도 허용하므로 길이가 달라도 계산할 수 있다.

```text
cat
cats
레벤슈타인 거리 = 1
```

## LCS와 비교

LCS는 가장 긴 공통 부분 수열을 찾는 문제이다.

레벤슈타인 거리는 편집 횟수를 구하는 문제이다.

```text
LCS:
    얼마나 많이 공통으로 남길 수 있는가?

레벤슈타인 거리:
    몇 번 편집해야 바뀌는가?
```

둘 다 문자열 DP로 자주 등장한다.

## 시간 복잡도

두 문자열의 길이를 각각 `N`, `M`이라고 하자.

DP 테이블의 크기는 `(N + 1) x (M + 1)`이다.

각 칸을 한 번씩 계산한다.

```text
시간 복잡도: O(NM)
```

## 공간 복잡도

전체 DP 테이블을 저장하면 다음과 같다.

```text
공간 복잡도: O(NM)
```

거리 값만 필요하면 두 줄만 사용해 줄일 수 있다.

```text
최적화 공간 복잡도: O(min(N, M))
```

## 장점

```text
두 문자열의 유사도를 숫자로 표현할 수 있다.
삽입, 삭제, 교체를 모두 고려한다.
동적 계획법 구조가 명확하다.
실제 편집 과정도 복원할 수 있다.
```

## 단점

```text
문자열이 길면 O(NM)이 부담될 수 있다.
문자의 의미나 발음까지 이해하는 것은 아니다.
공백, 대소문자, 특수문자 처리에 따라 결과가 달라질 수 있다.
```

## 언제 사용하면 좋은가

레벤슈타인 거리는 다음 상황에서 좋다.

```text
오타 교정
검색어 추천
문자열 유사도 비교
두 단어가 얼마나 비슷한지 판단
자동 완성 후보 정렬
```

## 주의할 점

레벤슈타인 거리는 "비슷해 보이는 정도"를 완벽히 표현하지는 않는다.

예를 들어 의미가 비슷한 단어라도 글자가 많이 다르면 거리가 크게 나올 수 있다.

반대로 의미가 전혀 달라도 글자가 비슷하면 거리가 작게 나올 수 있다.

또한 한글처럼 글자 조합이 있는 문자에서는 자모 단위로 볼지, 완성형 글자 단위로 볼지 결정해야 한다.

## 한 문장 요약

레벤슈타인 거리는 삽입, 삭제, 교체를 이용해 한 문자열을 다른 문자열로 바꾸는 최소 편집 횟수이다.

## 알고리즘 흐름도

```text
시작
  |
  v
DP 테이블 만들기
  |
  v
첫 줄과 첫 열 초기화
  |
  v
i, j 위치의 문자 비교
  |
  v
문자가 같은가?
  | yes
  v
대각선 값 그대로 사용
  |
  no
  v
삽입, 삭제, 교체 중 최소값 + 1
  |
  v
모든 칸을 채웠는가?
  | no
  v
다음 칸 계산
  |
  yes
  v
마지막 칸 반환
```

## 더 짧게 외우는 흐름

```text
DP 표 만들기
빈 문자열 기준 초기화
문자가 같으면 대각선
다르면 삽입, 삭제, 교체 중 최소
마지막 칸이 답
```

## 머릿속 결정 순서

```text
1. 두 문자열 길이를 확인했는가?
2. DP 테이블을 n + 1, m + 1 크기로 만들었는가?
3. 첫 줄과 첫 열을 초기화했는가?
4. 문자가 같을 때 대각선 값을 가져오는가?
5. 문자가 다를 때 세 편집 비용 중 최소를 고르는가?
6. 마지막 칸을 답으로 반환하는가?
```
