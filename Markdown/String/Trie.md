# 트라이

트라이는 문자열을 저장하고 빠르게 찾기 위한 트리 형태의 자료 구조이다.

영어로는 Trie라고 쓰고, 접두사 트리라고도 부른다.

트라이는 문자열을 문자 단위로 나누어 저장한다.

같은 접두사를 가진 문자열들은 앞부분 노드를 공유한다.

```text
접두사: 문자열의 앞부분
```

예를 들어 `cat`, `car`, `care`는 모두 `ca`로 시작한다.

트라이에서는 이 `c`, `a` 부분을 함께 사용한다.

## 핵심 생각

다음 단어들을 저장한다고 하자.

```text
cat
car
care
dog
```

트라이로 나타내면 다음처럼 생각할 수 있다.

```text
root
├─ c
│  └─ a
│     ├─ t
│     └─ r
│        └─ e
└─ d
   └─ o
      └─ g
```

`cat`과 `car`는 `c -> a`까지 같은 경로를 사용한다.

그 뒤 `t`와 `r`로 갈라진다.

```text
cat: c -> a -> t
car: c -> a -> r
```

이렇게 공통 접두사를 공유하기 때문에 접두사 검색에 강하다.

## 문자열에서 사용하는 말

트라이를 이해하려면 다음 말을 알아두면 좋다.

```text
노드: 트라이의 한 지점
루트: 시작 노드
간선: 문자 하나를 따라 이동하는 연결
접두사: 문자열의 앞부분
단어 끝 표시: 어떤 노드가 단어의 끝인지 표시하는 값
자식 노드: 현재 노드에서 다음 문자로 갈 수 있는 노드
```

트라이에서 중요한 점은 단어의 끝을 표시해야 한다는 것이다.

예를 들어 `car`와 `care`가 모두 저장되어 있다면:

```text
c -> a -> r -> e
```

`r`에서 `car`가 끝난다는 표시가 있어야 하고, `e`에서 `care`가 끝난다는 표시도 있어야 한다.

## 노드 구조

트라이의 각 노드는 보통 다음 정보를 가진다.

```text
children: 다음 문자로 갈 수 있는 자식 노드들
is_end: 이 노드가 단어의 끝인지 나타내는 값
```

예를 들어:

```text
node.children["a"] = 다음 노드
node.is_end = true 또는 false
```

자식 노드는 배열로 만들 수도 있고, 해시맵으로 만들 수도 있다.

알파벳 소문자만 다룬다면 크기 26짜리 배열을 사용할 수 있다.

문자의 종류가 많다면 해시맵이 더 편하다.

## 삽입 의사코드

단어를 트라이에 넣을 때는 루트에서 시작해 문자 하나씩 따라간다.

```text
insert(단어):
    현재 노드를 루트로 둔다

    단어의 문자를 앞에서부터 하나씩 확인한다:
        현재 노드에 해당 문자 자식이 없다면:
            새 노드를 만든다

        현재 노드를 해당 문자 자식으로 옮긴다

    마지막 노드에 단어 끝 표시를 한다
```

단어 끝 표시를 해야 검색할 때 정확한 단어인지 알 수 있다.

## 검색 의사코드

단어가 트라이에 있는지 확인할 때도 문자 하나씩 따라간다.

```text
search(단어):
    현재 노드를 루트로 둔다

    단어의 문자를 앞에서부터 하나씩 확인한다:
        현재 노드에 해당 문자 자식이 없다면:
            false를 반환한다

        현재 노드를 해당 문자 자식으로 옮긴다

    현재 노드가 단어 끝 표시를 가지고 있다면:
        true를 반환한다
    아니면:
        false를 반환한다
```

문자 경로가 존재하더라도 단어 끝 표시가 없으면 저장된 단어가 아니다.

## 접두사 검색 의사코드

어떤 접두사로 시작하는 단어가 있는지 확인할 때는 끝 표시를 확인하지 않아도 된다.

```text
starts_with(접두사):
    현재 노드를 루트로 둔다

    접두사의 문자를 앞에서부터 하나씩 확인한다:
        현재 노드에 해당 문자 자식이 없다면:
            false를 반환한다

        현재 노드를 해당 문자 자식으로 옮긴다

    true를 반환한다
```

접두사 경로만 존재하면 그 접두사로 시작하는 단어가 있다는 뜻이다.

## 조금 더 코드처럼 쓴 의사코드

```text
class TrieNode:
    children = map
    is_end = false

class Trie:
    root = new TrieNode()

    function insert(word):
        current = root

        for each char in word:
            if char is not in current.children:
                current.children[char] = new TrieNode()

            current = current.children[char]

        current.is_end = true

    function search(word):
        current = root

        for each char in word:
            if char is not in current.children:
                return false

            current = current.children[char]

        return current.is_end

    function starts_with(prefix):
        current = root

        for each char in prefix:
            if char is not in current.children:
                return false

            current = current.children[char]

        return true
```

트라이의 기본 기능은 삽입, 정확한 검색, 접두사 검색이다.

## 흐름을 쉽게 풀어보기

트라이는 이렇게 생각하면 쉽다.

```text
1. 루트에서 시작한다.
2. 문자열의 문자를 하나씩 본다.
3. 해당 문자의 길이 없으면 새로 만든다.
4. 있으면 그 길을 따라간다.
5. 단어가 끝나는 지점에 끝 표시를 한다.
```

즉, 단어 전체를 한 번에 저장하는 것이 아니라 문자 경로로 저장한다.

## 예시

다음 단어들을 삽입한다고 하자.

```text
cat
car
dog
```

먼저 `cat`을 넣는다.

```text
root -> c -> a -> t
```

`t` 노드에 단어 끝 표시를 한다.

다음으로 `car`를 넣는다.

`c -> a`는 이미 있으므로 그대로 따라간다.

그 다음 `r`이 없으므로 새로 만든다.

```text
root -> c -> a -> r
```

`r` 노드에 단어 끝 표시를 한다.

마지막으로 `dog`를 넣는다.

```text
root -> d -> o -> g
```

결과 트라이는 다음과 같다.

```text
root
├─ c
│  └─ a
│     ├─ t*
│     └─ r*
└─ d
   └─ o
      └─ g*
```

`*`는 단어 끝 표시이다.

## 검색 예시

트라이에 다음 단어들이 저장되어 있다고 하자.

```text
cat
car
dog
```

`cat`을 검색한다.

```text
c 있음
a 있음
t 있음
t가 단어 끝임
```

결과는 `true`이다.

`ca`를 검색한다.

```text
c 있음
a 있음
a는 단어 끝이 아님
```

결과는 `false`이다.

하지만 `starts_with("ca")`는 `true`이다.

## 접두사 검색 예시

자동 완성에서 `ca`로 시작하는 단어를 찾고 싶다고 하자.

먼저 `c -> a`까지 이동한다.

그 아래에 있는 모든 단어 끝 지점을 찾는다.

```text
ca -> t = cat
ca -> r = car
ca -> r -> e = care
```

결과는 다음과 같다.

```text
cat
car
care
```

이처럼 트라이는 자동 완성에 잘 어울린다.

## 삭제

트라이에서 단어를 삭제할 때는 단어 끝 표시를 먼저 제거한다.

```text
current.is_end = false
```

그 뒤 더 이상 필요 없는 노드를 뒤에서부터 지울 수 있다.

예를 들어 `care`와 `car`가 모두 있다면:

```text
c -> a -> r -> e
```

`care`를 삭제해도 `car`는 남아야 한다.

그래서 `r` 노드는 지우면 안 된다.

삭제할 때는 다음을 확인해야 한다.

```text
자식 노드가 없는가?
다른 단어의 끝이 아닌가?
```

두 조건을 만족할 때만 노드를 제거할 수 있다.

## 자동 완성

트라이의 대표적인 활용은 자동 완성이다.

사용자가 `app`까지 입력했다고 하자.

트라이에서 `a -> p -> p`까지 이동한다.

그 아래에 있는 단어들을 DFS로 모으면 자동 완성 후보가 된다.

```text
app
apple
apply
application
```

접두사를 빠르게 찾고, 그 아래 단어들을 탐색하는 구조가 자연스럽다.

## 사전 검색

트라이는 사전처럼 단어를 저장하고 검색할 때도 사용할 수 있다.

```text
insert("hello")
search("hello") -> true
search("hell") -> false
starts_with("hell") -> true
```

정확한 단어 검색과 접두사 검색을 구분할 수 있다는 점이 중요하다.

## 배열과 해시맵 구현

자식 노드를 저장하는 방법은 크게 두 가지가 있다.

```text
배열:
    알파벳 종류가 정해져 있을 때 빠르다.
    사용하지 않는 칸이 많을 수 있다.

해시맵:
    필요한 문자만 저장한다.
    다양한 문자를 다룰 때 편하다.
```

예를 들어 영어 소문자만 다룬다면 배열이 좋을 수 있다.

한글, 숫자, 특수문자까지 다룬다면 해시맵이 더 자연스럽다.

## 빈 문자열

빈 문자열을 저장할 수도 있다.

```text
insert("")
```

이 경우 루트 노드 자체에 단어 끝 표시를 할 수 있다.

하지만 대부분의 문제에서는 빈 문자열을 따로 다루지 않는다.

필요하다면 처음에 규칙을 정해 두는 것이 좋다.

## 시간 복잡도

단어 길이를 `L`이라고 하자.

삽입은 문자를 하나씩 따라가므로 다음과 같다.

```text
insert: O(L)
```

정확한 검색도 문자 하나씩 따라간다.

```text
search: O(L)
```

접두사 검색도 접두사 길이를 `P`라고 하면 다음과 같다.

```text
starts_with: O(P)
```

자동 완성처럼 접두사 아래의 모든 단어를 가져오면, 찾은 단어들의 전체 길이만큼 추가 시간이 걸린다.

## 공간 복잡도

저장한 모든 문자열의 전체 길이를 `S`라고 하자.

최악의 경우 공통 접두사가 거의 없다면 문자마다 노드가 하나씩 필요하다.

```text
공간 복잡도: O(S)
```

공통 접두사가 많으면 노드를 공유하므로 실제 사용 공간은 줄어들 수 있다.

## 장점

```text
문자열 검색이 빠르다.
접두사 검색이 매우 효율적이다.
자동 완성 기능을 만들기 좋다.
공통 접두사를 공유해 저장할 수 있다.
```

## 단점

```text
해시 테이블보다 메모리를 많이 쓸 수 있다.
구현이 단순 배열 검색보다 복잡하다.
정렬된 전체 순회가 필요하면 별도 탐색이 필요하다.
문자 종류가 많으면 자식 관리가 복잡해질 수 있다.
```

## 해시 테이블과 비교

```text
해시 테이블:
    정확한 단어 검색이 빠르다.
    접두사 검색은 따로 처리하기 어렵다.

트라이:
    정확한 단어 검색도 빠르다.
    접두사 검색과 자동 완성에 강하다.
```

단어가 있는지만 확인하면 해시 테이블이 더 간단할 수 있다.

접두사 기반 기능이 필요하면 트라이가 더 잘 맞는다.

## 언제 사용하면 좋은가

트라이는 다음 상황에서 좋다.

```text
자동 완성
사전 검색
접두사 검색
문자열 집합 관리
단어 추천
IP 라우팅 같은 prefix 기반 문제
```

## 주의할 점

단어 끝 표시를 빼먹으면 정확한 검색과 접두사 검색을 구분할 수 없다.

예를 들어 `care`만 저장했는데 `car`를 검색한다고 하자.

```text
c -> a -> r 경로는 존재한다.
```

하지만 `r`이 단어 끝이 아니라면 `car`는 저장된 단어가 아니다.

```text
search("car") = false
starts_with("car") = true
```

이 차이를 꼭 기억해야 한다.

## 한 문장 요약

트라이는 문자열을 문자 경로로 저장해 정확한 검색과 접두사 검색을 빠르게 하는 트리 자료 구조이다.

## 머릿속 결정 순서

```text
1. 저장할 문자열을 문자 단위로 볼 것인가?
2. 루트 노드를 만들었는가?
3. 각 노드에 자식 정보가 있는가?
4. 삽입할 때 없는 자식 노드를 새로 만드는가?
5. 단어 끝 표시를 했는가?
6. search와 starts_with를 구분하는가?
```
