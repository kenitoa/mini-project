# 라빈-카프 알고리즘

라빈-카프 알고리즘은 문자열에서 특정 패턴을 찾는 문자열 검색 알고리즘이다.

문자 하나하나를 계속 비교하는 대신, 문자열 조각의 해시 값을 비교한다.

```text
텍스트: 전체 문자열
패턴: 찾고 싶은 문자열
해시: 문자열을 숫자로 바꾼 값
```

라빈-카프는 특히 롤링 해시를 사용해 다음 문자열 조각의 해시 값을 빠르게 계산한다.

## 핵심 생각

예를 들어 다음 텍스트에서 `CAB`를 찾는다고 하자.

```text
text    = ABCABC
pattern = CAB
```

패턴 길이는 `3`이다.

텍스트에서 길이 `3`인 부분 문자열을 차례대로 본다.

```text
ABC
BCA
CAB
ABC
```

일반적인 방식은 각 부분 문자열과 패턴을 문자 단위로 비교한다.

라빈-카프는 먼저 각 문자열의 해시 값을 비교한다.

```text
hash(ABC)
hash(BCA)
hash(CAB)
hash(pattern)
```

해시 값이 다르면 문자열도 다르다고 보고 바로 넘어간다.

해시 값이 같으면 실제 문자열을 한 번 더 비교해 확인한다.

## 문자열에서 사용하는 말

라빈-카프를 이해하려면 다음 말을 알아두면 좋다.

```text
텍스트: 검색 대상이 되는 전체 문자열
패턴: 텍스트 안에서 찾고 싶은 문자열
부분 문자열: 텍스트의 일부 연속된 문자열
해시 값: 문자열을 계산해서 만든 숫자
롤링 해시: 한 칸 이동한 문자열의 해시를 빠르게 갱신하는 방법
충돌: 서로 다른 문자열인데 해시 값이 같은 경우
```

해시 값만 믿으면 충돌 때문에 틀릴 수 있다.

그래서 해시가 같을 때는 실제 문자열도 비교하는 것이 안전하다.

## 롤링 해시

라빈-카프의 핵심은 롤링 해시이다.

길이가 같은 부분 문자열을 한 칸씩 옮길 때, 해시 값을 처음부터 다시 계산하지 않는다.

예를 들어 길이 `3`의 부분 문자열을 보자.

```text
ABC -> BCA
```

앞의 `A`는 빠지고, 뒤에 `A`가 새로 들어온다.

```text
빠지는 문자: A
남는 문자: B, C
들어오는 문자: A
```

롤링 해시는 기존 해시에서 빠지는 문자의 영향을 제거하고, 새 문자의 영향을 추가한다.

그래서 매번 `O(M)`이 아니라 `O(1)`에 가까운 시간으로 다음 해시를 만들 수 있다.

## 해시 계산 방식

문자열을 숫자로 보는 방식은 여러 가지가 있다.

자주 쓰는 방식은 진법처럼 계산하는 것이다.

```text
hash("ABC") = A * base^2 + B * base^1 + C * base^0
```

여기서 `A`, `B`, `C`는 문자에 대응하는 숫자이고, `base`는 기준이 되는 수이다.

값이 너무 커지지 않도록 나머지 연산을 함께 사용한다.

```text
hash = hash % mod
```

보통 `base`와 `mod`를 잘 골라 해시 충돌 가능성을 낮춘다.

## 라빈-카프 의사코드

```text
rabin_karp(텍스트, 패턴):
    패턴 길이를 m이라고 한다
    텍스트 길이를 n이라고 한다

    패턴의 해시 값을 계산한다
    텍스트의 첫 m글자 해시 값을 계산한다

    i를 0부터 n - m까지 반복한다:
        패턴 해시와 현재 부분 문자열 해시가 같다면:
            실제 문자열도 같은지 비교한다
            같다면 현재 위치를 결과에 추가한다

        다음 부분 문자열이 있다면:
            롤링 해시로 해시 값을 갱신한다

    결과를 반환한다
```

해시가 같은 경우에만 실제 문자열 비교를 한다.

## 조금 더 코드처럼 쓴 의사코드

```text
function rabin_karp(text, pattern):
    n = length(text)
    m = length(pattern)
    result = []

    pattern_hash = hash(pattern)
    window_hash = hash(text[0:m])

    for i from 0 to n - m:
        if pattern_hash == window_hash:
            if text[i:i + m] == pattern:
                add i to result

        if i < n - m:
            window_hash = roll_hash(
                window_hash,
                text[i],
                text[i + m],
                m
            )

    return result
```

`roll_hash`는 맨 앞 문자를 빼고 새 문자를 뒤에 추가한 해시 값을 만든다.

## 롤링 해시 의사코드

다음은 개념적인 롤링 해시이다.

```text
function roll_hash(old_hash, old_char, new_char, m):
    old_hash에서 old_char * base^(m - 1)을 뺀다
    남은 값에 base를 곱한다
    new_char 값을 더한다
    mod로 나눈 나머지를 반환한다
```

수식으로 보면 다음과 같다.

```text
new_hash = (old_hash - old_char * base^(m - 1)) * base + new_char
new_hash = new_hash % mod
```

음수가 나올 수 있으므로 실제 구현에서는 보정이 필요하다.

```text
new_hash = (new_hash + mod) % mod
```

## 흐름을 쉽게 풀어보기

라빈-카프는 이렇게 생각하면 쉽다.

```text
1. 찾을 패턴을 숫자로 바꾼다.
2. 텍스트의 같은 길이 조각도 숫자로 바꾼다.
3. 숫자가 다르면 바로 넘어간다.
4. 숫자가 같으면 실제 문자열을 확인한다.
5. 다음 조각의 숫자는 롤링 해시로 빠르게 만든다.
```

즉, 문자열 비교를 매번 길게 하지 않기 위해 해시를 먼저 비교한다.

## 예시

다음 텍스트에서 `CAB`를 찾는다고 하자.

```text
text    = ABCABC
pattern = CAB
```

패턴 길이는 `3`이다.

비교할 부분 문자열은 다음과 같다.

```text
ABC
BCA
CAB
ABC
```

각 부분 문자열의 해시를 비교한다.

```text
hash(ABC) != hash(CAB)
hash(BCA) != hash(CAB)
hash(CAB) == hash(CAB)
```

해시가 같은 위치에서 실제 문자열도 비교한다.

```text
text[2:5] == CAB
```

따라서 패턴은 인덱스 `2`에서 발견된다.

```text
result = [2]
```

## 해시 충돌

해시 충돌은 서로 다른 문자열인데 해시 값이 같은 경우이다.

```text
hash("ABC") == hash("XYZ")
```

이런 일이 생길 수 있기 때문에 해시 값만 같다고 바로 정답으로 처리하면 위험하다.

라빈-카프에서는 보통 해시가 같을 때 실제 문자열을 한 번 더 비교한다.

```text
if hash is same:
    compare actual strings
```

이 확인 덕분에 충돌이 있어도 틀린 결과를 줄일 수 있다.

## 여러 패턴 검색

라빈-카프는 여러 패턴을 한 번에 찾을 때도 응용할 수 있다.

패턴들의 길이가 같다면 각 패턴의 해시 값을 미리 집합에 넣어 둔다.

```text
pattern_hashes = {hash(pattern1), hash(pattern2), hash(pattern3)}
```

텍스트의 각 부분 문자열 해시가 이 집합에 있는지 확인한다.

```text
if window_hash is in pattern_hashes:
    실제 문자열 비교
```

이런 방식은 많은 패턴을 검색할 때 유용할 수 있다.

## 패턴이 없는 경우

패턴이 텍스트에 없으면 해시가 같아지는 위치가 없거나, 해시가 같아도 실제 문자열 비교에서 실패한다.

```text
text    = ABCDE
pattern = XYZ
```

결과는 빈 리스트이다.

```text
result = []
```

## 패턴이 텍스트보다 긴 경우

패턴이 텍스트보다 길면 찾을 수 없다.

```text
text    = ABC
pattern = ABCD
```

이 경우 바로 빈 결과를 반환하면 된다.

```text
if length(pattern) > length(text):
    return []
```

## 빈 패턴인 경우

패턴이 빈 문자열이면 문제에 따라 처리 방식이 달라진다.

```text
pattern = ""
```

일반적으로 알고리즘 구현에서는 별도로 처리한다.

```text
if pattern is empty:
    return []
```

또는 문제 요구에 맞게 모든 위치를 반환할 수도 있다.

## KMP와 비교

```text
라빈-카프:
    해시를 이용한다.
    여러 패턴 검색에 응용하기 좋다.
    해시 충돌을 조심해야 한다.

KMP:
    접두사와 접미사 정보를 이용한다.
    충돌 문제가 없다.
    단일 패턴 검색에서 안정적으로 O(N + M)이다.
```

정확하고 안정적인 단일 패턴 검색에는 KMP가 자주 쓰인다.

해시 기반으로 빠르게 후보를 거르고 싶다면 라빈-카프가 유용하다.

## 시간 복잡도

텍스트 길이를 `N`, 패턴 길이를 `M`이라고 하자.

초기 해시 계산은 `O(M)`이다.

롤링 해시를 사용하면 각 위치에서 해시 갱신은 `O(1)`이다.

평균적으로는 다음과 같다.

```text
평균 시간 복잡도: O(N + M)
```

하지만 해시 충돌이 많이 발생하면 실제 문자열 비교가 자주 일어난다.

최악의 경우는 다음과 같다.

```text
최악 시간 복잡도: O(NM)
```

## 공간 복잡도

기본적으로 해시 값과 몇 개의 변수만 필요하다.

```text
공간 복잡도: O(1)
```

결과 위치를 저장하면 찾은 개수만큼 추가 공간이 필요하다.

여러 패턴의 해시를 저장하면 패턴 개수만큼 공간이 필요하다.

## 장점

```text
롤링 해시를 사용하면 빠르게 검색할 수 있다.
여러 패턴 검색으로 확장하기 좋다.
아이디어가 직관적이다.
부분 문자열 비교 후보를 빠르게 줄일 수 있다.
```

## 단점

```text
해시 충돌을 조심해야 한다.
최악의 경우 O(NM)이 될 수 있다.
mod와 base 선택에 신경 써야 한다.
정확성을 위해 실제 문자열 비교가 필요할 수 있다.
```

## 언제 사용하면 좋은가

라빈-카프는 다음 상황에서 좋다.

```text
긴 텍스트에서 패턴을 빠르게 찾고 싶을 때
여러 패턴을 동시에 검색하고 싶을 때
부분 문자열 해시를 활용해야 할 때
문자열 비교 후보를 빠르게 걸러내고 싶을 때
```

## 주의할 점

해시 값이 같다고 문자열이 반드시 같은 것은 아니다.

```text
해시 같음 -> 후보
문자열 직접 비교 -> 최종 확인
```

또한 롤링 해시에서 음수가 생길 수 있으므로 나머지 연산을 할 때 보정해야 한다.

```text
value = (value + mod) % mod
```

## 한 문장 요약

라빈-카프는 부분 문자열의 해시 값을 롤링 해시로 빠르게 갱신하며 패턴을 찾는 문자열 검색 알고리즘이다.

## 알고리즘 흐름도

```text
시작
  |
  v
패턴 해시 계산
  |
  v
텍스트 첫 윈도우 해시 계산
  |
  v
현재 윈도우 해시와 패턴 해시 비교
  |
  v
해시가 같은가?
  | yes
  v
실제 문자열 비교
  |
  v
같으면 위치 기록
  |
  no
  v
다음 윈도우가 있는가?
  | yes
  v
롤링 해시로 갱신
  |
  v
반복
  |
  no
  v
결과 반환
```

## 더 짧게 외우는 흐름

```text
패턴 해시 만들기
텍스트 조각 해시 만들기
해시 비교
같으면 문자열 확인
한 칸 이동하며 해시 갱신
```

## 머릿속 결정 순서

```text
1. 패턴 길이를 정했는가?
2. 패턴 해시와 첫 윈도우 해시를 계산했는가?
3. 해시가 같을 때 실제 문자열을 확인하는가?
4. 다음 윈도우 해시를 롤링 방식으로 갱신하는가?
5. 해시 충돌 가능성을 고려했는가?
6. 패턴이 텍스트보다 긴 경우를 처리했는가?
```
