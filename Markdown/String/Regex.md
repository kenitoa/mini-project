# 정규 표현식

정규 표현식은 문자열에서 특정한 패턴을 찾거나 검사하기 위한 표현 방법이다.

영어로는 Regular Expression이라고 하고, 줄여서 Regex라고 부른다.

정규 표현식은 문자 그대로의 검색뿐 아니라, 숫자, 공백, 반복, 시작과 끝 같은 규칙을 한 줄의 패턴으로 표현할 수 있다.

```text
문자열 검색
문자열 검증
문자열 치환
문자열 분리
```

예를 들어 이메일 형식인지 확인하거나, 문장에서 숫자만 찾는 일에 사용할 수 있다.

## 핵심 생각

예를 들어 다음 문자열에서 숫자를 찾고 싶다고 하자.

```text
오늘은 2026년 4월 30일이다.
```

숫자를 직접 하나씩 찾을 수도 있지만, 정규 표현식을 쓰면 숫자 패턴을 표현할 수 있다.

```text
\d+
```

여기서 `\d`는 숫자 하나를 뜻하고, `+`는 하나 이상 반복을 뜻한다.

그래서 다음 숫자들을 찾을 수 있다.

```text
2026
4
30
```

정규 표현식은 "어떤 문자열을 찾을 것인가"를 규칙으로 적는 도구이다.

## 문자열에서 사용하는 말

정규 표현식을 이해하려면 다음 말을 알아두면 좋다.

```text
패턴: 찾거나 검사하고 싶은 규칙
매칭: 문자열이 패턴과 맞는 것
메타 문자: 특별한 의미를 가진 문자
리터럴: 특별한 의미 없이 문자 그대로 보는 문자
그룹: 패턴의 일부를 묶은 것
캡처: 매칭된 일부 문자열을 따로 저장하는 것
```

예를 들어 `cat`이라는 패턴은 문자 그대로 `cat`을 찾는다.

```text
cat
```

하지만 `.`은 문자 그대로의 점이 아니라, 보통 임의의 한 문자를 뜻한다.

```text
c.t
```

이 패턴은 다음과 매칭될 수 있다.

```text
cat
cut
c9t
```

## 자주 쓰는 기본 패턴

정규 표현식에는 자주 쓰는 기호들이 있다.

```text
.      임의의 문자 하나
\d     숫자 하나
\D     숫자가 아닌 문자 하나
\w     문자, 숫자, 밑줄 중 하나
\W     문자, 숫자, 밑줄이 아닌 문자 하나
\s     공백 문자 하나
\S     공백이 아닌 문자 하나
```

예를 들어 숫자 세 개를 찾고 싶다면 다음처럼 쓸 수 있다.

```text
\d\d\d
```

또는 반복 기호를 사용해 다음처럼 쓸 수 있다.

```text
\d{3}
```

## 문자 클래스

대괄호 `[]` 안에 가능한 문자를 적으면 그중 하나와 매칭된다.

```text
[abc]
```

이 패턴은 `a`, `b`, `c` 중 하나와 매칭된다.

범위를 사용할 수도 있다.

```text
[a-z]
[A-Z]
[0-9]
```

예를 들어 영어 소문자 하나를 찾고 싶다면:

```text
[a-z]
```

영어 대소문자 또는 숫자 하나를 찾고 싶다면:

```text
[a-zA-Z0-9]
```

## 부정 문자 클래스

대괄호 안에서 `^`를 맨 앞에 쓰면 해당 문자들이 아닌 것을 뜻한다.

```text
[^0-9]
```

이 패턴은 숫자가 아닌 문자 하나와 매칭된다.

```text
[^a-z]
```

이 패턴은 영어 소문자가 아닌 문자 하나와 매칭된다.

## 반복 기호

반복 기호는 앞의 패턴이 몇 번 나와야 하는지 나타낸다.

```text
*      0번 이상
+      1번 이상
?      0번 또는 1번
{n}    정확히 n번
{n,}   n번 이상
{n,m}  n번 이상 m번 이하
```

예를 들어 숫자가 하나 이상 나오는 부분을 찾고 싶다면:

```text
\d+
```

전화번호처럼 숫자 3개, 숫자 4개, 숫자 4개를 찾고 싶다면:

```text
\d{3}-\d{4}-\d{4}
```

## 시작과 끝

문자열의 시작과 끝을 표현할 수도 있다.

```text
^    문자열의 시작
$    문자열의 끝
```

예를 들어 문자열 전체가 숫자로만 되어 있는지 확인하려면 다음처럼 쓴다.

```text
^\d+$
```

`^`와 `$`가 없으면 문자열 일부에 숫자가 있기만 해도 매칭될 수 있다.

```text
abc123
```

`\d+`는 `123`과 매칭되지만, `^\d+$`는 전체가 숫자가 아니므로 매칭되지 않는다.

## 그룹

괄호 `()`를 사용하면 패턴을 묶을 수 있다.

```text
(ab)+
```

이 패턴은 `ab`가 하나 이상 반복되는 문자열과 매칭된다.

```text
ab
abab
ababab
```

그룹은 일부 문자열을 따로 추출할 때도 사용한다.

예를 들어 날짜를 연, 월, 일로 나누고 싶다면:

```text
(\d{4})-(\d{2})-(\d{2})
```

문자열 `2026-04-30`에서 다음 값을 얻을 수 있다.

```text
그룹 1: 2026
그룹 2: 04
그룹 3: 30
```

## 또는

세로선 `|`는 또는을 뜻한다.

```text
cat|dog
```

이 패턴은 `cat` 또는 `dog`와 매칭된다.

그룹과 함께 쓰면 범위를 더 명확하게 만들 수 있다.

```text
(cat|dog)s?
```

이 패턴은 다음과 매칭될 수 있다.

```text
cat
cats
dog
dogs
```

## 이스케이프

정규 표현식에서 특별한 의미를 가진 문자를 문자 그대로 찾고 싶다면 이스케이프가 필요하다.

예를 들어 점 `.`을 실제 점으로 찾고 싶다면:

```text
\.
```

괄호를 문자 그대로 찾고 싶다면:

```text
\(
\)
```

역슬래시 자체를 문자열 안에 적을 때는 사용하는 언어에 따라 한 번 더 이스케이프해야 할 수 있다.

```text
정규 표현식: \d+
문자열 리터럴: "\\d+"
```

## 정규 표현식 의사코드

정규 표현식 자체는 알고리즘이라기보다 패턴 표현 방식이다.

사용 흐름은 보통 다음과 같다.

```text
regex_search(문자열, 패턴):
    패턴을 정규 표현식으로 준비한다
    문자열에서 패턴과 맞는 부분을 찾는다

    찾았다면:
        매칭된 문자열 또는 위치를 반환한다
    찾지 못했다면:
        없음 값을 반환한다
```

검증할 때는 문자열 전체가 패턴과 맞는지 확인한다.

```text
regex_match(문자열, 패턴):
    패턴 앞에 시작 표시를 둔다
    패턴 뒤에 끝 표시를 둔다
    문자열 전체가 패턴과 맞는지 확인한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function find_numbers(text):
    pattern = "\d+"
    result = regex_find_all(text, pattern)
    return result
```

문자열 전체 형식을 검사하는 경우:

```text
function is_number(text):
    pattern = "^\d+$"

    if regex_match(text, pattern):
        return true
    else:
        return false
```

치환하는 경우:

```text
function hide_numbers(text):
    pattern = "\d"
    return regex_replace(text, pattern, "*")
```

## 흐름을 쉽게 풀어보기

정규 표현식은 이렇게 생각하면 쉽다.

```text
1. 찾고 싶은 문자열의 규칙을 생각한다.
2. 그 규칙을 기호로 표현한다.
3. 문자열 일부를 찾을지, 전체를 검사할지 정한다.
4. 필요한 경우 그룹으로 일부 값을 추출한다.
5. 반복, 선택, 시작과 끝을 조합한다.
```

즉, 정규 표현식은 문자열을 직접 하나씩 비교하는 코드 대신 패턴으로 조건을 적는 방법이다.

## 예시: 숫자 찾기

문자열에서 숫자 덩어리를 모두 찾고 싶다고 하자.

```text
text = "A12 B3 C456"
pattern = "\d+"
```

결과는 다음과 같다.

```text
12
3
456
```

`\d`는 숫자 하나이고, `+`는 하나 이상 반복이므로 숫자가 이어진 덩어리를 찾는다.

## 예시: 이메일 형식 검사

간단한 이메일 형식은 다음처럼 검사할 수 있다.

```text
^[\w.-]+@[\w.-]+\.[A-Za-z]{2,}$
```

의미를 나누어 보면 다음과 같다.

```text
^[\w.-]+     앞부분은 문자, 숫자, 점, 하이픈 등이 하나 이상
@            @ 문자
[\w.-]+      도메인 이름
\.           실제 점
[A-Za-z]{2,} 끝부분은 영어 2글자 이상
$            문자열 끝
```

실제 이메일 형식은 더 복잡할 수 있으므로, 중요한 서비스에서는 검증 규칙을 신중하게 정해야 한다.

## 예시: 날짜 추출

다음 문자열에서 날짜를 찾는다고 하자.

```text
오늘 날짜는 2026-04-30 입니다.
```

패턴:

```text
(\d{4})-(\d{2})-(\d{2})
```

결과:

```text
전체 매칭: 2026-04-30
그룹 1: 2026
그룹 2: 04
그룹 3: 30
```

그룹을 사용하면 필요한 부분만 따로 가져올 수 있다.

## 탐욕적 반복과 게으른 반복

반복 기호는 기본적으로 가능한 많이 매칭하려고 한다.

이를 탐욕적이라고 한다.

```text
<.*>
```

문자열이 다음과 같다면:

```text
<b>hello</b>
```

`<.*>`는 처음 `<`부터 마지막 `>`까지 잡을 수 있다.

```text
<b>hello</b>
```

가능한 적게 잡고 싶다면 `?`를 붙여 게으른 반복으로 만들 수 있다.

```text
<.*?>
```

이 경우 먼저 나오는 태그 단위로 매칭될 수 있다.

```text
<b>
</b>
```

## 검색과 전체 매칭의 차이

검색은 문자열 안에 패턴이 포함되어 있는지 본다.

```text
pattern = "\d+"
text = "abc123"
```

검색은 성공한다.

```text
123
```

전체 매칭은 문자열 전체가 패턴과 맞아야 한다.

```text
pattern = "^\d+$"
text = "abc123"
```

이 경우 실패한다.

문자열 전체를 검사할 때는 `^`와 `$`를 자주 사용한다.

## 정규 표현식 엔진

정규 표현식은 사용하는 언어와 엔진에 따라 세부 문법이 조금 다를 수 있다.

```text
JavaScript
Python
Java
C#
PCRE
```

기본적인 기호는 비슷하지만, lookahead, lookbehind, 유니코드 처리, 플래그 사용 방식은 다를 수 있다.

따라서 실제 코드에서는 사용하는 언어의 정규 표현식 문법을 확인해야 한다.

## 시간 복잡도

정규 표현식의 시간 복잡도는 패턴과 엔진에 따라 달라진다.

단순한 패턴은 문자열 길이에 비례해 빠르게 동작할 수 있다.

```text
단순 검색: 대체로 O(N)
```

하지만 복잡한 백트래킹 패턴은 매우 느려질 수 있다.

예를 들어 중첩 반복이 있는 패턴은 조심해야 한다.

```text
(a+)+
```

이런 패턴은 특정 입력에서 많은 경우를 되돌아가며 확인할 수 있다.

## 공간 복잡도

일반적인 사용에서는 패턴과 매칭 결과를 저장하는 정도의 공간이 필요하다.

```text
공간 복잡도: 패턴과 결과 크기에 따라 달라짐
```

그룹을 많이 캡처하거나 모든 매칭 결과를 저장하면 그만큼 공간이 늘어난다.

## 장점

```text
문자열 규칙을 짧게 표현할 수 있다.
검색, 검증, 추출, 치환에 모두 사용할 수 있다.
반복과 선택 같은 조건을 쉽게 표현할 수 있다.
여러 언어에서 널리 지원된다.
```

## 단점

```text
복잡해지면 읽기 어렵다.
엔진마다 세부 문법이 다를 수 있다.
잘못 작성하면 너무 많은 문자열을 매칭할 수 있다.
복잡한 패턴은 성능 문제가 생길 수 있다.
```

## 언제 사용하면 좋은가

정규 표현식은 다음 상황에서 좋다.

```text
문자열에서 특정 형식을 찾을 때
입력값 형식을 검사할 때
필요한 부분 문자열을 추출할 때
문자열을 규칙에 따라 바꿀 때
로그나 텍스트 데이터를 빠르게 처리할 때
```

## 주의할 점

정규 표현식은 강력하지만 너무 많은 일을 한 패턴에 넣으면 읽기 어려워진다.

간단한 문자열 처리로 충분하면 정규 표현식을 쓰지 않는 편이 더 명확할 수 있다.

또한 전체 검증이 필요할 때는 시작과 끝을 꼭 확인해야 한다.

```text
^\d+$
```

그냥 `\d+`만 쓰면 문자열 일부에 숫자가 있는 경우도 매칭될 수 있다.

## 한 문장 요약

정규 표현식은 문자열에서 원하는 규칙을 패턴으로 표현해 검색, 검증, 추출, 치환을 하는 방법이다.

## 알고리즘 흐름도

```text
시작
  |
  v
찾고 싶은 문자열 규칙 정하기
  |
  v
정규 표현식 패턴 작성
  |
  v
검색인지 전체 검증인지 결정
  |
  v
문자열에 패턴 적용
  |
  v
매칭되는가?
  | yes
  v
매칭 결과 또는 그룹 추출
  |
  no
  v
없음 또는 실패 반환
```

## 더 짧게 외우는 흐름

```text
규칙 생각하기
패턴으로 쓰기
문자열에 적용하기
매칭 결과 확인하기
필요하면 추출하거나 치환하기
```

## 머릿속 결정 순서

```text
1. 문자 그대로 찾을 것인가, 규칙을 찾을 것인가?
2. 문자열 일부 검색인가, 전체 검증인가?
3. 반복 횟수를 정해야 하는가?
4. 그룹으로 추출할 값이 있는가?
5. 특수 문자를 이스케이프했는가?
6. 너무 복잡하거나 느린 패턴은 아닌가?
```
