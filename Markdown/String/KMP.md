# KMP 알고리즘

KMP 알고리즘은 문자열에서 특정 패턴이 어디에 나타나는지 빠르게 찾는 문자열 검색 알고리즘이다.

정식 이름은 Knuth-Morris-Pratt 알고리즘이다.

일반적인 문자열 비교는 틀릴 때마다 패턴을 한 칸씩 옮기며 처음부터 다시 비교한다.

KMP는 이미 비교한 정보를 이용해서, 다시 비교하지 않아도 되는 부분을 건너뛴다.

```text
텍스트: 전체 문자열
패턴: 찾고 싶은 문자열
```

KMP의 핵심은 패턴 안에서 접두사와 접미사가 얼마나 겹치는지 미리 계산해 두는 것이다.

이 정보를 보통 `LPS 배열` 또는 `실패 함수`라고 부른다.

## 핵심 생각

예를 들어 다음 문자열에서 `ABABC`를 찾는다고 하자.

```text
text    = ABABABC
pattern = ABABC
```

앞에서부터 비교하면 다음처럼 진행된다.

```text
text:    A B A B A B C
pattern: A B A B C
```

처음 네 글자 `A B A B`는 같다.

하지만 다섯 번째에서 다르다.

```text
text의 글자:    A
pattern의 글자: C
```

일반적인 방식이라면 패턴을 한 칸 옮기고 처음부터 다시 비교한다.

하지만 이미 `ABAB`까지 같다는 사실을 알고 있다.

`ABAB` 안에는 앞부분 `AB`와 뒷부분 `AB`가 겹친다.

```text
ABAB
앞의 AB == 뒤의 AB
```

그래서 패턴을 무작정 처음으로 돌리지 않고, 겹치는 부분을 활용해서 비교 위치를 조정한다.

이 덕분에 텍스트의 같은 문자를 여러 번 되돌아가며 비교하지 않는다.

## 문자열에서 사용하는 말

KMP를 이해하려면 다음 말을 알아두면 좋다.

```text
텍스트: 검색 대상이 되는 전체 문자열
패턴: 텍스트 안에서 찾고 싶은 문자열
접두사: 문자열의 앞에서 시작하는 부분 문자열
접미사: 문자열의 뒤에서 끝나는 부분 문자열
LPS: 접두사와 접미사가 같은 가장 긴 길이
```

예를 들어 `ABAB`의 접두사와 접미사를 보자.

```text
접두사: A, AB, ABA
접미사: B, AB, BAB
```

접두사와 접미사가 같은 것 중 가장 긴 것은 `AB`이다.

```text
LPS 길이 = 2
```

## LPS 배열

LPS는 Longest Prefix Suffix의 줄임말이다.

각 위치까지의 문자열에서, 접두사와 접미사가 같은 가장 긴 길이를 저장한다.

패턴이 다음과 같다고 하자.

```text
pattern = ABABC
```

각 위치까지의 LPS를 구하면 다음과 같다.

```text
문자열      LPS
A          0
AB         0
ABA        1
ABAB       2
ABABC      0
```

그래서 LPS 배열은 다음과 같다.

```text
[0, 0, 1, 2, 0]
```

이 배열은 비교가 실패했을 때 패턴의 어느 위치부터 다시 비교하면 되는지 알려준다.

## KMP 의사코드

KMP는 크게 두 단계로 나뉜다.

```text
1. 패턴의 LPS 배열을 만든다.
2. LPS 배열을 이용해 텍스트에서 패턴을 찾는다.
```

전체 흐름은 다음과 같다.

```text
kmp_search(텍스트, 패턴):
    패턴의 LPS 배열을 만든다
    텍스트 위치 i를 0으로 둔다
    패턴 위치 j를 0으로 둔다

    텍스트 끝까지 반복한다:
        텍스트[i]와 패턴[j]가 같다면:
            i와 j를 1씩 증가시킨다

            j가 패턴 길이와 같아졌다면:
                패턴을 찾은 위치를 기록한다
                j를 LPS[j - 1]로 옮긴다

        다르다면:
            j가 0보다 크면:
                j를 LPS[j - 1]로 옮긴다
            아니면:
                i를 1 증가시킨다
```

`i`는 텍스트의 위치이고, `j`는 패턴의 위치이다.

## 조금 더 코드처럼 쓴 의사코드

먼저 LPS 배열을 만든다.

```text
function build_lps(pattern):
    lps = array filled with 0
    length = 0
    i = 1

    while i < length of pattern:
        if pattern[i] == pattern[length]:
            length = length + 1
            lps[i] = length
            i = i + 1
        else:
            if length != 0:
                length = lps[length - 1]
            else:
                lps[i] = 0
                i = i + 1

    return lps
```

그 다음 텍스트에서 패턴을 찾는다.

```text
function kmp_search(text, pattern):
    lps = build_lps(pattern)
    result = []
    i = 0
    j = 0

    while i < length of text:
        if text[i] == pattern[j]:
            i = i + 1
            j = j + 1

            if j == length of pattern:
                add i - j to result
                j = lps[j - 1]
        else:
            if j != 0:
                j = lps[j - 1]
            else:
                i = i + 1

    return result
```

`result`에는 패턴이 시작하는 위치들이 들어간다.

## 흐름을 쉽게 풀어보기

KMP는 이렇게 생각하면 쉽다.

```text
1. 패턴 안에서 겹치는 앞부분과 뒷부분을 미리 계산한다.
2. 텍스트와 패턴을 앞에서부터 비교한다.
3. 문자가 같으면 둘 다 앞으로 간다.
4. 문자가 다르면 패턴만 적절한 위치로 되돌린다.
5. 텍스트 위치는 되돌리지 않는다.
```

즉, 실패했을 때 처음부터 다시 비교하지 않기 위해 준비해 둔 알고리즘이다.

## 예시

다음 텍스트에서 `ABABC`를 찾는다고 하자.

```text
text    = ABABABC
pattern = ABABC
```

패턴의 LPS 배열은 다음과 같다.

```text
pattern: A B A B C
lps:     0 0 1 2 0
```

처음 비교:

```text
text:    A B A B A B C
pattern: A B A B C
```

`A B A B`까지 같고, 다음 글자에서 실패한다.

```text
text[4] = A
pattern[4] = C
```

이때 `j = 4`였으므로 `j = lps[3]`으로 이동한다.

```text
j = 2
```

패턴의 앞부분 `AB`는 이미 맞는 것으로 볼 수 있으므로, 이어서 비교한다.

```text
text:      A B A B A B C
pattern:      A B A B C
```

이제 끝까지 일치한다.

패턴은 텍스트의 인덱스 `2`에서 시작한다.

```text
result = [2]
```

## 패턴이 여러 번 나오는 경우

KMP는 패턴이 여러 번 나오는 경우도 찾을 수 있다.

```text
text    = ABABAB
pattern = ABAB
```

패턴은 다음 위치에서 나온다.

```text
0
2
```

두 번째 결과는 겹쳐서 등장한다.

```text
ABABAB
ABAB
  ABAB
```

KMP는 패턴을 찾은 뒤에도 `j = lps[j - 1]`로 이동하기 때문에 겹치는 패턴도 찾을 수 있다.

## 패턴이 없는 경우

다음처럼 패턴이 텍스트 안에 없을 수 있다.

```text
text    = ABCDE
pattern = ACD
```

끝까지 비교해도 패턴 길이만큼 일치하는 순간이 없다.

이 경우 결과는 빈 리스트가 된다.

```text
result = []
```

## 빈 패턴인 경우

패턴이 빈 문자열이면 문제에 따라 처리 방식이 달라진다.

```text
pattern = ""
```

어떤 구현에서는 빈 패턴이 모든 위치에 있다고 보기도 하고, 어떤 구현에서는 검색하지 않는 것으로 처리한다.

알고리즘을 구현할 때는 처음에 규칙을 정해 두는 것이 좋다.

```text
if pattern is empty:
    return []
```

또는 문제에서 요구하는 방식에 맞춘다.

## 일반 문자열 검색과 비교

일반적인 완전 탐색 방식은 각 위치에서 패턴을 처음부터 비교한다.

```text
text의 각 위치마다:
    pattern의 처음부터 비교
```

최악의 경우 같은 문자를 여러 번 비교한다.

KMP는 LPS 배열을 이용해 불필요한 비교를 줄인다.

```text
텍스트 위치는 되돌리지 않는다.
패턴 위치만 LPS에 따라 이동한다.
```

## 라빈-카프와 비교

```text
KMP:
    접두사와 접미사 정보를 이용한다.
    충돌 문제가 없다.
    항상 O(N + M)에 동작한다.

라빈-카프:
    해시 값을 이용한다.
    여러 패턴 검색에 응용하기 좋다.
    해시 충돌을 조심해야 한다.
```

문자열 하나의 정확한 검색에는 KMP가 안정적이다.

해시 기반 검색이나 여러 패턴을 다룰 때는 라빈-카프가 편할 수 있다.

## 시간 복잡도

텍스트 길이를 `N`, 패턴 길이를 `M`이라고 하자.

LPS 배열을 만드는 데 `O(M)`이 걸린다.

텍스트를 검색하는 데 `O(N)`이 걸린다.

```text
시간 복잡도: O(N + M)
```

텍스트 위치 `i`는 뒤로 가지 않기 때문에 전체적으로 선형 시간에 끝난다.

## 공간 복잡도

LPS 배열이 필요하다.

```text
lps: O(M)
```

결과 위치를 저장하면 찾은 개수만큼 추가 공간이 필요하다.

```text
공간 복잡도: O(M)
```

결과 리스트까지 포함하면 `O(M + 찾은 개수)`라고 볼 수 있다.

## 장점

```text
최악의 경우에도 O(N + M)에 동작한다.
해시 충돌 같은 문제가 없다.
겹쳐서 등장하는 패턴도 찾을 수 있다.
문자열 검색의 원리를 이해하기 좋다.
```

## 단점

```text
LPS 배열을 이해하기 어렵게 느낄 수 있다.
구현에서 인덱스 실수가 자주 난다.
아주 짧은 문자열에서는 단순 검색이 더 간단할 수 있다.
```

## 언제 사용하면 좋은가

KMP는 다음 상황에서 좋다.

```text
긴 텍스트에서 하나의 패턴을 빠르게 찾을 때
최악의 시간 복잡도를 보장해야 할 때
겹치는 패턴까지 모두 찾아야 할 때
문자열 검색 알고리즘을 직접 구현해야 할 때
```

## 주의할 점

LPS 배열은 패턴에 대해서만 만든다.

텍스트에 대해서 만드는 것이 아니다.

또한 비교가 실패했을 때 `i`를 되돌리지 않고 `j`만 이동한다는 점이 중요하다.

```text
j = lps[j - 1]
```

이 부분이 KMP의 핵심이다.

## 한 문장 요약

KMP는 패턴의 접두사와 접미사 정보를 이용해 불필요한 비교를 건너뛰는 문자열 검색 알고리즘이다.

## 더 짧게 외우는 흐름

```text
LPS 만들기
문자 비교하기
맞으면 둘 다 이동
틀리면 패턴만 LPS만큼 이동
패턴 끝까지 맞으면 위치 기록
```

## 머릿속 결정 순서

```text
1. 찾을 패턴이 무엇인가?
2. 패턴의 LPS 배열을 만들었는가?
3. 텍스트 인덱스 i와 패턴 인덱스 j를 나누어 관리하는가?
4. 문자가 같을 때 i, j를 함께 증가시키는가?
5. 문자가 다를 때 j를 LPS로 이동시키는가?
6. 패턴을 찾은 뒤 겹치는 경우까지 처리하는가?
```
