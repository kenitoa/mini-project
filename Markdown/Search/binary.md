# 이진 탐색

이진 탐색은 정렬되어 있는 데이터에서 가운데 값을 기준으로 탐색 범위를 절반씩 줄여 가며 원하는 값을 찾는 탐색 방법이다.

영어로는 Binary Search라고 부른다.

이진 탐색의 가장 중요한 조건은 데이터가 반드시 정렬되어 있어야 한다는 점이다.

정렬되어 있지 않다면 가운데 값을 보고 왼쪽으로 갈지 오른쪽으로 갈지 판단할 수 없기 때문이다.

## 핵심 생각

예를 들어 이런 정렬된 숫자들이 있다고 하자.

```text
[1, 3, 5, 7, 9, 11, 13]
```

여기서 `11`을 찾고 싶다면, 이진 탐색은 처음부터 하나씩 확인하지 않는다.

먼저 가운데 값을 확인한다.

```text
가운데 값: 7
```

찾는 값 `11`은 `7`보다 크다.

리스트가 오름차순으로 정렬되어 있으므로 `7`의 왼쪽에는 `11`이 있을 수 없다.

그래서 왼쪽 절반을 버리고 오른쪽 절반만 다시 탐색한다.

```text
[9, 11, 13]
```

다시 가운데 값을 확인한다.

```text
가운데 값: 11
```

찾는 값과 같으므로 탐색을 끝낸다.

즉, 이진 탐색은 매번 가운데 값을 확인하고, 필요 없는 절반을 버리는 방식이다.

## 이진 탐색 의사코드

```text
binary_search(리스트, 찾는 값):
    왼쪽 위치를 0으로 정한다
    오른쪽 위치를 리스트의 마지막 위치로 정한다

    왼쪽 위치가 오른쪽 위치보다 작거나 같은 동안 반복한다:
        가운데 위치를 구한다

        가운데 값이 찾는 값과 같다면:
            가운데 위치를 반환한다

        가운데 값이 찾는 값보다 작다면:
            왼쪽 위치를 가운데 위치 + 1로 옮긴다

        가운데 값이 찾는 값보다 크다면:
            오른쪽 위치를 가운데 위치 - 1로 옮긴다

    끝까지 찾지 못했다면:
        찾지 못했다는 값을 반환한다
```

## 조금 더 코드처럼 쓴 의사코드

```text
function binary_search(arr, target):
    left = 0
    right = length(arr) - 1

    while left <= right:
        mid = (left + right) // 2

        if arr[mid] == target:
            return mid

        if arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return -1
```

여기서 `-1`은 원하는 값을 찾지 못했다는 뜻으로 자주 사용된다.

실제 코드에서는 상황에 따라 `false`, `null`, `None` 같은 값을 반환하기도 한다.

## 변수의 의미

이진 탐색에서는 보통 세 개의 위치 변수를 사용한다.

```text
left  : 현재 탐색 범위의 가장 왼쪽 위치
right : 현재 탐색 범위의 가장 오른쪽 위치
mid   : 현재 탐색 범위의 가운데 위치
```

예를 들어 다음 리스트가 있다고 하자.

```text
[1, 3, 5, 7, 9, 11, 13]
```

처음에는 전체 리스트를 탐색해야 하므로 다음과 같이 시작한다.

```text
left = 0
right = 6
mid = 3
```

`mid`가 `3`이므로 가운데 값은 `arr[3]`, 즉 `7`이다.

## 빠르게 떠올려보기

이진 탐색은 이렇게 생각하면 쉽다.

```text
1. 데이터가 정렬되어 있는지 확인한다.
2. 탐색 범위의 왼쪽과 오른쪽 끝을 정한다.
3. 가운데 위치를 구한다.
4. 가운데 값과 찾는 값을 비교한다.
5. 같으면 가운데 위치를 반환한다.
6. 찾는 값이 더 크면 오른쪽 절반만 남긴다.
7. 찾는 값이 더 작으면 왼쪽 절반만 남긴다.
8. 탐색 범위가 사라질 때까지 반복한다.
9. 끝까지 없으면 -1을 반환한다.
```

즉, 이진 탐색은 정렬된 데이터를 반씩 버리면서 찾는 방식이다.

## 예시

```text
binary_search([1, 3, 5, 7, 9, 11, 13], 11)
```

찾는 값은 `11`이다.

처음 탐색 범위는 전체 리스트다.

```text
left = 0
right = 6
mid = (0 + 6) // 2 = 3
arr[mid] = 7
```

가운데 값 `7`은 찾는 값 `11`보다 작다.

정렬된 리스트에서 `11`은 `7`보다 오른쪽에 있어야 하므로 왼쪽 범위를 버린다.

```text
left = mid + 1 = 4
right = 6
```

이제 탐색 범위는 다음과 같다.

```text
[9, 11, 13]
```

다시 가운데 위치를 구한다.

```text
mid = (4 + 6) // 2 = 5
arr[mid] = 11
```

가운데 값이 찾는 값과 같다.

따라서 인덱스 `5`를 반환한다.

```text
return 5
```

## 찾는 값이 없는 예시

```text
binary_search([1, 3, 5, 7, 9, 11, 13], 8)
```

찾는 값은 `8`이다.

처음 가운데 값은 `7`이다.

```text
left = 0
right = 6
mid = 3
arr[mid] = 7
```

`8`은 `7`보다 크므로 오른쪽만 탐색한다.

```text
left = 4
right = 6
```

다음 가운데 값은 `11`이다.

```text
mid = 5
arr[mid] = 11
```

`8`은 `11`보다 작으므로 왼쪽만 탐색한다.

```text
left = 4
right = 4
```

다음 가운데 값은 `9`이다.

```text
mid = 4
arr[mid] = 9
```

`8`은 `9`보다 작으므로 오른쪽 위치를 줄인다.

```text
right = 3
```

이제 상태는 다음과 같다.

```text
left = 4
right = 3
```

`left`가 `right`보다 커졌으므로 더 이상 탐색할 범위가 없다.

따라서 찾지 못했다는 뜻으로 `-1`을 반환한다.

```text
return -1
```

## 종료 조건

이진 탐색이 끝나는 경우는 크게 두 가지다.

첫 번째는 원하는 값을 찾은 경우다.

```text
if arr[mid] == target:
    return mid
```

가운데 값이 찾는 값과 같으면 더 이상 탐색할 필요가 없으므로 바로 종료한다.

두 번째는 탐색 범위가 사라진 경우다.

```text
while left <= right:
```

이 조건이 거짓이 되면 반복을 멈춘다.

즉, 다음과 같은 상태가 되면 찾는 값이 없다는 뜻이다.

```text
left > right
```

## 빈 리스트인 경우

리스트에 값이 하나도 없다면 탐색할 범위도 없다.

```text
binary_search([], 3)
```

처음 값은 다음과 같다.

```text
left = 0
right = -1
```

처음부터 `left <= right` 조건이 거짓이다.

따라서 반복문이 실행되지 않고 바로 `-1`을 반환한다.

```text
return -1
```

이진 탐색은 빈 리스트에서도 자연스럽게 동작한다.

## 값이 하나만 있는 경우

리스트에 값이 하나만 있어도 이진 탐색은 같은 방식으로 동작한다.

```text
binary_search([5], 5)
```

처음 상태는 다음과 같다.

```text
left = 0
right = 0
mid = 0
```

`arr[mid]`가 `5`이고 찾는 값도 `5`이므로 `0`을 반환한다.

```text
return 0
```

찾는 값이 없다면 다음처럼 된다.

```text
binary_search([5], 3)
```

`3`은 `5`보다 작으므로 `right`가 `-1`이 된다.

```text
left = 0
right = -1
```

탐색 범위가 사라졌으므로 `-1`을 반환한다.

## 정렬 조건

이진 탐색은 반드시 정렬된 데이터에서 사용해야 한다.

예를 들어 다음 리스트는 정렬되어 있지 않다.

```text
[7, 1, 11, 3, 9]
```

이런 리스트에서 가운데 값만 보고 왼쪽으로 갈지 오른쪽으로 갈지 판단하면 틀릴 수 있다.

반면 다음 리스트는 오름차순으로 정렬되어 있다.

```text
[1, 3, 7, 9, 11]
```

이 경우 가운데 값보다 찾는 값이 작으면 왼쪽에 있고, 크면 오른쪽에 있다는 판단이 가능하다.

그래서 이진 탐색을 사용할 수 있다.

## 내림차순 정렬인 경우

이진 탐색은 내림차순으로 정렬된 데이터에도 사용할 수 있다.

하지만 비교 방향이 반대가 된다.

예를 들어 다음 리스트가 있다고 하자.

```text
[13, 11, 9, 7, 5, 3, 1]
```

오름차순에서는 가운데 값보다 찾는 값이 크면 오른쪽으로 간다.

하지만 내림차순에서는 가운데 값보다 찾는 값이 크면 왼쪽으로 가야 한다.

```text
if arr[mid] < target:
    right = mid - 1
else:
    left = mid + 1
```

즉, 이진 탐색의 원리는 같지만 정렬 방향에 따라 비교 조건이 달라진다.

## 같은 값이 여러 개 있는 경우

기본적인 이진 탐색은 같은 값이 여러 개 있을 때 어떤 위치를 반환할지 보장하지 않는다.

예를 들어 다음 리스트에서 `5`를 찾는다고 하자.

```text
[1, 3, 5, 5, 5, 7, 9]
```

일반 이진 탐색은 가운데에서 발견한 `5`의 위치를 바로 반환한다.

```text
return 3
```

하지만 첫 번째 `5`의 위치를 찾고 싶다면 값을 찾은 뒤에도 왼쪽을 계속 확인해야 한다.

## 첫 번째 위치 찾기

중복된 값 중 가장 왼쪽 위치를 찾는 방법은 다음과 같다.

```text
function lower_bound(arr, target):
    left = 0
    right = length(arr) - 1
    answer = -1

    while left <= right:
        mid = (left + right) // 2

        if arr[mid] == target:
            answer = mid
            right = mid - 1
        else if arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return answer
```

값을 찾았을 때 바로 끝내지 않고 `right = mid - 1`로 왼쪽 범위를 더 탐색한다.

## 마지막 위치 찾기

중복된 값 중 가장 오른쪽 위치를 찾는 방법은 다음과 같다.

```text
function upper_bound(arr, target):
    left = 0
    right = length(arr) - 1
    answer = -1

    while left <= right:
        mid = (left + right) // 2

        if arr[mid] == target:
            answer = mid
            left = mid + 1
        else if arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return answer
```

값을 찾았을 때 바로 끝내지 않고 `left = mid + 1`로 오른쪽 범위를 더 탐색한다.

## lower bound와 upper bound

코딩 테스트나 알고리즘 문제에서는 이진 탐색을 살짝 변형해서 사용하는 경우가 많다.

대표적인 개념이 `lower bound`와 `upper bound`다.

```text
lower bound: target 이상인 값이 처음 나오는 위치
upper bound: target보다 큰 값이 처음 나오는 위치
```

예를 들어 다음 리스트가 있다고 하자.

```text
[1, 3, 5, 5, 5, 7, 9]
```

`target = 5`일 때 결과는 다음과 같다.

```text
lower bound = 2
upper bound = 5
```

`5`는 인덱스 `2`에서 처음 나오고, `5`보다 큰 값인 `7`은 인덱스 `5`에서 처음 나오기 때문이다.

이 두 값을 이용하면 특정 값이 몇 개 있는지도 구할 수 있다.

```text
개수 = upper bound - lower bound
```

위 예시에서는 다음과 같다.

```text
5 - 2 = 3
```

따라서 `5`는 총 3개 있다.

## 반복문 방식과 재귀 방식

이진 탐색은 반복문으로도 만들 수 있고 재귀로도 만들 수 있다.

반복문 방식은 보통 다음처럼 쓴다.

```text
function binary_search(arr, target):
    left = 0
    right = length(arr) - 1

    while left <= right:
        mid = (left + right) // 2

        if arr[mid] == target:
            return mid
        else if arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return -1
```

재귀 방식은 다음처럼 쓸 수 있다.

```text
function binary_search_recursive(arr, target, left, right):
    if left > right:
        return -1

    mid = (left + right) // 2

    if arr[mid] == target:
        return mid
    else if arr[mid] < target:
        return binary_search_recursive(arr, target, mid + 1, right)
    else:
        return binary_search_recursive(arr, target, left, mid - 1)
```

둘 다 같은 원리로 동작한다.

다만 반복문 방식은 추가 호출이 없어서 더 단순하게 쓰이는 경우가 많다.

## 시간 복잡도

이진 탐색은 한 번 비교할 때마다 탐색 범위를 절반으로 줄인다.

예를 들어 데이터가 16개 있다면 범위는 대략 다음처럼 줄어든다.

```text
16 -> 8 -> 4 -> 2 -> 1
```

데이터가 1개가 될 때까지 절반씩 줄어들기 때문에 시간 복잡도는 다음과 같다.

```text
O(log n)
```

찾는 값이 첫 번째 가운데 값이라면 한 번에 끝날 수도 있다.

```text
최선의 경우: O(1)
평균의 경우: O(log n)
최악의 경우: O(log n)
```

선형 탐색의 최악 시간 복잡도는 `O(n)`이지만, 이진 탐색은 `O(log n)`이다.

그래서 데이터가 많을수록 차이가 크게 난다.

## 공간 복잡도

반복문으로 구현한 이진 탐색은 추가 배열을 만들지 않는다.

`left`, `right`, `mid` 같은 변수만 사용한다.

따라서 공간 복잡도는 다음과 같다.

```text
O(1)
```

재귀 방식으로 구현하면 함수 호출이 쌓이기 때문에 호출 스택이 추가로 필요하다.

이 경우 공간 복잡도는 다음과 같다.

```text
O(log n)
```

## 장점

```text
탐색 속도가 매우 빠르다.
데이터가 많을수록 선형 탐색보다 훨씬 효율적이다.
구현이 비교적 단순하다.
정렬된 배열이나 리스트에서 사용하기 좋다.
lower bound, upper bound처럼 다양한 변형이 가능하다.
```

## 단점

```text
데이터가 반드시 정렬되어 있어야 한다.
정렬되어 있지 않다면 먼저 정렬해야 하므로 추가 비용이 든다.
연결 리스트처럼 가운데 위치에 바로 접근하기 어려운 자료구조에는 적합하지 않다.
중복 값이 있을 때 첫 번째 위치나 마지막 위치를 찾으려면 변형이 필요하다.
```

## 선형 탐색과 비교

선형 탐색은 앞에서부터 하나씩 확인한다.

```text
[1, 3, 5, 7, 9, 11, 13]
```

`13`을 찾는다면 거의 끝까지 확인해야 한다.

반면 이진 탐색은 가운데를 기준으로 범위를 절반씩 줄인다.

```text
[1, 3, 5, 7, 9, 11, 13]
       ^
     가운데
```

그래서 정렬된 데이터에서는 이진 탐색이 훨씬 빠르다.

하지만 정렬되어 있지 않은 데이터라면 선형 탐색은 바로 사용할 수 있고, 이진 탐색은 바로 사용할 수 없다.

```text
선형 탐색: 정렬이 없어도 가능
이진 탐색: 정렬이 반드시 필요
```

## 언제 사용하면 좋은가

이진 탐색은 다음과 같은 상황에서 사용하기 좋다.

```text
데이터가 이미 정렬되어 있을 때
많은 데이터 중에서 특정 값을 빠르게 찾고 싶을 때
특정 값 이상 또는 이하가 처음 나오는 위치를 찾고 싶을 때
가능한 답의 범위가 정해져 있고, 조건을 만족하는 최소값이나 최대값을 찾고 싶을 때
```

마지막 경우처럼 실제 배열이 아니라 정답의 범위를 대상으로 이진 탐색을 하는 경우도 있다.

이런 방식을 흔히 답을 이진 탐색한다고 표현한다.

## 답을 이진 탐색하는 경우

이진 탐색은 꼭 리스트에서 값만 찾는 데 사용되는 것은 아니다.

예를 들어 다음과 같은 문제가 있다고 하자.

```text
어떤 조건을 만족하는 가장 작은 숫자를 찾아라.
```

이때 가능한 숫자의 범위가 정해져 있고, 어떤 숫자가 조건을 만족하는지 빠르게 검사할 수 있다면 이진 탐색을 사용할 수 있다.

예를 들어 `1`부터 `100` 사이에서 조건을 만족하는 가장 작은 값을 찾는다면 다음처럼 생각할 수 있다.

```text
left = 1
right = 100
```

가운데 값을 검사하고, 조건을 만족하면 더 작은 값이 가능한지 왼쪽을 탐색한다.

조건을 만족하지 못하면 더 큰 값이 필요하므로 오른쪽을 탐색한다.

이런 유형은 코딩 테스트에서 자주 등장한다.

## 주의할 점

가운데 위치를 구할 때 보통 다음처럼 쓴다.

```text
mid = (left + right) // 2
```

하지만 일부 언어에서는 `left + right`가 너무 커지면 정수 범위를 넘을 수 있다.

그래서 더 안전하게 다음처럼 쓰기도 한다.

```text
mid = left + (right - left) // 2
```

파이썬처럼 정수 크기에 사실상 제한이 없는 언어에서는 큰 문제가 되지 않지만, C나 Java 같은 언어에서는 알아두면 좋다.

또한 반복문에서 `left`와 `right`를 제대로 갱신하지 않으면 무한 반복이 생길 수 있다.

값을 찾지 못했을 때는 반드시 탐색 범위가 줄어들도록 해야 한다.

```text
left = mid + 1
right = mid - 1
```

## 한 문장 요약

이진 탐색은 정렬된 데이터에서 가운데 값을 기준으로 탐색 범위를 절반씩 줄여 가며 원하는 값을 빠르게 찾는 탐색 알고리즘이다.

## 알고리즘 흐름도

아래 흐름은 이진 탐색을 언제 어디서든 다시 만들 수 있게 정리한 것이다.

```text
시작
  |
  v
정렬된 리스트와 찾는 값을 받는다
  |
  v
left = 0, right = 마지막 인덱스로 설정한다
  |
  v
left <= right 인가?
  |
  +-- 아니오 --> 찾지 못했다는 값을 반환한다
  |
  +-- 예
        |
        v
      mid = (left + right) // 2 를 구한다
        |
        v
      arr[mid]가 찾는 값과 같은가?
        |
        +-- 예 --> mid를 반환한다
        |
        +-- 아니오
              |
              v
            arr[mid]가 찾는 값보다 작은가?
              |
              +-- 예 --> left = mid + 1
              |
              +-- 아니오 --> right = mid - 1
              |
              v
            다시 반복 조건을 확인한다
```

## 머릿속 결정 순서

이진 탐색을 코드로 만들 때는 이 질문 순서만 기억하면 된다.

```text
데이터가 정렬되어 있는가?
  -> 아니면 이진 탐색을 바로 사용할 수 없다

어디부터 어디까지 탐색할 것인가?
  -> left와 right 정하기

가운데는 어디인가?
  -> mid 구하기

가운데 값이 찾는 값인가?
  -> 같으면 mid 반환

가운데 값이 찾는 값보다 작은가?
  -> 오름차순이라면 오른쪽으로 이동

가운데 값이 찾는 값보다 큰가?
  -> 오름차순이라면 왼쪽으로 이동

탐색 범위가 남아 있는가?
  -> left <= right 동안 반복

끝까지 없으면 어떻게 할 것인가?
  -> -1 같은 실패 값을 반환
```
