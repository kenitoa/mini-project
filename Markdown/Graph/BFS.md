# BFS

BFS는 Breadth First Search의 줄임말이고, 한국어로는 너비 우선 탐색이라고 부른다.

BFS는 그래프나 트리에서 시작 정점과 가까운 정점부터 차례대로 방문하는 탐색 방법이다.

한 방향으로 깊게 들어가는 DFS와 달리, BFS는 현재 위치에서 갈 수 있는 곳들을 먼저 모두 확인한 뒤 다음 단계로 넘어간다.

즉, 가까운 곳부터 넓게 퍼져 나가면서 탐색하는 방식이다.

BFS는 보통 다음 자료 구조를 사용해 구현한다.

```text
큐
```

큐는 먼저 들어온 값이 먼저 나가는 자료 구조이다.

```text
enqueue: 큐에 넣기
dequeue: 큐에서 꺼내기
```

먼저 발견한 정점부터 차례대로 처리해야 하므로 BFS에서는 큐가 매우 중요하다.

## 핵심 생각

예를 들어 다음과 같은 그래프가 있다고 하자.

```text
A -- B -- D
|    |
C    E
|
F
```

이 그래프를 연결 관계로 나타내면 다음과 같다.

```text
A: B, C
B: A, D, E
C: A, F
D: B
E: B
F: C
```

`A`에서 시작해서 BFS를 한다면 `A`와 가까운 정점부터 방문한다.

예를 들어 `A`의 이웃을 `B`, `C` 순서로 확인한다고 하자.

```text
A에서 시작
A와 가까운 B, C를 확인
B에서 갈 수 있는 D, E를 확인
C에서 갈 수 있는 F를 확인
```

방문 순서는 다음과 같을 수 있다.

```text
A -> B -> C -> D -> E -> F
```

즉, BFS는 시작 정점에서 거리가 `1`인 정점들을 먼저 보고, 그 다음 거리가 `2`인 정점들을 보는 식으로 탐색한다.

## 그래프에서 사용하는 말

BFS를 이해하려면 그래프에서 자주 사용하는 말을 알아두면 좋다.

```text
정점 또는 노드: 데이터가 있는 지점
간선: 정점과 정점을 연결하는 선
인접 정점: 현재 정점에서 바로 갈 수 있는 정점
방문: 어떤 정점에 도착해서 확인했다는 뜻
방문 체크: 이미 방문한 정점을 다시 방문하지 않도록 표시하는 것
거리: 시작 정점에서 어떤 정점까지 가는 데 필요한 간선의 수
```

예를 들어 다음 그래프에서:

```text
A -- B
|
C
```

`A`의 인접 정점은 `B`와 `C`이다.

`A`에서 `B`까지의 거리는 `1`이고, `A`에서 `C`까지의 거리도 `1`이다.

BFS는 이런 가까운 정점들을 먼저 방문한다.

## 왜 방문 체크가 필요한가

그래프에는 사이클이 있을 수 있다.

사이클은 다시 자기 자신 쪽으로 돌아올 수 있는 길이다.

예를 들어:

```text
A -- B
|    |
C -- D
```

`A`에서 `B`로 가고, `B`에서 `D`로 가고, `D`에서 `C`로 가고, `C`에서 다시 `A`로 갈 수 있다.

방문 체크를 하지 않으면 같은 정점을 큐에 계속 넣을 수 있다.

```text
A -> B -> C -> D -> A -> B -> C -> D -> ...
```

그래서 BFS에서는 보통 `visited`라는 집합이나 배열을 사용한다.

```text
visited = 이미 방문한 정점들
```

어떤 정점을 발견하면 `visited`에 기록한다.

그리고 다음 정점을 큐에 넣기 전에 이미 방문했는지 확인한다.

```text
이미 방문했다면 큐에 넣지 않는다.
방문하지 않았다면 방문 표시를 하고 큐에 넣는다.
```

## 큐를 사용하는 이유

BFS는 먼저 발견한 정점부터 먼저 처리해야 한다.

그래야 가까운 정점부터 차례대로 탐색할 수 있다.

큐는 먼저 들어온 값이 먼저 나오는 자료 구조이다.

```text
먼저 들어온 정점 -> 먼저 처리
나중에 들어온 정점 -> 나중에 처리
```

예를 들어 `A`에서 `B`, `C`를 발견했다면 먼저 `B`를 처리하고 그 다음 `C`를 처리한다.

```text
queue = [B, C]
```

`B`를 꺼내서 처리한다.

```text
dequeue -> B
queue = [C]
```

`B`에서 새로 발견한 `D`, `E`를 뒤에 넣는다.

```text
queue = [C, D, E]
```

이렇게 하면 `C`가 `D`, `E`보다 먼저 처리된다.

그래서 `A`와 더 가까운 정점들이 먼저 방문된다.

## BFS 의사코드

가장 기본적인 BFS는 시작 정점에서 갈 수 있는 모든 정점을 방문한다.

```text
bfs(그래프, 시작 정점):
    큐를 만든다
    시작 정점을 방문했다고 표시한다
    시작 정점을 큐에 넣는다

    큐가 비어 있지 않은 동안 반복한다:
        큐에서 정점을 하나 꺼낸다
        꺼낸 정점을 처리한다

        꺼낸 정점과 연결된 다음 정점들을 하나씩 확인한다:
            다음 정점을 아직 방문하지 않았다면:
                다음 정점을 방문했다고 표시한다
                다음 정점을 큐에 넣는다
```

여기서 정점을 처리한다는 말은 문제에 따라 달라진다.

```text
정점 이름 출력하기
정점 개수 세기
목표 정점인지 확인하기
거리 기록하기
```

## 조금 더 코드처럼 쓴 의사코드

```text
function bfs(graph, start):
    queue = new Queue()
    visited = new Set()

    add start to visited
    enqueue start to queue

    while queue is not empty:
        current = dequeue from queue
        print current

        for each next in graph[current]:
            if next is not in visited:
                add next to visited
                enqueue next to queue
```

이 코드는 `start`에서 시작해서 연결된 모든 정점을 너비 우선으로 방문한다.

`visited`는 이미 발견한 정점을 저장한다.

## 목표 값을 찾는 BFS 의사코드

BFS는 특정 목표 정점을 찾는 데에도 사용할 수 있다.

예를 들어 `target`을 찾는 BFS는 다음처럼 쓸 수 있다.

```text
function bfs_search(graph, start, target):
    queue = new Queue()
    visited = new Set()

    add start to visited
    enqueue start to queue

    while queue is not empty:
        current = dequeue from queue

        if current == target:
            return current

        for each next in graph[current]:
            if next is not in visited:
                add next to visited
                enqueue next to queue

    return null
```

목표 값을 찾으면 바로 반환할 수 있다.

끝까지 찾지 못하면 `null`을 반환한다.

## 흐름을 쉽게 떠올려보기

BFS는 이렇게 생각하면 쉽다.

```text
1. 시작 정점을 큐에 넣는다.
2. 큐에서 하나 꺼낸다.
3. 꺼낸 정점의 이웃들을 확인한다.
4. 아직 방문하지 않은 이웃을 큐 뒤에 넣는다.
5. 큐가 빌 때까지 반복한다.
```

즉, 먼저 발견한 정점부터 차례대로 처리하는 방식이다.

## 예시

```text
graph = {
    A: [B, C],
    B: [A, D, E],
    C: [A, F],
    D: [B],
    E: [B],
    F: [C]
}
```

`A`에서 BFS를 시작한다.

처음 상태:

```text
queue = [A]
visited = {A}
```

`A`를 꺼낸다.

```text
처리: A
이웃: B, C
queue = [B, C]
visited = {A, B, C}
```

`B`를 꺼낸다.

```text
처리: B
이웃: A, D, E
A는 이미 방문
queue = [C, D, E]
visited = {A, B, C, D, E}
```

`C`를 꺼낸다.

```text
처리: C
이웃: A, F
A는 이미 방문
queue = [D, E, F]
visited = {A, B, C, D, E, F}
```

결과 방문 순서는 다음과 같다.

```text
A -> B -> C -> D -> E -> F
```

## 목표 정점을 찾는 예시

`A`에서 시작해서 `E`를 찾는다고 하자.

```text
A -> B -> C -> D -> E
```

BFS는 `E`를 꺼내는 순간 목표를 찾았다고 판단하고 종료할 수 있다.

```text
current == E
return E
```

## 찾는 정점이 없는 예시

그래프에 `Z`가 없거나, 시작 정점에서 `Z`까지 갈 수 없다고 하자.

큐가 빌 때까지 탐색해도 `Z`를 만나지 못한다.

```text
queue = []
```

이 경우 탐색이 끝났으므로 찾지 못했다는 값을 반환한다.

```text
return null
```

## 종료 조건

BFS의 기본 종료 조건은 큐가 비는 것이다.

```text
while queue is not empty
```

큐가 비었다는 것은 더 이상 방문할 수 있는 정점이 없다는 뜻이다.

목표 정점을 찾는 문제라면 목표를 찾는 순간 바로 종료할 수도 있다.

```text
if current == target:
    return current
```

## 빈 그래프인 경우

그래프가 비어 있다면 시작 정점도 없다.

```text
graph = {}
```

이 경우 BFS를 실행하기 전에 시작 정점이 그래프에 있는지 확인해야 한다.

```text
if start is not in graph:
    return
```

## 시작 정점만 있는 경우

정점이 하나만 있고 간선이 없는 그래프도 있다.

```text
A
```

연결 관계는 다음과 같다.

```text
A: []
```

이 경우 BFS 결과는 시작 정점 하나뿐이다.

```text
A
```

## 연결되지 않은 그래프

그래프가 여러 덩어리로 나뉘어 있을 수 있다.

```text
A -- B

C -- D
```

`A`에서 BFS를 시작하면 `A`, `B`만 방문한다.

```text
A -> B
```

`C`, `D`는 `A`에서 갈 수 없으므로 방문하지 않는다.

그래프 전체를 모두 탐색하려면 모든 정점을 하나씩 확인하면서 아직 방문하지 않은 정점에서 BFS를 새로 시작해야 한다.

```text
for each vertex in graph:
    if vertex is not in visited:
        bfs(graph, vertex)
```

## 큐로 구현하는 BFS

BFS는 큐가 핵심이다.

스택을 쓰면 DFS처럼 깊게 들어가고, 큐를 쓰면 BFS처럼 넓게 퍼진다.

```text
큐: 먼저 들어온 정점부터 처리
스택: 나중에 들어온 정점부터 처리
```

그래서 BFS를 구현할 때는 큐의 앞에서 꺼내고 뒤에 넣는 흐름을 유지해야 한다.

## 방문 표시를 언제 해야 하는가

BFS에서는 보통 정점을 큐에 넣을 때 방문 표시를 한다.

```text
방문 표시
큐에 넣기
```

꺼낼 때 방문 표시를 하면 같은 정점이 큐에 여러 번 들어갈 수 있다.

예를 들어 `A`와 `B`가 모두 `C`로 연결되어 있다면, `C`가 중복으로 들어갈 수 있다.

```text
queue = [C, C]
```

그래서 발견하는 순간 방문 표시를 해 중복 삽입을 막는다.

## 거리 계산하는 BFS

가중치가 없는 그래프에서 BFS는 최단 거리를 구할 수 있다.

```text
distance[start] = 0
```

새 정점을 발견하면 현재 정점의 거리보다 `1` 큰 값을 넣는다.

```text
distance[next] = distance[current] + 1
```

의사코드는 다음과 같다.

```text
function bfs_distance(graph, start):
    queue = new Queue()
    distance = map with default -1

    distance[start] = 0
    enqueue start to queue

    while queue is not empty:
        current = dequeue from queue

        for each next in graph[current]:
            if distance[next] == -1:
                distance[next] = distance[current] + 1
                enqueue next to queue

    return distance
```

`-1`은 아직 방문하지 않았다는 뜻으로 사용할 수 있다.

## 최단 경로를 저장하는 BFS

최단 거리뿐 아니라 실제 경로를 알고 싶다면 이전 정점을 저장한다.

```text
prev[next] = current
```

예를 들어 `A`에서 `F`까지의 경로를 찾을 때:

```text
prev[B] = A
prev[C] = A
prev[F] = C
```

그러면 `F`에서 거꾸로 따라가면 된다.

```text
F -> C -> A
```

뒤집으면 실제 경로가 된다.

```text
A -> C -> F
```

## 레벨 단위 탐색

BFS는 거리가 같은 정점들을 같은 레벨로 볼 수 있다.

```text
level 0: A
level 1: B, C
level 2: D, E, F
```

그래서 트리의 높이나 그래프의 단계별 확산을 구할 때 자주 사용한다.

## 트리에서의 BFS

트리에서 BFS는 보통 레벨 순회라고 부른다.

```text
        A
      /   \
     B     C
    / \     \
   D   E     F
```

레벨 순회 결과는 다음과 같다.

```text
A -> B -> C -> D -> E -> F
```

## 인접 리스트와 인접 행렬

그래프를 저장하는 방법에 따라 BFS의 구현이 조금 달라진다.

인접 리스트는 각 정점마다 연결된 정점 목록을 저장한다.

```text
A: B, C
B: A, D
```

인접 행렬은 정점 사이의 연결 여부를 표처럼 저장한다.

```text
    A B C
A   0 1 1
B   1 0 0
C   1 0 0
```

보통 간선이 많지 않은 그래프에서는 인접 리스트가 더 효율적이다.

## 시간 복잡도

정점 수를 `V`, 간선 수를 `E`라고 하자.

인접 리스트를 사용하면 BFS는 각 정점과 각 간선을 한 번씩 확인한다.

```text
시간 복잡도: O(V + E)
```

인접 행렬을 사용하면 각 정점마다 모든 정점을 확인해야 할 수 있다.

```text
시간 복잡도: O(V^2)
```

## 공간 복잡도

BFS는 방문 기록과 큐가 필요하다.

```text
visited: O(V)
queue: O(V)
```

그래서 기본 공간 복잡도는 다음과 같다.

```text
공간 복잡도: O(V)
```

그래프 자체를 저장하는 공간은 인접 리스트라면 `O(V + E)`이다.

## 장점

```text
가까운 정점부터 차례대로 탐색한다.
가중치가 없는 그래프에서 최단 거리를 구할 수 있다.
재귀를 쓰지 않아 호출 깊이 문제가 적다.
레벨 단위 탐색에 적합하다.
```

## 단점

```text
큐에 많은 정점이 들어가면 메모리를 많이 쓸 수 있다.
가중치가 있는 최단 경로에는 그대로 사용할 수 없다.
깊은 경로를 빠르게 파고드는 데에는 DFS보다 어울리지 않을 수 있다.
```

## DFS와 비교

```text
BFS:
    가까운 정점부터 방문한다.
    큐를 사용한다.
    가중치 없는 최단 거리 문제에 좋다.

DFS:
    한 방향으로 깊게 들어간다.
    재귀 또는 스택을 사용한다.
    경로 탐색, 백트래킹, 연결 요소 탐색에 자주 쓴다.
```

둘 다 그래프 탐색의 기본이지만, 방문 순서와 잘 맞는 문제 유형이 다르다.

## 언제 사용하면 좋은가

BFS는 다음 상황에서 좋다.

```text
가중치가 없는 그래프에서 최단 거리를 구할 때
시작점에서 가까운 순서대로 확인해야 할 때
단계별 확산을 표현할 때
트리의 레벨 순회를 할 때
```

## 주의할 점

방문 표시를 늦게 하면 같은 정점이 큐에 여러 번 들어갈 수 있다.

그래서 보통 큐에 넣는 순간 방문 표시를 한다.

또한 그래프가 연결되어 있지 않다면 시작 정점에서 갈 수 있는 부분만 탐색된다는 점을 기억해야 한다.

## 한 문장 요약

BFS는 큐를 사용해 시작 정점에서 가까운 곳부터 넓게 퍼져 나가는 탐색 알고리즘이다.

## 알고리즘 흐름도

```text
시작
  |
  v
시작 정점 방문 표시
  |
  v
시작 정점을 큐에 넣기
  |
  v
큐가 비어 있는가?
  | no
  v
큐에서 정점 꺼내기
  |
  v
정점 처리
  |
  v
인접 정점 확인
  |
  v
방문하지 않은 정점인가?
  | yes
  v
방문 표시 후 큐에 넣기
  |
  v
큐 확인으로 돌아가기
  |
  yes
  v
종료
```

## 더 짧게 외우는 흐름

```text
시작점을 큐에 넣는다
큐에서 하나 꺼낸다
이웃을 큐에 넣는다
큐가 빌 때까지 반복한다
```

## 머릿속 결정 순서

```text
1. 시작 정점이 있는가?
2. 방문 표시를 했는가?
3. 큐에 넣었는가?
4. 큐에서 꺼낸 정점을 처리했는가?
5. 이웃 중 방문하지 않은 정점만 큐에 넣었는가?
6. 큐가 빌 때까지 반복했는가?
```
