# 크루스칼 알고리즘

크루스칼 알고리즘은 최소 신장 트리를 만드는 알고리즘이다.

최소 신장 트리는 그래프의 모든 정점을 연결하면서, 사용한 간선의 가중치 합이 가장 작아지도록 만든 트리이다.

```text
신장 트리: 모든 정점을 연결하는 트리
최소 신장 트리: 간선 비용의 합이 가장 작은 신장 트리
```

크루스칼은 간선을 가중치가 작은 순서대로 정렬한 뒤, 사이클이 생기지 않는 간선만 선택한다.

## 핵심 생각

예를 들어 다음과 같은 무방향 그래프가 있다고 하자.

```text
A --1-- B
|       |
4       2
|       |
C --3-- D
```

모든 간선을 가중치 순서대로 정렬한다.

```text
A-B (1)
B-D (2)
C-D (3)
A-C (4)
```

이제 가장 작은 간선부터 확인한다.

```text
A-B 선택
B-D 선택
C-D 선택
```

이 세 간선을 선택하면 모든 정점 `A, B, C, D`가 연결된다.

간선 수는 정점 수보다 하나 적은 `3`개이다.

```text
선택된 간선:
A-B (1)
B-D (2)
C-D (3)

총 비용 = 6
```

`A-C (4)`는 이미 모든 정점이 연결된 뒤라 필요 없다.

## 그래프에서 사용하는 말

크루스칼을 이해하려면 다음 말을 알아두면 좋다.

```text
정점: 그래프의 지점
간선: 정점과 정점을 연결하는 길
가중치: 간선을 선택했을 때 드는 비용
트리: 사이클이 없는 연결 그래프
신장 트리: 모든 정점을 포함하는 트리
최소 신장 트리: 전체 비용이 가장 작은 신장 트리
사이클: 한 바퀴 돌아서 자기 자신으로 돌아올 수 있는 길
```

최소 신장 트리는 무방향 연결 그래프에서 주로 다룬다.

## 사이클을 피해야 하는 이유

신장 트리는 트리이므로 사이클이 없어야 한다.

예를 들어 다음처럼 세 정점이 모두 연결되면 사이클이 생긴다.

```text
A -- B
|    |
C ---
```

사이클이 있으면 어떤 간선 하나를 제거해도 여전히 정점들이 연결될 수 있다.

그러므로 최소 비용으로 연결하려면 사이클을 만들 필요가 없다.

크루스칼은 간선을 추가할 때마다 사이클이 생기는지 확인한다.

## 유니온 파인드

크루스칼은 사이클을 빠르게 확인하기 위해 유니온 파인드 자료 구조를 자주 사용한다.

유니온 파인드는 정점들이 같은 집합에 속하는지 확인하고, 서로 다른 집합을 합치는 자료 구조이다.

```text
find(x): x가 속한 집합의 대표를 찾는다
union(a, b): a와 b가 속한 집합을 합친다
```

간선 `A-B`를 보았을 때:

```text
find(A) == find(B) 이면:
    이미 같은 집합이므로 연결하면 사이클이 생긴다.

find(A) != find(B) 이면:
    서로 다른 집합이므로 간선을 선택하고 union(A, B)를 한다.
```

## 크루스칼 의사코드

```text
kruskal(정점 목록, 간선 목록):
    간선을 가중치가 작은 순서대로 정렬한다
    각 정점을 자기 자신만 있는 집합으로 만든다
    결과 간선 목록을 비워 둔다

    정렬된 간선을 하나씩 확인한다:
        간선의 두 정점이 서로 다른 집합에 있다면:
            이 간선을 결과에 추가한다
            두 정점의 집합을 합친다

        결과 간선 수가 정점 수 - 1개가 되면:
            종료한다

    결과 간선 목록을 반환한다
```

핵심은 가장 싼 간선부터 보되, 사이클을 만들지 않는 간선만 선택하는 것이다.

## 조금 더 코드처럼 쓴 의사코드

```text
function kruskal(vertices, edges):
    sort edges by weight ascending
    parent = make_set(vertices)
    mst = []
    total_cost = 0

    for each edge in edges:
        u = edge.from
        v = edge.to
        w = edge.weight

        if find(parent, u) != find(parent, v):
            union(parent, u, v)
            add edge to mst
            total_cost = total_cost + w

        if length(mst) == length(vertices) - 1:
            break

    return mst, total_cost
```

`mst`에는 최소 신장 트리에 포함되는 간선들이 들어간다.

## 흐름을 쉽게 풀어보기

크루스칼은 이렇게 생각하면 쉽다.

```text
1. 모든 간선을 비용이 작은 순서로 정렬한다.
2. 가장 싼 간선부터 하나씩 본다.
3. 그 간선을 넣었을 때 사이클이 생기면 버린다.
4. 사이클이 생기지 않으면 선택한다.
5. 선택한 간선이 정점 수 - 1개가 되면 끝난다.
```

즉, 전체 그래프를 한 번에 키우는 것이 아니라, 싼 간선을 골라 서로 다른 덩어리를 합쳐 나간다.

## 예시

다음 간선들이 있다고 하자.

```text
A-B (1)
A-C (4)
B-C (2)
B-D (5)
C-D (3)
```

가중치 순서대로 정렬한다.

```text
A-B (1)
B-C (2)
C-D (3)
A-C (4)
B-D (5)
```

처음에는 모든 정점이 따로 있다.

```text
{A} {B} {C} {D}
```

`A-B (1)`을 선택한다.

```text
{A, B} {C} {D}
```

`B-C (2)`를 선택한다.

```text
{A, B, C} {D}
```

`C-D (3)`을 선택한다.

```text
{A, B, C, D}
```

모든 정점이 연결되었고 선택한 간선은 `3`개이다.

정점이 `4`개이므로 최소 신장 트리는 간선 `3`개를 가진다.

```text
총 비용 = 1 + 2 + 3 = 6
```

## 사이클이 생기는 예시

앞의 예시에서 `A-C (4)`를 추가로 선택하려고 하면 어떻게 될까?

이미 `A`와 `C`는 `A-B-C`로 연결되어 있다.

```text
A -- B -- C
```

여기에 `A-C`를 추가하면 사이클이 생긴다.

```text
A -- B
|    |
C ---
```

따라서 `A-C (4)`는 선택하지 않는다.

## 종료 조건

최소 신장 트리는 정점이 `V`개일 때 간선이 `V - 1`개이다.

그래서 크루스칼은 다음 조건에서 종료할 수 있다.

```text
선택한 간선 수 == 정점 수 - 1
```

간선을 모두 확인했는데도 이 조건을 만족하지 못했다면 그래프가 연결되어 있지 않을 수 있다.

이 경우 최소 신장 트리가 아니라 최소 신장 숲이 만들어진다.

## 연결되지 않은 그래프

그래프가 연결되어 있지 않으면 모든 정점을 하나의 트리로 연결할 수 없다.

```text
A -- B

C -- D
```

이 경우 크루스칼을 실행하면 각 연결 요소마다 최소 신장 트리가 만들어진다.

이것을 최소 신장 숲이라고 볼 수 있다.

```text
A-B
C-D
```

모든 정점을 하나로 연결해야 하는 문제라면 연결 여부를 따로 확인해야 한다.

## 프림과 비교

크루스칼과 프림은 둘 다 최소 신장 트리를 구한다.

```text
크루스칼:
    간선을 비용 순서로 정렬한다.
    사이클이 생기지 않는 간선을 선택한다.
    유니온 파인드를 자주 사용한다.

프림:
    하나의 정점에서 시작한다.
    현재 트리와 연결되는 가장 싼 간선을 선택한다.
    우선순위 큐를 자주 사용한다.
```

간선 목록을 정렬하기 쉬우면 크루스칼이 직관적이다.

정점 주변 간선을 확장하는 방식이 편하면 프림이 좋다.

## 시간 복잡도

정점 수를 `V`, 간선 수를 `E`라고 하자.

크루스칼에서 가장 큰 비용은 간선을 정렬하는 부분이다.

```text
간선 정렬: O(E log E)
```

유니온 파인드 연산은 거의 상수 시간에 가깝게 처리할 수 있다.

그래서 전체 시간 복잡도는 보통 다음과 같이 본다.

```text
시간 복잡도: O(E log E)
```

`E`는 최대 `V^2`에 가까울 수 있으므로 `O(E log V)`라고 표현하기도 한다.

## 공간 복잡도

유니온 파인드를 위한 배열이 필요하다.

```text
parent: O(V)
rank 또는 size: O(V)
```

결과 간선 목록은 최대 `V - 1`개이다.

```text
mst: O(V)
```

간선 목록 저장은 입력에 따라 `O(E)`이다.

## 장점

```text
아이디어가 직관적이다.
간선 목록 중심으로 구현하기 쉽다.
유니온 파인드와 함께 쓰면 효율적이다.
희소 그래프에서 잘 동작한다.
```

## 단점

```text
간선을 정렬해야 한다.
유니온 파인드 구현이 필요하다.
연결되지 않은 그래프에서는 하나의 신장 트리가 나오지 않는다.
```

## 언제 사용하면 좋은가

크루스칼은 다음 상황에서 좋다.

```text
최소 비용으로 모든 정점을 연결해야 할 때
간선 목록이 주어질 때
간선 수가 비교적 적은 희소 그래프일 때
유니온 파인드를 연습하거나 활용해야 할 때
```

## 주의할 점

크루스칼은 무방향 그래프의 최소 신장 트리를 구하는 알고리즘이다.

방향 그래프의 최단 경로나 최소 비용 경로 문제와 헷갈리면 안 된다.

또한 간선을 선택할 때는 반드시 사이클 여부를 확인해야 한다.

```text
같은 집합이면 선택하지 않는다.
다른 집합이면 선택한다.
```

## 한 문장 요약

크루스칼은 가장 싼 간선부터 보면서 사이클이 생기지 않는 간선만 선택해 최소 신장 트리를 만드는 알고리즘이다.

## 알고리즘 흐름도

```text
시작
  |
  v
간선을 가중치 순서로 정렬
  |
  v
각 정점을 독립 집합으로 만들기
  |
  v
가장 싼 간선부터 확인
  |
  v
두 정점이 같은 집합인가?
  | yes
  v
간선 버리기
  |
  no
  v
간선 선택
  |
  v
두 집합 합치기
  |
  v
선택 간선 수가 V - 1인가?
  | no
  v
다음 간선 확인
  |
  yes
  v
최소 신장 트리 완성
```

## 더 짧게 외우는 흐름

```text
간선 정렬
싼 간선부터 확인
사이클 없으면 선택
V - 1개 선택하면 종료
```

## 머릿속 결정 순서

```text
1. 무방향 그래프인가?
2. 간선을 가중치 순서로 정렬했는가?
3. 두 정점이 같은 집합인지 확인했는가?
4. 사이클이 없을 때만 선택했는가?
5. 선택할 때 union을 했는가?
6. 간선 V - 1개를 선택했는가?
```
