# DFS

DFS는 Depth First Search의 줄임말이고, 한국어로는 깊이 우선 탐색이라고 부른다.

DFS는 그래프나 트리에서 한 방향으로 갈 수 있는 만큼 깊게 들어간 뒤, 더 이상 갈 곳이 없으면 이전 위치로 되돌아와서 다른 길을 탐색하는 방법이다.

즉, 가까운 곳을 먼저 넓게 살피는 방식이 아니라, 한 길을 끝까지 따라가 보는 방식이다.

DFS는 보통 다음 두 가지 방식으로 구현한다.

```text
재귀 함수
스택
```

재귀 함수를 사용하면 함수 호출이 자연스럽게 스택처럼 동작한다.

직접 스택 자료 구조를 만들어서 DFS를 구현할 수도 있다.

## 핵심 생각

예를 들어 다음과 같은 그래프가 있다고 하자.

```text
A -- B -- D
|    |
C    E
|
F
```

이 그래프를 연결 관계로 나타내면 다음과 같다.

```text
A: B, C
B: A, D, E
C: A, F
D: B
E: B
F: C
```

`A`에서 시작해서 DFS를 한다면 먼저 한 방향으로 깊게 들어간다.

예를 들어 `A`의 이웃을 `B`, `C` 순서로 확인한다고 하자.

```text
A에서 시작
A -> B로 이동
B -> D로 이동
D에서 더 갈 곳이 없음
B로 되돌아감
B -> E로 이동
E에서 더 갈 곳이 없음
B로 되돌아감
A로 되돌아감
A -> C로 이동
C -> F로 이동
```

방문 순서는 다음과 같을 수 있다.

```text
A -> B -> D -> E -> C -> F
```

즉, DFS는 현재 위치에서 갈 수 있는 다음 위치로 계속 들어가다가, 막히면 되돌아오는 방식이다.

## 그래프에서 사용하는 말

DFS를 이해하려면 그래프에서 자주 사용하는 말을 알아두면 좋다.

```text
정점 또는 노드: 데이터가 있는 지점
간선: 정점과 정점을 연결하는 선
인접 정점: 현재 정점에서 바로 갈 수 있는 정점
방문: 어떤 정점에 도착해서 확인했다는 뜻
방문 체크: 이미 방문한 정점을 다시 방문하지 않도록 표시하는 것
```

예를 들어 다음 그래프에서:

```text
A -- B
|
C
```

`A`의 인접 정점은 `B`와 `C`이다.

DFS는 현재 정점의 인접 정점을 하나씩 확인하면서 깊게 이동한다.

## 왜 방문 체크가 필요한가

그래프에는 사이클이 있을 수 있다.

사이클은 다시 자기 자신 쪽으로 돌아올 수 있는 길이다.

예를 들어:

```text
A -- B
|    |
C -- D
```

`A`에서 `B`로 가고, `B`에서 `D`로 가고, `D`에서 `C`로 가고, `C`에서 다시 `A`로 갈 수 있다.

방문 체크를 하지 않으면 다음처럼 같은 정점을 계속 반복해서 방문할 수 있다.

```text
A -> B -> D -> C -> A -> B -> D -> C -> ...
```

그래서 DFS에서는 보통 `visited`라는 집합이나 배열을 사용한다.

```text
visited = 이미 방문한 정점들
```

어떤 정점을 방문하면 `visited`에 기록한다.

그리고 다음 정점으로 이동하기 전에 이미 방문했는지 확인한다.

```text
이미 방문했다면 다시 가지 않는다.
방문하지 않았다면 이동한다.
```

## DFS 의사코드

가장 기본적인 DFS는 현재 정점에서 시작해 갈 수 있는 정점을 모두 방문한다.

```text
dfs(그래프, 현재 정점, 방문 기록):
    현재 정점을 방문했다고 표시한다
    현재 정점을 처리한다

    현재 정점과 연결된 다음 정점들을 하나씩 확인한다:
        다음 정점을 아직 방문하지 않았다면:
            dfs(그래프, 다음 정점, 방문 기록)을 실행한다
```

여기서 현재 정점을 처리한다는 말은 문제에 따라 달라진다.

```text
정점 이름 출력하기
정점 개수 세기
목표 정점인지 확인하기
경로에 추가하기
```

## 조금 더 코드처럼 쓴 의사코드

```text
function dfs(graph, current, visited):
    add current to visited
    print current

    for each next in graph[current]:
        if next is not in visited:
            dfs(graph, next, visited)
```

이 코드는 `current`에서 시작해서 연결된 모든 정점을 깊이 우선으로 방문한다.

`visited`는 이미 방문한 정점을 저장한다.

## 목표 값을 찾는 DFS 의사코드

DFS는 단순히 모든 정점을 방문할 수도 있고, 특정 목표 정점을 찾는 데 사용할 수도 있다.

예를 들어 `target`을 찾는 DFS는 다음처럼 쓸 수 있다.

```text
function dfs_search(graph, current, target, visited):
    add current to visited

    if current == target:
        return current

    for each next in graph[current]:
        if next is not in visited:
            result = dfs_search(graph, next, target, visited)

            if result is not null:
                return result

    return null
```

목표 값을 찾으면 그 값을 반환하고, 끝까지 찾지 못하면 `null`을 반환한다.

## 흐름을 쉽게 떠올려보기

DFS는 이렇게 생각하면 쉽다.

```text
1. 현재 정점을 방문한다.
2. 갈 수 있는 다음 정점을 찾는다.
3. 아직 방문하지 않았다면 그 정점으로 들어간다.
4. 더 이상 갈 곳이 없으면 이전 정점으로 돌아온다.
5. 모든 길을 확인할 때까지 반복한다.
```

즉, 하나의 길을 깊게 파고든 뒤 다시 돌아와 다른 길을 확인한다.

## 예시

```text
graph = {
    A: [B, C],
    B: [A, D, E],
    C: [A, F],
    D: [B],
    E: [B],
    F: [C]
}
```

`A`에서 DFS를 시작한다.

```text
dfs(A)
```

방문 과정은 다음과 같을 수 있다.

```text
A 방문
B 방문
D 방문
D에서 더 갈 곳 없음
B로 돌아감
E 방문
E에서 더 갈 곳 없음
B로 돌아감
A로 돌아감
C 방문
F 방문
```

결과 방문 순서는 다음과 같다.

```text
A -> B -> D -> E -> C -> F
```

## 목표 정점을 찾는 예시

`A`에서 시작해서 `E`를 찾는다고 하자.

```text
A -> B -> D
```

`D`에서 `E`를 찾지 못하면 `B`로 돌아온다.

그 다음 `E`로 이동한다.

```text
B -> E
```

이때 목표 정점인 `E`를 찾았으므로 탐색을 종료할 수 있다.

## 찾는 정점이 없는 예시

그래프에 `Z`가 없다고 하자.

DFS는 갈 수 있는 모든 정점을 방문한 뒤에도 `Z`를 찾지 못한다.

```text
A -> B -> D -> E -> C -> F
```

모든 경로를 확인했는데도 목표가 없다면 찾지 못했다는 값을 반환한다.

```text
return null
```

## 종료 조건

재귀 DFS에서 중요한 종료 조건은 더 이상 방문할 다음 정점이 없는 경우이다.

```text
for each next in graph[current]:
    if next is not in visited:
        dfs(next)
```

방문하지 않은 다음 정점이 없다면 함수는 자연스럽게 끝난다.

목표를 찾는 DFS라면 목표 정점을 만났을 때도 종료할 수 있다.

```text
if current == target:
    return current
```

## 빈 그래프인 경우

그래프가 비어 있다면 시작 정점도 없다.

```text
graph = {}
```

이 경우 DFS를 실행하기 전에 시작 정점이 그래프에 있는지 확인해야 한다.

```text
if start is not in graph:
    return
```

## 시작 정점만 있는 경우

정점이 하나만 있고 간선이 없는 그래프도 있다.

```text
A
```

연결 관계는 다음과 같다.

```text
A: []
```

이 경우 DFS 결과는 시작 정점 하나뿐이다.

```text
A
```

## 연결되지 않은 그래프

그래프가 여러 덩어리로 나뉘어 있을 수 있다.

```text
A -- B

C -- D
```

`A`에서 DFS를 시작하면 `A`, `B`만 방문한다.

```text
A -> B
```

`C`, `D`는 `A`에서 갈 수 없으므로 방문하지 않는다.

그래프 전체를 모두 탐색하려면 모든 정점을 하나씩 확인하면서 아직 방문하지 않은 정점에서 DFS를 새로 시작해야 한다.

```text
for each vertex in graph:
    if vertex is not in visited:
        dfs(graph, vertex, visited)
```

## 재귀로 구현하는 DFS

재귀 DFS는 코드가 짧고 직관적이다.

```text
function dfs(current):
    visited[current] = true
    print current

    for each next in graph[current]:
        if visited[next] == false:
            dfs(next)
```

함수가 자기 자신을 다시 호출하면서 깊게 들어간다.

이때 함수 호출이 쌓이는 구조가 스택처럼 동작한다.

## 스택으로 구현하는 DFS

재귀를 사용하지 않고 직접 스택을 사용할 수도 있다.

```text
function dfs_stack(graph, start):
    stack = new Stack()
    visited = new Set()

    push start to stack

    while stack is not empty:
        current = pop from stack

        if current is in visited:
            continue

        add current to visited
        print current

        for each next in graph[current]:
            if next is not in visited:
                push next to stack
```

스택은 나중에 넣은 값이 먼저 나온다.

그래서 한 방향으로 계속 깊게 들어가는 DFS의 성질과 잘 맞는다.

## 재귀 DFS와 스택 DFS의 차이

```text
재귀 DFS:
    코드가 간단하다.
    함수 호출 깊이가 너무 깊으면 문제가 생길 수 있다.

스택 DFS:
    직접 스택을 관리한다.
    재귀 깊이 제한을 피할 수 있다.
```

방문 순서는 인접 정점을 어떤 순서로 넣느냐에 따라 달라질 수 있다.

## 방문 순서

DFS의 방문 순서는 하나로 고정되어 있지 않다.

인접 정점을 어떤 순서로 확인하느냐에 따라 달라진다.

```text
A: B, C
```

`B`를 먼저 확인하면 다음과 같을 수 있다.

```text
A -> B -> ...
```

`C`를 먼저 확인하면 다음과 같을 수 있다.

```text
A -> C -> ...
```

그래서 문제에서 방문 순서를 요구한다면 인접 정점을 정렬해야 할 수 있다.

## 트리에서의 DFS

트리에서 DFS는 루트에서 시작해 자식 방향으로 깊게 내려간다.

```text
        A
      /   \
     B     C
    / \     \
   D   E     F
```

왼쪽 자식부터 방문한다면 DFS 순서는 다음과 같을 수 있다.

```text
A -> B -> D -> E -> C -> F
```

## 전위 순회, 중위 순회, 후위 순회

트리 DFS에서는 방문 위치에 따라 여러 순회 방식이 있다.

```text
전위 순회: 현재 노드를 먼저 처리한다.
중위 순회: 왼쪽을 본 뒤 현재 노드를 처리하고 오른쪽을 본다.
후위 순회: 자식들을 먼저 본 뒤 현재 노드를 처리한다.
```

이진 트리에서는 특히 이 세 가지 순회가 자주 등장한다.

## 경로를 저장하는 DFS

DFS는 현재까지 지나온 경로를 저장하면서 탐색할 수 있다.

```text
path = []
```

정점에 들어갈 때 경로에 추가하고, 되돌아갈 때 제거한다.

```text
add current to path
dfs(next)
remove current from path
```

이 방식은 모든 경로를 찾는 문제에서 자주 사용한다.

## 백트래킹과 DFS

백트래킹은 DFS를 사용해 가능한 경우를 탐색하다가, 조건에 맞지 않으면 되돌아가는 방식이다.

```text
선택한다
다음 단계로 간다
조건에 맞지 않으면 되돌아온다
다른 선택을 시도한다
```

그래서 순열, 조합, 미로 탐색, 퍼즐 문제에서 DFS와 백트래킹이 자주 함께 쓰인다.

## 인접 리스트와 인접 행렬

그래프를 저장하는 방법에 따라 DFS의 구현이 조금 달라진다.

인접 리스트는 각 정점마다 연결된 정점 목록을 저장한다.

```text
A: B, C
B: A, D
```

인접 행렬은 정점 사이의 연결 여부를 표처럼 저장한다.

```text
    A B C
A   0 1 1
B   1 0 0
C   1 0 0
```

보통 간선이 많지 않은 그래프에서는 인접 리스트가 더 효율적이다.

## 시간 복잡도

정점 수를 `V`, 간선 수를 `E`라고 하자.

인접 리스트를 사용하면 DFS는 각 정점과 각 간선을 한 번씩 확인한다.

```text
시간 복잡도: O(V + E)
```

인접 행렬을 사용하면 각 정점마다 모든 정점을 확인해야 할 수 있다.

```text
시간 복잡도: O(V^2)
```

## 공간 복잡도

DFS는 방문 기록이 필요하다.

```text
visited: O(V)
```

재귀 DFS에서는 호출 스택도 필요하다.

최악의 경우 정점 수만큼 깊어질 수 있다.

```text
호출 스택: O(V)
```

그래서 기본 공간 복잡도는 다음과 같다.

```text
공간 복잡도: O(V)
```

## 장점

```text
구현이 간단하다.
경로 탐색에 좋다.
백트래킹과 잘 어울린다.
메모리를 비교적 적게 사용할 수 있다.
```

## 단점

```text
최단 거리를 보장하지 않는다.
재귀 구현은 호출 깊이 제한에 걸릴 수 있다.
방문 순서가 인접 정점 순서에 영향을 받는다.
```

## BFS와 비교

```text
DFS:
    깊게 들어간다.
    재귀 또는 스택을 사용한다.
    경로 탐색, 백트래킹에 좋다.

BFS:
    가까운 정점부터 넓게 본다.
    큐를 사용한다.
    가중치 없는 최단 거리 문제에 좋다.
```

둘 다 그래프 탐색의 기본이지만, 어떤 순서로 탐색해야 하는지에 따라 선택이 달라진다.

## 언제 사용하면 좋은가

DFS는 다음 상황에서 좋다.

```text
모든 경로를 탐색해야 할 때
연결 요소를 찾을 때
사이클 여부를 확인할 때
백트래킹 문제를 풀 때
트리 순회를 할 때
```

## 주의할 점

그래프에 사이클이 있으면 반드시 방문 체크가 필요하다.

재귀로 구현할 때는 그래프가 너무 깊으면 호출 깊이 문제가 생길 수 있다.

또한 DFS는 처음 발견한 경로가 최단 경로라는 보장이 없다.

## 한 문장 요약

DFS는 한 길을 끝까지 깊게 따라간 뒤 되돌아오며 다른 길을 확인하는 그래프 탐색 알고리즘이다.

## 알고리즘 흐름도

```text
시작
  |
  v
현재 정점 방문 표시
  |
  v
현재 정점 처리
  |
  v
인접 정점 확인
  |
  v
방문하지 않은 정점이 있는가?
  | yes
  v
그 정점으로 DFS 실행
  |
  v
```
