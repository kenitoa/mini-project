# 플로이드-워셜 알고리즘

플로이드-워셜 알고리즘은 모든 정점에서 모든 정점까지의 최단 거리를 구하는 알고리즘이다.

다익스트라나 벨만-포드가 한 시작 정점에서 다른 정점까지의 최단 거리를 구한다면, 플로이드-워셜은 모든 시작 정점을 한 번에 다룬다.

```text
다익스트라: 한 정점 -> 모든 정점
벨만-포드: 한 정점 -> 모든 정점
플로이드-워셜: 모든 정점 -> 모든 정점
```

플로이드-워셜은 동적 계획법을 사용한다.

핵심은 어떤 정점 `k`를 중간에 거쳐 가는 것이 더 짧은지 계속 확인하는 것이다.

## 핵심 생각

예를 들어 `A`에서 `C`로 가는 길을 생각해 보자.

직접 가는 길이 있을 수 있다.

```text
A -> C = 10
```

그런데 `B`를 거쳐 가면 더 짧을 수도 있다.

```text
A -> B = 3
B -> C = 2
A -> B -> C = 5
```

그러면 `A`에서 `C`까지의 최단 거리는 `10`이 아니라 `5`가 된다.

플로이드-워셜은 모든 정점 쌍에 대해 이런 질문을 반복한다.

```text
i에서 j로 바로 가는 것보다
i에서 k를 거쳐 j로 가는 것이 더 짧은가?
```

더 짧으면 거리 값을 바꾼다.

## 그래프에서 사용하는 말

플로이드-워셜을 이해하려면 다음 말을 알아두면 좋다.

```text
정점: 그래프의 지점
간선: 정점과 정점을 연결하는 길
가중치: 간선을 지날 때 드는 비용
최단 거리: 두 정점 사이를 이동하는 최소 비용
중간 정점: i에서 j로 갈 때 거쳐 가는 정점
거리 행렬: 모든 정점 쌍의 거리를 저장한 표
```

거리 행렬은 다음처럼 생각할 수 있다.

```text
dist[i][j] = i에서 j까지의 현재까지 알려진 최단 거리
```

처음에는 직접 연결된 간선 비용을 넣고, 자기 자신으로 가는 비용은 `0`으로 둔다.

## 거리 행렬

예를 들어 다음 그래프가 있다고 하자.

```text
A -> B (3)
A -> C (10)
B -> C (2)
C -> A (1)
```

처음 거리 행렬은 다음과 같다.

```text
      A   B   C
A     0   3  10
B   INF   0   2
C     1 INF   0
```

`INF`는 아직 갈 수 있는 길을 모른다는 뜻이다.

이제 중간 정점으로 `A`, `B`, `C`를 차례대로 사용해 보면서 거리를 갱신한다.

## 플로이드-워셜 의사코드

```text
floyd_warshall(그래프):
    거리 행렬을 만든다
    자기 자신으로 가는 거리는 0으로 둔다
    직접 연결된 간선의 비용을 거리 행렬에 넣는다

    모든 중간 정점 k에 대해 반복한다:
        모든 출발 정점 i에 대해 반복한다:
            모든 도착 정점 j에 대해 반복한다:
                i에서 j로 바로 가는 것보다
                i에서 k를 거쳐 j로 가는 것이 더 짧다면:
                    dist[i][j]를 갱신한다

    거리 행렬을 반환한다
```

핵심 식은 다음과 같다.

```text
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

## 조금 더 코드처럼 쓴 의사코드

```text
function floyd_warshall(dist):
    n = number of vertices

    for k from 0 to n - 1:
        for i from 0 to n - 1:
            for j from 0 to n - 1:
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]

    return dist
```

`k` 반복문이 가장 바깥에 있어야 한다.

`k`를 중간 정점으로 사용할 수 있는 범위를 하나씩 늘려 가는 동적 계획법이기 때문이다.

## 흐름을 쉽게 풀어보기

플로이드-워셜은 이렇게 생각하면 쉽다.

```text
1. 모든 정점 쌍의 거리를 표로 만든다.
2. 자기 자신으로 가는 거리는 0으로 둔다.
3. 직접 연결된 간선 비용을 넣는다.
4. 중간 정점을 하나 고른다.
5. 그 정점을 거쳐 가는 길이 더 짧은지 모든 쌍에 대해 확인한다.
6. 모든 정점을 중간 정점으로 사용해 볼 때까지 반복한다.
```

즉, 모든 정점 쌍에 대해 "어딘가를 거쳐 가면 더 짧아지는가"를 계속 확인한다.

## 예시

다음 그래프를 보자.

```text
A -> B (3)
A -> C (10)
B -> C (2)
C -> A (1)
```

처음 거리 행렬:

```text
      A   B   C
A     0   3  10
B   INF   0   2
C     1 INF   0
```

중간 정점으로 `B`를 사용할 때, `A -> C`를 확인한다.

```text
A -> C = 10
A -> B -> C = 3 + 2 = 5
```

`5`가 더 짧으므로 갱신한다.

```text
dist[A][C] = 5
```

중간 정점으로 `C`를 사용할 때, `B -> A`를 확인한다.

```text
B -> A = INF
B -> C -> A = 2 + 1 = 3
```

`B`에서 `A`로 갈 수 있는 길을 새로 찾았다.

```text
dist[B][A] = 3
```

최종적으로 모든 정점 쌍의 최단 거리를 얻는다.

## 경로 저장하기

최단 거리뿐 아니라 실제 경로를 알고 싶다면 다음 정점을 저장하는 배열을 둔다.

```text
next[i][j] = i에서 j로 갈 때 바로 다음에 가야 하는 정점
```

거리 갱신이 일어날 때 함께 갱신한다.

```text
if dist[i][k] + dist[k][j] < dist[i][j]:
    dist[i][j] = dist[i][k] + dist[k][j]
    next[i][j] = next[i][k]
```

그러면 `i`에서 `j`까지의 경로를 하나씩 따라갈 수 있다.

```text
current = i
while current != j:
    current = next[current][j]
```

## 음수 가중치

플로이드-워셜은 음수 가중치가 있는 간선도 처리할 수 있다.

하지만 음수 사이클이 있으면 최단 거리가 정해지지 않는다.

```text
음수 간선: 가능
음수 사이클: 최단 거리 없음
```

음수 사이클은 알고리즘 실행 후 대각선 값을 확인해서 찾을 수 있다.

```text
dist[i][i] < 0 이면 음수 사이클이 있다.
```

자기 자신으로 돌아오는 비용이 음수가 되었다는 것은, 돌수록 비용이 계속 줄어드는 사이클이 있다는 뜻이다.

## INF 처리

`INF`는 갈 수 없는 길을 나타낸다.

계산할 때 `INF`를 더하면 여전히 갈 수 없는 길로 보아야 한다.

```text
if dist[i][k] is INF:
    건너뛴다

if dist[k][j] is INF:
    건너뛴다
```

실제 코드에서는 아주 큰 숫자를 `INF`로 사용할 때 오버플로를 조심해야 한다.

## 방향 그래프와 무방향 그래프

방향 그래프에서는 간선 방향대로만 거리 행렬을 채운다.

```text
A -> B (3)
dist[A][B] = 3
```

무방향 그래프에서는 양쪽 방향을 모두 채운다.

```text
A -- B (3)
dist[A][B] = 3
dist[B][A] = 3
```

## 종료 조건

플로이드-워셜은 세 겹 반복문이 모두 끝나면 종료한다.

```text
for k
    for i
        for j
```

특정 한 쌍만 필요해도 기본 알고리즘은 모든 쌍을 계산한다.

한 시작 정점에서만 최단 거리가 필요하다면 다익스트라나 벨만-포드가 더 적합할 수 있다.

## 다익스트라와 비교

```text
다익스트라:
    한 시작 정점에서 모든 정점까지의 최단 거리
    음수 가중치가 없어야 함
    큰 희소 그래프에 적합

플로이드-워셜:
    모든 정점 쌍의 최단 거리
    음수 간선 가능
    정점 수가 아주 많으면 느림
```

모든 정점 쌍이 필요하면 플로이드-워셜을 생각할 수 있다.

한 시작 정점만 필요하면 다익스트라가 더 효율적인 경우가 많다.

## 시간 복잡도

정점 수를 `V`라고 하자.

플로이드-워셜은 세 겹 반복문을 사용한다.

```text
시간 복잡도: O(V^3)
```

간선 수와 관계없이 정점 수의 세제곱에 가까운 시간이 걸린다.

정점 수가 크면 사용하기 어렵다.

## 공간 복잡도

거리 행렬은 `V x V` 크기이다.

```text
공간 복잡도: O(V^2)
```

경로 복원을 위한 `next` 행렬도 저장하면 추가로 `O(V^2)`가 필요하다.

## 장점

```text
모든 정점 쌍의 최단 거리를 한 번에 구할 수 있다.
구현이 간단하다.
음수 가중치 간선도 처리할 수 있다.
음수 사이클 확인도 가능하다.
```

## 단점

```text
시간 복잡도가 O(V^3)이라 정점 수가 많으면 느리다.
거리 행렬 때문에 메모리를 많이 쓸 수 있다.
한 시작 정점만 필요할 때는 비효율적이다.
음수 사이클이 있으면 최단 거리 결과를 사용할 수 없다.
```

## 언제 사용하면 좋은가

플로이드-워셜은 다음 상황에서 좋다.

```text
모든 정점 쌍의 최단 거리가 필요할 때
정점 수가 크지 않을 때
구현을 단순하게 하고 싶을 때
음수 간선은 있지만 음수 사이클은 없을 때
```

## 주의할 점

반복문의 순서는 중요하다.

```text
for k
    for i
        for j
```

`k`가 가장 바깥에 있어야 중간 정점의 범위를 하나씩 늘리는 의미가 유지된다.

또한 `INF + 값` 계산을 조심해야 한다.

## 한 문장 요약

플로이드-워셜은 모든 정점을 중간 정점으로 한 번씩 사용해 보며 모든 정점 쌍의 최단 거리를 구하는 알고리즘이다.

## 알고리즘 흐름도

```text
시작
  |
  v
거리 행렬 초기화
  |
  v
자기 자신 거리 = 0
  |
  v
직접 간선 비용 입력
  |
  v
중간 정점 k 선택
  |
  v
모든 i, j 확인
  |
  v
i -> k -> j가 더 짧은가?
  | yes
  v
dist[i][j] 갱신
  |
  v
모든 k를 확인했는가?
  | no
  v
다음 k 선택
  |
  yes
  v
거리 행렬 반환
```

## 더 짧게 외우는 흐름

```text
거리 표 만들기
중간 정점 하나 선택
거쳐 가면 더 짧은지 확인
모든 정점을 중간 정점으로 써 보기
```

## 머릿속 결정 순서

```text
1. 모든 정점 쌍의 최단 거리가 필요한가?
2. 거리 행렬을 만들었는가?
3. 자기 자신은 0으로 두었는가?
4. 직접 간선 비용을 넣었는가?
5. k, i, j 순서로 반복했는가?
6. 음수 사이클 여부를 확인해야 하는가?
```
