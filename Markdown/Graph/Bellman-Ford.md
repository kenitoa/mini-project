# 벨만-포드 알고리즘

벨만-포드 알고리즘은 한 시작 정점에서 다른 모든 정점까지의 최단 거리를 구하는 그래프 알고리즘이다.

다익스트라 알고리즘과 비슷하게 최단 거리를 구하지만, 벨만-포드는 음수 가중치가 있는 간선도 처리할 수 있다.

단, 음수 사이클이 있으면 최단 거리가 정해지지 않는다.

```text
음수 가중치: 비용이 음수인 간선
음수 사이클: 한 바퀴 돌 때마다 전체 비용이 계속 줄어드는 사이클
```

벨만-포드는 모든 간선을 여러 번 확인하면서 더 짧은 길이 있으면 거리를 갱신한다.

이 과정을 `완화(relaxation)`라고 부른다.

## 핵심 생각

예를 들어 다음과 같은 방향 그래프가 있다고 하자.

```text
A -> B (4)
A -> C (5)
B -> C (-2)
C -> D (3)
B -> D (6)
```

`A`에서 시작한다고 하면 처음에는 `A`까지의 거리만 `0`이고, 나머지는 아직 모른다.

```text
A: 0
B: 무한대
C: 무한대
D: 무한대
```

이제 모든 간선을 하나씩 보면서 더 짧은 길이 있는지 확인한다.

```text
A -> B (4): A까지 0이므로 B는 4가 된다.
A -> C (5): A까지 0이므로 C는 5가 된다.
B -> C (-2): B까지 4이므로 C는 4 + (-2) = 2가 된다.
C -> D (3): C까지 2이므로 D는 5가 된다.
B -> D (6): B까지 4이므로 D는 10이지만, 기존 5가 더 짧다.
```

한 번 확인했다고 끝나는 것은 아니다.

간선의 순서에 따라 어떤 거리는 나중에야 좋아질 수 있기 때문이다.

그래서 정점이 `V`개라면 모든 간선을 최대 `V - 1`번 반복해서 확인한다.

## 그래프에서 사용하는 말

벨만-포드를 이해하려면 다음 말을 알아두면 좋다.

```text
정점: 그래프의 지점
간선: 정점과 정점을 연결하는 길
가중치: 간선을 지날 때 드는 비용
시작 정점: 최단 거리를 구하기 시작하는 정점
최단 거리: 시작 정점에서 어떤 정점까지 가는 최소 비용
완화: 더 짧은 경로를 발견했을 때 거리 값을 갱신하는 것
음수 사이클: 돌수록 비용이 계속 작아지는 사이클
```

간선 `u -> v`의 가중치가 `w`라면, 완화는 다음처럼 생각한다.

```text
dist[v] > dist[u] + w 이면
dist[v] = dist[u] + w 로 바꾼다.
```

즉, `u`까지 가는 길을 알고 있고, `u`에서 `v`로 가는 비용까지 더했을 때 더 짧다면 `v`의 거리를 바꾼다.

## 왜 V - 1번 반복하는가

사이클을 사용하지 않는 최단 경로는 같은 정점을 두 번 지나지 않는다.

정점이 `V`개라면, 같은 정점을 반복하지 않는 경로에서 사용할 수 있는 간선 수는 최대 `V - 1`개이다.

```text
정점 4개: A -> B -> C -> D
사용한 간선 수: 3개
```

그래서 모든 간선을 `1`번 확인하면 간선 `1`개짜리 최단 경로를 찾을 수 있고,

`2`번 확인하면 간선 `2`개짜리 최단 경로까지 반영될 수 있다.

결국 `V - 1`번 반복하면 가능한 최단 경로가 모두 반영된다.

## 벨만-포드 의사코드

```text
bellman_ford(그래프, 시작 정점):
    모든 정점의 거리를 무한대로 둔다
    시작 정점의 거리를 0으로 둔다

    정점 개수 - 1번 반복한다:
        모든 간선을 하나씩 확인한다:
            출발 정점까지의 거리가 무한대가 아니고
            출발 정점을 거쳐 도착 정점으로 가는 거리가 더 짧다면:
                도착 정점의 거리를 갱신한다

    모든 간선을 한 번 더 확인한다:
        아직도 더 짧아지는 간선이 있다면:
            음수 사이클이 있다고 판단한다

    거리 배열을 반환한다
```

핵심은 모든 간선을 반복해서 확인한다는 점이다.

한 정점에서 가장 가까운 정점만 고르는 다익스트라와 달리, 벨만-포드는 간선 전체를 계속 훑는다.

## 조금 더 코드처럼 쓴 의사코드

```text
function bellman_ford(vertices, edges, start):
    dist = map with default infinity
    dist[start] = 0

    for i from 1 to number of vertices - 1:
        for each edge in edges:
            u = edge.start
            v = edge.end
            w = edge.weight

            if dist[u] is not infinity and dist[v] > dist[u] + w:
                dist[v] = dist[u] + w

    for each edge in edges:
        u = edge.start
        v = edge.end
        w = edge.weight

        if dist[u] is not infinity and dist[v] > dist[u] + w:
            return "negative cycle exists"

    return dist
```

`dist[u]`가 무한대라면 아직 `u`까지 갈 수 없다는 뜻이다.

이 상태에서 `u -> v` 간선을 사용하면 잘못된 계산이 될 수 있으므로 건너뛴다.

## 흐름을 쉽게 풀어보기

벨만-포드는 이렇게 생각하면 쉽다.

```text
1. 시작 정점만 거리 0으로 둔다.
2. 나머지 정점은 아직 모르므로 무한대로 둔다.
3. 모든 간선을 하나씩 본다.
4. 어떤 간선을 통해 더 싸게 갈 수 있으면 거리를 바꾼다.
5. 이 과정을 정점 개수 - 1번 반복한다.
6. 한 번 더 확인했는데 또 줄어들면 음수 사이클이다.
```

즉, 한 번에 정답을 확정하지 않는다.

좋아질 수 있는 거리 값을 계속 퍼뜨려 나가면서 최단 거리를 만든다.

## 예시

```text
간선:
A -> B (4)
A -> C (5)
B -> C (-2)
C -> D (3)
B -> D (6)

start = A
```

처음 거리:

```text
A: 0
B: 무한대
C: 무한대
D: 무한대
```

첫 번째 반복에서 간선을 확인한다.

```text
A -> B: B = 4
A -> C: C = 5
B -> C: C = 2
C -> D: D = 5
B -> D: D는 기존 5가 더 짧으므로 그대로
```

결과:

```text
A: 0
B: 4
C: 2
D: 5
```

두 번째 반복에서 다시 확인해도 더 줄어드는 값이 없다.

따라서 최단 거리는 다음과 같다.

```text
A에서 A까지: 0
A에서 B까지: 4
A에서 C까지: 2
A에서 D까지: 5
```

## 음수 간선이 있는 예시

음수 간선은 지나면 비용이 줄어드는 간선이다.

```text
A -> B (10)
A -> C (3)
C -> B (-5)
```

`A`에서 `B`로 바로 가면 비용은 `10`이다.

하지만 `A -> C -> B`로 가면 비용은 다음과 같다.

```text
3 + (-5) = -2
```

따라서 `B`까지의 최단 거리는 `10`이 아니라 `-2`이다.

벨만-포드는 이런 경우도 처리할 수 있다.

## 음수 사이클

음수 사이클이 있으면 최단 거리가 존재하지 않는다.

예를 들어 다음과 같은 사이클이 있다고 하자.

```text
B -> C (-2)
C -> B (-3)
```

`B -> C -> B`를 한 바퀴 돌면 비용이 `-5` 줄어든다.

두 바퀴 돌면 `-10`, 세 바퀴 돌면 `-15`가 된다.

```text
계속 돌수록 비용이 작아진다.
```

이 경우에는 "가장 짧은 거리"라는 값이 정해질 수 없다.

벨만-포드는 `V - 1`번 완화한 뒤, 한 번 더 완화가 가능한지 확인해서 음수 사이클을 찾는다.

## 종료 조건

기본 종료 조건은 다음과 같다.

```text
모든 간선을 정점 개수 - 1번 확인하면 종료한다.
```

하지만 최적화를 한다면, 어떤 반복에서 거리 갱신이 한 번도 일어나지 않을 때 바로 멈출 수 있다.

```text
updated = false

간선을 모두 확인했는데 updated가 false라면:
    더 이상 바뀔 거리가 없으므로 종료한다
```

이렇게 하면 실제 입력에 따라 더 빨리 끝날 수 있다.

## 시작 정점에서 갈 수 없는 정점

시작 정점에서 도달할 수 없는 정점의 거리는 계속 무한대로 남는다.

```text
A -- B

C -- D
```

`A`에서 시작하면 `C`, `D`에는 갈 수 없다.

```text
A: 0
B: 어떤 값
C: 무한대
D: 무한대
```

이 값은 "아직 계산하지 않았다"가 아니라 "시작 정점에서 도달할 수 없다"는 뜻이다.

## 방향 그래프와 무방향 그래프

벨만-포드는 보통 방향 그래프의 간선 목록을 기준으로 설명한다.

무방향 그래프에서 간선 `A -- B (3)`이 있다면 양방향 간선 두 개로 생각할 수 있다.

```text
A -> B (3)
B -> A (3)
```

단, 무방향 그래프에 음수 간선이 있으면 주의해야 한다.

무방향 음수 간선 하나는 사실상 음수 사이클처럼 동작할 수 있다.

```text
A -> B (-3)
B -> A (-3)
```

이 두 간선을 반복해서 지나면 비용이 계속 줄어든다.

## 경로 저장하기

최단 거리뿐 아니라 실제 경로도 알고 싶다면 이전 정점을 저장한다.

```text
prev[v] = v로 오기 직전에 지나온 정점
```

거리 갱신이 일어날 때 `prev`도 함께 바꾼다.

```text
if dist[v] > dist[u] + w:
    dist[v] = dist[u] + w
    prev[v] = u
```

예를 들어 `D`의 이전 정점이 `C`, `C`의 이전 정점이 `B`, `B`의 이전 정점이 `A`라면 경로는 다음과 같다.

```text
A -> B -> C -> D
```

## 다익스트라와 비교

```text
다익스트라:
    음수 가중치가 없을 때 빠르게 최단 거리를 구한다.
    보통 우선순위 큐를 사용한다.

벨만-포드:
    음수 가중치가 있어도 사용할 수 있다.
    음수 사이클을 찾을 수 있다.
    다익스트라보다 느리다.
```

음수 가중치가 없다면 보통 다익스트라가 더 효율적이다.

음수 가중치나 음수 사이클 검사가 필요하다면 벨만-포드가 적합하다.

## 시간 복잡도

정점 수를 `V`, 간선 수를 `E`라고 하자.

벨만-포드는 모든 간선을 `V - 1`번 확인한다.

```text
시간 복잡도: O(VE)
```

간선을 한 번 더 확인해서 음수 사이클을 검사해도 `O(E)`가 추가될 뿐이다.

전체 시간 복잡도는 그대로 `O(VE)`이다.

## 공간 복잡도

거리 배열이 필요하다.

```text
dist: O(V)
```

경로를 복원하려면 이전 정점 배열도 필요하다.

```text
prev: O(V)
```

그래서 기본 공간 복잡도는 다음과 같다.

```text
공간 복잡도: O(V)
```

간선 목록 자체를 저장하는 공간은 그래프 입력 저장 방식에 따라 `O(E)`가 된다.

## 장점

```text
음수 가중치가 있어도 사용할 수 있다.
음수 사이클을 감지할 수 있다.
구현이 비교적 단순하다.
간선 목록만 있으면 구현하기 쉽다.
```

## 단점

```text
다익스트라보다 느리다.
정점과 간선이 많으면 시간이 오래 걸린다.
음수 사이클이 있으면 최단 거리 결과를 낼 수 없다.
```

## 언제 사용하면 좋은가

벨만-포드는 다음 상황에서 좋다.

```text
간선 가중치에 음수가 포함될 수 있을 때
음수 사이클이 있는지 확인해야 할 때
그래프가 아주 크지 않을 때
최단 거리 개념을 완화 방식으로 이해하고 싶을 때
```

## 주의할 점

음수 간선이 있다고 무조건 문제가 되는 것은 아니다.

문제가 되는 것은 음수 사이클이다.

```text
음수 간선: 가능
음수 사이클: 최단 거리 없음
```

또한 시작 정점에서 도달할 수 없는 정점은 완화하면 안 된다.

```text
dist[u]가 무한대라면 u에서 출발하는 간선은 건너뛴다.
```

## 한 문장 요약

벨만-포드는 모든 간선을 반복해서 완화하며, 음수 가중치와 음수 사이클까지 다룰 수 있는 최단 거리 알고리즘이다.

## 알고리즘 흐름도

```text
시작
  |
  v
모든 거리 = 무한대
  |
  v
시작 정점 거리 = 0
  |
  v
V - 1번 반복
  |
  v
모든 간선 확인
  |
  v
더 짧은 경로가 있는가?
  | yes
  v
거리 갱신
  |
  v
반복 종료 후 모든 간선 한 번 더 확인
  |
  v
또 갱신 가능한가?
  | yes
  v
음수 사이클 있음
  |
  no
  v
최단 거리 반환
```

## 더 짧게 외우는 흐름

```text
거리 초기화
모든 간선 V - 1번 완화
한 번 더 완화되면 음수 사이클
아니면 최단 거리 완성
```

## 머릿속 결정 순서

```text
1. 시작 정점이 어디인가?
2. 모든 거리를 무한대로 두었는가?
3. 시작 정점만 0으로 두었는가?
4. 모든 간선을 V - 1번 확인했는가?
5. 더 짧은 길이 있으면 갱신했는가?
6. 마지막에 음수 사이클 검사를 했는가?
```
