# 프림 알고리즘

프림 알고리즘은 최소 신장 트리를 만드는 알고리즘이다.

최소 신장 트리는 그래프의 모든 정점을 연결하면서, 사용한 간선의 가중치 합이 가장 작아지도록 만든 트리이다.

```text
신장 트리: 모든 정점을 연결하는 트리
최소 신장 트리: 간선 비용의 합이 가장 작은 신장 트리
```

프림은 하나의 정점에서 시작해서, 현재 만들어진 트리와 연결되는 가장 싼 간선을 계속 선택한다.

## 핵심 생각

예를 들어 다음과 같은 무방향 그래프가 있다고 하자.

```text
A --1-- B
|       |
4       2
|       |
C --3-- D
```

`A`에서 시작한다고 하자.

처음에는 트리에 `A`만 들어 있다.

```text
트리: {A}
```

`A`와 연결된 간선 중 가장 싼 것은 `A-B (1)`이다.

```text
트리: {A, B}
선택 간선: A-B (1)
```

이제 트리 `{A, B}`에서 밖으로 나가는 간선을 본다.

```text
A-C (4)
B-D (2)
```

가장 싼 것은 `B-D (2)`이다.

```text
트리: {A, B, D}
선택 간선: B-D (2)
```

다음으로 밖에 있는 정점 `C`와 연결되는 가장 싼 간선은 `D-C (3)`이다.

```text
트리: {A, B, C, D}
선택 간선: D-C (3)
```

총 비용은 다음과 같다.

```text
1 + 2 + 3 = 6
```

## 그래프에서 사용하는 말

프림을 이해하려면 다음 말을 알아두면 좋다.

```text
정점: 그래프의 지점
간선: 정점과 정점을 연결하는 길
가중치: 간선을 선택했을 때 드는 비용
트리: 사이클이 없는 연결 그래프
신장 트리: 모든 정점을 포함하는 트리
최소 신장 트리: 전체 비용이 가장 작은 신장 트리
방문한 정점: 이미 트리에 포함된 정점
```

프림은 현재 트리에 포함된 정점들과 아직 포함되지 않은 정점 사이의 간선 중 가장 싼 것을 선택한다.

## 우선순위 큐를 사용하는 이유

프림은 매번 가장 싼 간선을 골라야 한다.

모든 간선을 매번 직접 확인하면 시간이 오래 걸릴 수 있다.

그래서 보통 우선순위 큐를 사용한다.

```text
우선순위 큐에 저장하는 값:
(간선 비용, 도착 정점, 출발 정점)
```

가장 비용이 작은 간선을 빠르게 꺼낼 수 있다.

이미 트리에 포함된 정점으로 가는 간선은 꺼내더라도 건너뛴다.

## 프림 의사코드

```text
prim(그래프, 시작 정점):
    방문 기록을 만든다
    결과 간선 목록을 비워 둔다
    총 비용을 0으로 둔다

    시작 정점과 연결된 간선들을 우선순위 큐에 넣는다
    시작 정점을 방문했다고 표시한다

    우선순위 큐가 비어 있지 않은 동안 반복한다:
        가장 비용이 작은 간선을 꺼낸다

        간선이 가리키는 정점이 이미 방문한 정점이라면:
            건너뛴다

        그 간선을 선택한다
        도착 정점을 방문했다고 표시한다
        총 비용에 간선 비용을 더한다

        새로 방문한 정점과 연결된 간선들을 우선순위 큐에 넣는다

        선택한 간선 수가 정점 수 - 1개가 되면:
            종료한다

    결과 간선 목록과 총 비용을 반환한다
```

핵심은 현재 트리에서 밖으로 나가는 가장 싼 간선을 선택하는 것이다.

## 조금 더 코드처럼 쓴 의사코드

```text
function prim(graph, start):
    visited = new Set()
    priority_queue = new PriorityQueue()
    mst = []
    total_cost = 0

    add start to visited

    for each edge from start:
        push (edge.weight, start, edge.to) to priority_queue

    while priority_queue is not empty:
        weight, from, to = pop smallest from priority_queue

        if to is in visited:
            continue

        add to to visited
        add (from, to, weight) to mst
        total_cost = total_cost + weight

        for each edge from to:
            if edge.to is not in visited:
                push (edge.weight, to, edge.to) to priority_queue

        if length(mst) == number of vertices - 1:
            break

    return mst, total_cost
```

`visited`는 현재 최소 신장 트리에 포함된 정점을 뜻한다.

## 흐름을 쉽게 풀어보기

프림은 이렇게 생각하면 쉽다.

```text
1. 아무 정점 하나에서 시작한다.
2. 현재 트리와 연결된 간선 중 가장 싼 간선을 고른다.
3. 그 간선이 새로운 정점으로 간다면 선택한다.
4. 새 정점에서 나가는 간선을 후보에 추가한다.
5. 모든 정점이 연결될 때까지 반복한다.
```

즉, 작은 트리를 하나 만들고, 그 트리를 가장 싼 방식으로 계속 키워 나간다.

## 예시

다음 간선들이 있다고 하자.

```text
A-B (1)
A-C (4)
B-C (2)
B-D (5)
C-D (3)
```

`A`에서 시작한다.

```text
visited = {A}
candidate = A-B(1), A-C(4)
```

가장 싼 `A-B (1)`을 선택한다.

```text
visited = {A, B}
mst = [A-B]
total = 1
```

`B`에서 나가는 간선을 후보에 추가한다.

```text
B-C (2)
B-D (5)
```

후보 중 가장 싼 것은 `B-C (2)`이다.

```text
visited = {A, B, C}
mst = [A-B, B-C]
total = 3
```

`C`에서 나가는 간선을 후보에 추가한다.

```text
C-D (3)
```

후보 중 가장 싼 새 정점 연결 간선은 `C-D (3)`이다.

```text
visited = {A, B, C, D}
mst = [A-B, B-C, C-D]
total = 6
```

정점이 모두 연결되었으므로 종료한다.

## 이미 방문한 정점으로 가는 간선

우선순위 큐에는 이미 방문한 정점으로 가는 간선도 남아 있을 수 있다.

예를 들어 `A`, `B`, `C`가 이미 트리에 들어와 있는데 `A-C` 간선이 큐에 남아 있을 수 있다.

이 간선을 선택하면 사이클이 생긴다.

그래서 꺼낸 간선의 도착 정점이 이미 방문한 정점이면 건너뛴다.

```text
if to is in visited:
    continue
```

## 시작 정점

연결된 무방향 그래프라면 어떤 정점에서 시작해도 최소 신장 트리의 총 비용은 같다.

단, 최소 신장 트리가 여러 개 있을 수 있으므로 선택되는 간선 모양은 달라질 수 있다.

```text
총 비용은 같을 수 있지만
선택 간선은 다를 수 있다.
```

## 종료 조건

최소 신장 트리는 정점이 `V`개일 때 간선이 `V - 1`개이다.

그래서 다음 조건에서 종료할 수 있다.

```text
선택한 간선 수 == 정점 수 - 1
```

또는 방문한 정점 수가 전체 정점 수와 같을 때도 종료할 수 있다.

```text
visited 정점 수 == 전체 정점 수
```

## 연결되지 않은 그래프

그래프가 연결되어 있지 않으면 하나의 최소 신장 트리를 만들 수 없다.

```text
A -- B

C -- D
```

`A`에서 시작한 프림은 `A`, `B`만 연결하고 끝난다.

`C`, `D`는 도달할 수 없기 때문이다.

그래프 전체의 최소 신장 숲을 만들려면 방문하지 않은 정점에서 프림을 다시 시작해야 한다.

```text
for each vertex:
    if vertex is not in visited:
        prim from vertex
```

## 크루스칼과 비교

프림과 크루스칼은 둘 다 최소 신장 트리를 구한다.

```text
프림:
    하나의 정점에서 시작해 트리를 키운다.
    현재 트리와 연결되는 가장 싼 간선을 고른다.
    우선순위 큐를 자주 사용한다.

크루스칼:
    모든 간선을 비용 순서로 정렬한다.
    사이클이 생기지 않는 간선을 고른다.
    유니온 파인드를 자주 사용한다.
```

프림은 한 덩어리를 확장하는 느낌이고, 크루스칼은 여러 덩어리를 합치는 느낌이다.

## 시간 복잡도

정점 수를 `V`, 간선 수를 `E`라고 하자.

인접 리스트와 우선순위 큐를 사용하면 보통 다음과 같다.

```text
시간 복잡도: O(E log V)
```

간선을 우선순위 큐에 넣고 꺼내는 비용이 들어가기 때문이다.

인접 행렬을 사용해 매번 가장 싼 간선을 직접 찾는 방식은 다음과 같다.

```text
시간 복잡도: O(V^2)
```

## 공간 복잡도

방문 배열, 우선순위 큐, 결과 간선 목록이 필요하다.

```text
visited: O(V)
priority_queue: O(E)
mst: O(V)
```

그래서 그래프 저장 공간을 제외하면 보통 다음처럼 본다.

```text
공간 복잡도: O(V + E)
```

## 장점

```text
트리를 하나씩 확장하는 방식이라 이해하기 쉽다.
우선순위 큐를 사용하면 효율적이다.
연결된 그래프에서 최소 비용 연결 구조를 구할 수 있다.
간선이 많은 그래프에서도 구현 방식에 따라 잘 동작한다.
```

## 단점

```text
우선순위 큐 구현이 필요할 수 있다.
연결되지 않은 그래프에서는 시작 정점이 속한 부분만 처리된다.
방향 그래프의 최단 경로 문제와 헷갈리면 안 된다.
```

## 언제 사용하면 좋은가

프림은 다음 상황에서 좋다.

```text
최소 비용으로 모든 정점을 연결해야 할 때
무방향 연결 그래프가 주어졌을 때
한 정점에서 시작해 연결 영역을 확장하는 방식이 자연스러울 때
인접 리스트와 우선순위 큐를 활용할 수 있을 때
```

## 주의할 점

프림은 최단 경로 알고리즘이 아니다.

시작 정점에서 각 정점까지의 최단 거리를 구하는 것이 아니라, 모든 정점을 연결하는 전체 비용을 최소로 만드는 알고리즘이다.

```text
다익스트라: 시작점에서 각 정점까지 최단 거리
프림: 모든 정점을 연결하는 최소 비용 트리
```

또한 무방향 그래프의 최소 신장 트리 문제에서 사용한다.

## 한 문장 요약

프림은 하나의 정점에서 시작해 현재 트리와 연결되는 가장 싼 간선을 계속 선택하며 최소 신장 트리를 만드는 알고리즘이다.

## 알고리즘 흐름도

```text
시작
  |
  v
시작 정점을 방문 처리
  |
  v
시작 정점의 간선을 큐에 넣기
  |
  v
큐가 비었는가?
  | no
  v
가장 싼 간선 꺼내기
  |
  v
도착 정점이 이미 방문되었는가?
  | yes
  v
건너뛰기
  |
  no
  v
간선 선택
  |
  v
도착 정점 방문 처리
  |
  v
새 정점의 간선을 큐에 넣기
  |
  v
모든 정점을 방문했는가?
  | no
  v
큐 확인으로 돌아가기
  |
  yes
  v
최소 신장 트리 완성
```

## 더 짧게 외우는 흐름

```text
정점 하나에서 시작
가장 싼 연결 간선 선택
새 정점 추가
새 간선 후보 추가
모든 정점이 연결될 때까지 반복
```

## 머릿속 결정 순서

```text
1. 무방향 그래프인가?
2. 시작 정점을 방문 처리했는가?
3. 후보 간선을 우선순위 큐에 넣었는가?
4. 가장 싼 간선을 꺼냈는가?
5. 이미 방문한 정점으로 가면 건너뛰었는가?
6. 새 정점을 추가하고 간선 후보를 갱신했는가?
```
