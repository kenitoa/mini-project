# 다익스트라 알고리즘

다익스트라 알고리즘은 한 시작 정점에서 다른 모든 정점까지의 최단 거리를 구하는 그래프 알고리즘이다.

간선마다 비용이 있을 때, 시작 정점에서 각 정점까지 가는 가장 작은 비용을 계산한다.

단, 다익스트라는 음수 가중치가 없는 그래프에서 사용해야 한다.

```text
가능: 0 이상의 가중치
주의: 음수 가중치
```

보통 우선순위 큐를 사용해서 현재까지 가장 가까운 정점부터 처리한다.

## 핵심 생각

예를 들어 다음과 같은 그래프가 있다고 하자.

```text
A --2-- B --1-- C
|      /       |
5     2        3
|   /          |
D ------------ E
       1
```

`A`에서 시작한다고 하면 처음에는 `A`까지의 거리만 `0`이다.

```text
A: 0
B: 무한대
C: 무한대
D: 무한대
E: 무한대
```

`A`에서 바로 갈 수 있는 곳을 확인한다.

```text
A -> B 비용 2
A -> D 비용 5
```

그러면 거리 값은 다음처럼 바뀐다.

```text
A: 0
B: 2
D: 5
```

이제 아직 처리하지 않은 정점 중 거리가 가장 작은 `B`를 고른다.

`B`를 거쳐 다른 곳으로 가는 비용을 확인한다.

```text
A -> B -> C = 2 + 1 = 3
A -> B -> D = 2 + 2 = 4
```

기존 `D`까지의 거리는 `5`였는데, `B`를 거치면 `4`가 된다.

그래서 `D`의 거리를 갱신한다.

이렇게 가장 가까운 정점을 하나씩 확정해 나가며 최단 거리를 만든다.

## 그래프에서 사용하는 말

다익스트라를 이해하려면 다음 말을 알아두면 좋다.

```text
정점: 그래프의 지점
간선: 정점과 정점을 연결하는 길
가중치: 간선을 지날 때 드는 비용
시작 정점: 최단 거리를 계산하기 시작하는 정점
최단 거리: 시작 정점에서 어떤 정점까지 가는 최소 비용
완화: 더 짧은 경로를 발견했을 때 거리 값을 갱신하는 것
우선순위 큐: 가장 작은 거리 값을 가진 정점을 빨리 꺼내기 위한 자료 구조
```

간선 `u -> v`의 가중치가 `w`라면 완화는 다음과 같다.

```text
dist[v] > dist[u] + w 이면
dist[v] = dist[u] + w
```

## 우선순위 큐를 사용하는 이유

다익스트라는 아직 처리하지 않은 정점 중에서 현재 거리가 가장 짧은 정점을 먼저 처리한다.

매번 모든 정점을 훑어서 가장 작은 값을 찾으면 시간이 오래 걸린다.

우선순위 큐를 사용하면 가장 짧은 거리의 정점을 빠르게 꺼낼 수 있다.

```text
우선순위 큐에 저장하는 값:
(현재까지의 거리, 정점)
```

예를 들어 다음 값들이 있다면:

```text
(5, D)
(2, B)
(3, C)
```

우선순위 큐에서는 거리 값이 가장 작은 `(2, B)`가 먼저 나온다.

## 다익스트라 의사코드

```text
dijkstra(그래프, 시작 정점):
    모든 정점의 거리를 무한대로 둔다
    시작 정점의 거리를 0으로 둔다

    우선순위 큐를 만든다
    시작 정점을 거리 0으로 우선순위 큐에 넣는다

    우선순위 큐가 비어 있지 않은 동안 반복한다:
        가장 거리가 짧은 정점을 꺼낸다

        꺼낸 거리가 이미 저장된 거리보다 크면 건너뛴다

        현재 정점과 연결된 간선을 하나씩 확인한다:
            현재 정점을 거쳐 다음 정점으로 가는 거리가 더 짧다면:
                다음 정점의 거리를 갱신한다
                갱신된 거리를 우선순위 큐에 넣는다

    거리 배열을 반환한다
```

핵심은 가장 짧아 보이는 정점부터 꺼내고, 그 정점을 통해 주변 거리 값을 갱신하는 것이다.

## 조금 더 코드처럼 쓴 의사코드

```text
function dijkstra(graph, start):
    dist = map with default infinity
    dist[start] = 0

    priority_queue = new PriorityQueue()
    push (0, start) to priority_queue

    while priority_queue is not empty:
        current_dist, current = pop smallest from priority_queue

        if current_dist > dist[current]:
            continue

        for each edge from current:
            next = edge.to
            weight = edge.weight
            new_dist = current_dist + weight

            if new_dist < dist[next]:
                dist[next] = new_dist
                push (new_dist, next) to priority_queue

    return dist
```

`current_dist > dist[current]`인 경우는 이미 더 짧은 거리로 큐에 다시 들어간 적이 있다는 뜻이다.

그래서 오래된 값은 건너뛴다.

## 흐름을 쉽게 풀어보기

다익스트라는 이렇게 생각하면 쉽다.

```text
1. 시작 정점의 거리를 0으로 둔다.
2. 나머지 정점은 무한대로 둔다.
3. 현재 가장 가까운 정점을 고른다.
4. 그 정점을 거쳐 주변 정점으로 가는 비용을 계산한다.
5. 더 짧아지면 거리 값을 바꾼다.
6. 더 이상 처리할 정점이 없을 때까지 반복한다.
```

즉, 가까운 곳부터 확정해 나가며 최단 거리를 퍼뜨리는 방식이다.

## 예시

다음 그래프에서 `A`를 시작 정점으로 둔다.

```text
A -> B (2)
A -> C (5)
B -> C (1)
B -> D (2)
C -> D (3)
D -> E (1)
C -> E (5)
```

처음 상태:

```text
A: 0
B: 무한대
C: 무한대
D: 무한대
E: 무한대
```

`A`를 처리한다.

```text
A -> B: B = 2
A -> C: C = 5
```

현재 가장 가까운 정점은 `B`이다.

`B`를 처리한다.

```text
B -> C: 2 + 1 = 3
C는 기존 5보다 3이 짧으므로 갱신

B -> D: 2 + 2 = 4
D = 4
```

현재 가장 가까운 정점은 `C`이다.

`C`를 처리한다.

```text
C -> D: 3 + 3 = 6
D는 기존 4가 더 짧으므로 그대로

C -> E: 3 + 5 = 8
E = 8
```

그 다음 `D`를 처리한다.

```text
D -> E: 4 + 1 = 5
E는 기존 8보다 5가 짧으므로 갱신
```

최종 결과는 다음과 같다.

```text
A: 0
B: 2
C: 3
D: 4
E: 5
```

## 경로 저장하기

최단 거리뿐 아니라 실제 경로를 알고 싶다면 이전 정점을 저장한다.

```text
prev[next] = current
```

거리 갱신이 일어날 때 함께 기록한다.

```text
if new_dist < dist[next]:
    dist[next] = new_dist
    prev[next] = current
```

예를 들어 `E`까지의 이전 정점들이 다음과 같다면:

```text
prev[E] = D
prev[D] = B
prev[B] = A
```

경로는 거꾸로 따라간 뒤 뒤집으면 된다.

```text
A -> B -> D -> E
```

## 음수 가중치가 있으면 안 되는 이유

다익스트라는 현재 가장 가까운 정점을 고르면 그 거리가 최단 거리라고 믿고 진행한다.

이 믿음은 간선 가중치가 모두 0 이상일 때만 안전하다.

음수 가중치가 있으면 나중에 다른 경로를 통해 이미 처리한 정점의 거리가 더 작아질 수 있다.

```text
A -> B (2)
A -> C (5)
C -> B (-10)
```

처음에는 `B`가 `2`로 가장 가까워 보인다.

하지만 `A -> C -> B`로 가면 다음과 같다.

```text
5 + (-10) = -5
```

이런 경우 다익스트라의 기본 전제가 깨진다.

음수 가중치가 있다면 벨만-포드를 사용해야 한다.

## 시작 정점에서 갈 수 없는 정점

시작 정점에서 도달할 수 없는 정점은 거리 값이 무한대로 남는다.

```text
A -> B

C -> D
```

`A`에서 시작하면 `C`, `D`에는 갈 수 없다.

```text
C: 무한대
D: 무한대
```

이 값은 "아직 처리하지 않았다"가 아니라 "도달할 수 없다"는 뜻이다.

## 방향 그래프와 무방향 그래프

방향 그래프에서는 간선 방향대로만 이동한다.

```text
A -> B
```

이 경우 `A`에서 `B`로 갈 수 있지만, `B`에서 `A`로 갈 수 있는 것은 아니다.

무방향 그래프에서는 양쪽 방향 간선을 모두 넣어서 표현한다.

```text
A -- B (3)

A -> B (3)
B -> A (3)
```

## 종료 조건

기본 종료 조건은 우선순위 큐가 비는 것이다.

```text
while priority_queue is not empty
```

큐가 비었다는 것은 더 이상 갱신할 수 있는 정점이 없다는 뜻이다.

특정 목표 정점까지만 필요하다면, 목표 정점이 가장 작은 거리로 큐에서 나오는 순간 종료할 수 있다.

```text
if current == target:
    return dist[target]
```

## BFS와의 관계

가중치가 없는 그래프에서 최단 거리를 구할 때는 BFS를 사용한다.

모든 간선 비용이 `1`이라면 BFS가 더 간단하고 빠르다.

```text
BFS:
    모든 간선 비용이 같을 때 사용

다익스트라:
    간선 비용이 서로 다르지만 음수는 없을 때 사용
```

다익스트라는 BFS를 가중치 그래프에 맞게 확장한 것처럼 생각할 수 있다.

## 벨만-포드와 비교

```text
다익스트라:
    음수 가중치가 없을 때 빠르다.
    우선순위 큐를 사용하면 효율적이다.

벨만-포드:
    음수 가중치가 있어도 사용할 수 있다.
    음수 사이클을 감지할 수 있다.
    다익스트라보다 느리다.
```

음수 가중치가 없다면 보통 다익스트라를 먼저 생각한다.

## 시간 복잡도

정점 수를 `V`, 간선 수를 `E`라고 하자.

우선순위 큐를 사용하면 보통 다음과 같다.

```text
시간 복잡도: O((V + E) log V)
```

간선이 많지 않은 그래프에서는 매우 효율적이다.

우선순위 큐 없이 매번 가장 작은 거리의 정점을 직접 찾으면 다음과 같다.

```text
시간 복잡도: O(V^2)
```

## 공간 복잡도

거리 배열과 우선순위 큐가 필요하다.

```text
dist: O(V)
priority_queue: O(E)까지 들어갈 수 있음
```

그래서 보통 다음처럼 본다.

```text
공간 복잡도: O(V + E)
```

경로 복원을 위한 `prev` 배열을 추가하면 `O(V)`가 더 필요하다.

## 장점

```text
음수 가중치가 없는 최단 경로 문제에서 빠르다.
우선순위 큐를 사용하면 큰 그래프에도 잘 동작한다.
한 시작 정점에서 모든 정점까지의 최단 거리를 구할 수 있다.
경로 복원도 쉽게 붙일 수 있다.
```

## 단점

```text
음수 가중치가 있으면 사용할 수 없다.
음수 사이클을 감지하지 못한다.
우선순위 큐 구현을 정확히 해야 한다.
```

## 언제 사용하면 좋은가

다익스트라는 다음 상황에서 좋다.

```text
도로망처럼 비용이 모두 0 이상일 때
한 시작점에서 여러 지점까지의 최단 거리가 필요할 때
간선마다 비용이 다를 때
특정 목적지까지의 최소 비용을 구할 때
```

## 주의할 점

다익스트라를 쓰기 전에 음수 가중치가 있는지 확인해야 한다.

```text
음수 가중치 없음: 다익스트라 가능
음수 가중치 있음: 벨만-포드 고려
```

또한 우선순위 큐에 같은 정점이 여러 번 들어갈 수 있다.

그래서 꺼낸 값이 현재 거리보다 크면 건너뛰는 처리가 필요하다.

## 한 문장 요약

다익스트라는 음수 가중치가 없는 그래프에서 현재 가장 가까운 정점부터 처리해 최단 거리를 구하는 알고리즘이다.

## 알고리즘 흐름도

```text
시작
  |
  v
거리 배열 초기화
  |
  v
시작 정점 거리 = 0
  |
  v
우선순위 큐에 시작 정점 넣기
  |
  v
큐가 비었는가?
  | no
  v
가장 짧은 거리의 정점 꺼내기
  |
  v
오래된 거리인가?
  | yes
  v
건너뛰기
  |
  no
  v
인접 간선 확인
  |
  v
더 짧은 경로가 있는가?
  | yes
  v
거리 갱신 후 큐에 넣기
  |
  v
큐 확인으로 돌아가기
  |
  yes
  v
최단 거리 반환
```

## 더 짧게 외우는 흐름

```text
거리 초기화
가장 가까운 정점 꺼내기
주변 거리 갱신
큐가 빌 때까지 반복
```

## 머릿속 결정 순서

```text
1. 음수 가중치가 없는가?
2. 시작 정점을 0으로 두었는가?
3. 가장 작은 거리의 정점을 꺼냈는가?
4. 오래된 큐 값은 건너뛰었는가?
5. 인접 정점의 거리를 갱신했는가?
6. 필요한 경우 이전 정점을 저장했는가?
```
