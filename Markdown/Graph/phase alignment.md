# 위상 정렬

위상 정렬은 방향 그래프에서 어떤 일을 먼저 해야 하는지 순서를 정하는 알고리즘이다.

정확히는 사이클이 없는 방향 그래프에서, 모든 간선의 방향을 지키는 정점 순서를 찾는다.

이런 그래프를 DAG라고 부른다.

```text
DAG = Directed Acyclic Graph
방향이 있고 사이클이 없는 그래프
```

예를 들어 `A -> B`라는 간선이 있다면 `A`가 `B`보다 먼저 나와야 한다.

## 핵심 생각

예를 들어 다음과 같은 작업 순서가 있다고 하자.

```text
자료구조 공부 -> 그래프 공부
그래프 공부 -> 다익스트라 공부
그래프 공부 -> 최소 신장 트리 공부
```

이 관계를 그래프로 나타내면 다음과 같다.

```text
자료구조 -> 그래프 -> 다익스트라
                 |
                 v
              최소 신장 트리
```

위상 정렬 결과는 다음처럼 될 수 있다.

```text
자료구조 -> 그래프 -> 다익스트라 -> 최소 신장 트리
```

또는 다음도 가능하다.

```text
자료구조 -> 그래프 -> 최소 신장 트리 -> 다익스트라
```

중요한 것은 간선의 방향을 어기지 않는 것이다.

```text
자료구조는 그래프보다 먼저
그래프는 다익스트라보다 먼저
그래프는 최소 신장 트리보다 먼저
```

## 그래프에서 사용하는 말

위상 정렬을 이해하려면 다음 말을 알아두면 좋다.

```text
정점: 하나의 작업 또는 대상
간선: 먼저 해야 하는 관계
방향 그래프: 간선에 방향이 있는 그래프
사이클: 다시 자기 자신으로 돌아오는 구조
진입 차수: 어떤 정점으로 들어오는 간선의 개수
DAG: 방향이 있고 사이클이 없는 그래프
```

진입 차수는 위상 정렬에서 매우 중요하다.

```text
진입 차수가 0인 정점 = 지금 바로 처리할 수 있는 정점
```

어떤 정점으로 들어오는 간선이 없다는 것은, 그 정점보다 먼저 해야 할 일이 없다는 뜻이다.

## 진입 차수

다음 그래프를 보자.

```text
A -> C
B -> C
C -> D
```

각 정점의 진입 차수는 다음과 같다.

```text
A: 0
B: 0
C: 2
D: 1
```

`A`와 `B`는 진입 차수가 `0`이므로 먼저 처리할 수 있다.

`C`는 `A`와 `B`가 모두 처리되어야 진입 차수가 `0`이 된다.

## 위상 정렬 의사코드

큐를 사용하는 방식은 다음과 같다.

```text
topological_sort(그래프):
    모든 정점의 진입 차수를 계산한다
    진입 차수가 0인 정점을 큐에 넣는다
    결과 리스트를 비워 둔다

    큐가 비어 있지 않은 동안 반복한다:
        큐에서 정점을 하나 꺼낸다
        꺼낸 정점을 결과 리스트에 추가한다

        꺼낸 정점에서 나가는 간선을 하나씩 확인한다:
            연결된 다음 정점의 진입 차수를 1 줄인다
            다음 정점의 진입 차수가 0이 되면:
                큐에 넣는다

    결과 리스트의 길이가 정점 수와 같다면:
        위상 정렬 성공
    그렇지 않다면:
        사이클이 있음
```

핵심은 지금 처리할 수 있는 정점부터 하나씩 꺼내는 것이다.

## 조금 더 코드처럼 쓴 의사코드

```text
function topological_sort(graph):
    indegree = map with default 0

    for each vertex in graph:
        for each next in graph[vertex]:
            indegree[next] = indegree[next] + 1

    queue = new Queue()

    for each vertex in graph:
        if indegree[vertex] == 0:
            enqueue vertex to queue

    result = []

    while queue is not empty:
        current = dequeue from queue
        add current to result

        for each next in graph[current]:
            indegree[next] = indegree[next] - 1

            if indegree[next] == 0:
                enqueue next to queue

    if length(result) != number of vertices:
        return "cycle exists"

    return result
```

이 방식은 칸 알고리즘이라고도 부른다.

## 흐름을 쉽게 풀어보기

위상 정렬은 이렇게 생각하면 쉽다.

```text
1. 먼저 해야 할 일이 없는 정점을 찾는다.
2. 그 정점을 결과에 넣는다.
3. 그 정점이 끝났으므로, 그 정점에서 나가는 간선을 제거한 것처럼 생각한다.
4. 새롭게 먼저 할 일이 없어진 정점을 찾는다.
5. 모든 정점을 처리할 때까지 반복한다.
```

즉, "지금 당장 할 수 있는 일"부터 하나씩 처리하는 알고리즘이다.

## 예시

다음 그래프가 있다고 하자.

```text
A -> C
B -> C
C -> D
B -> E
```

진입 차수는 다음과 같다.

```text
A: 0
B: 0
C: 2
D: 1
E: 1
```

처음 큐에는 `A`, `B`가 들어간다.

```text
queue = [A, B]
result = []
```

`A`를 꺼낸다.

```text
result = [A]
C의 진입 차수: 2 -> 1
```

`B`를 꺼낸다.

```text
result = [A, B]
C의 진입 차수: 1 -> 0
E의 진입 차수: 1 -> 0
queue = [C, E]
```

`C`를 꺼낸다.

```text
result = [A, B, C]
D의 진입 차수: 1 -> 0
queue = [E, D]
```

`E`, `D`를 차례로 꺼내면 결과는 다음과 같을 수 있다.

```text
A -> B -> C -> E -> D
```

큐에 넣는 순서에 따라 다른 정답도 가능하다.

## 정답이 여러 개일 수 있는 이유

위상 정렬 결과는 하나로 고정되지 않을 수 있다.

예를 들어 `A`와 `B` 사이에 선후 관계가 없다면 둘 중 무엇이 먼저 나와도 된다.

```text
A -> C
B -> C
```

가능한 결과:

```text
A -> B -> C
B -> A -> C
```

둘 다 간선의 방향을 지키므로 올바른 위상 정렬이다.

## 사이클이 있는 경우

위상 정렬은 사이클이 있는 그래프에서는 할 수 없다.

```text
A -> B
B -> C
C -> A
```

이 경우 `A`를 하려면 `C`가 먼저 필요하고, `C`를 하려면 `B`가 먼저 필요하고, `B`를 하려면 `A`가 먼저 필요하다.

서로가 서로를 기다리므로 시작할 수 있는 정점이 없다.

위상 정렬에서는 결과 리스트의 길이를 확인해 사이클을 판단할 수 있다.

```text
result의 길이 < 정점 수 이면:
    사이클이 있다
```

## DFS로 구현하는 위상 정렬

위상 정렬은 DFS로도 구현할 수 있다.

DFS 방식은 어떤 정점에서 갈 수 있는 정점을 모두 처리한 뒤, 현재 정점을 결과에 넣는다.

```text
function dfs_topological_sort(graph):
    visited = new Set()
    result = []

    for each vertex in graph:
        if vertex is not in visited:
            dfs(vertex)

    reverse result
    return result

function dfs(current):
    add current to visited

    for each next in graph[current]:
        if next is not in visited:
            dfs(next)

    add current to result
```

후위 순회처럼 나중에 결과에 넣고, 마지막에 뒤집으면 위상 정렬 순서가 된다.

사이클 검사가 필요하다면 현재 재귀 경로에 있는 정점을 따로 표시해야 한다.

## 큐 방식과 DFS 방식 비교

```text
큐 방식:
    진입 차수를 사용한다.
    사이클 여부를 결과 길이로 확인하기 쉽다.
    작업 순서 문제에 직관적이다.

DFS 방식:
    깊이 우선 탐색을 사용한다.
    후위 순서의 역순으로 결과를 만든다.
    사이클 검사를 하려면 방문 상태를 세 단계로 나누면 좋다.
```

둘 다 시간 복잡도는 같다.

문제에서 진입 차수가 자연스럽게 보이면 큐 방식이 이해하기 쉽다.

## 종료 조건

큐 방식의 종료 조건은 큐가 비는 것이다.

```text
while queue is not empty
```

큐가 비었는데 아직 결과에 들어가지 않은 정점이 있다면 사이클이 있다는 뜻이다.

```text
if length(result) != number of vertices:
    cycle exists
```

## 연결되지 않은 그래프

그래프가 여러 덩어리로 나뉘어 있어도 위상 정렬은 가능하다.

```text
A -> B

C -> D
```

가능한 결과:

```text
A -> C -> B -> D
C -> A -> D -> B
A -> B -> C -> D
```

각 간선의 방향만 지키면 된다.

그래서 처음에 진입 차수 `0`인 정점이 여러 개 있을 수 있다.

## 실제 사용 예시

위상 정렬은 선후 관계가 있는 문제에 자주 사용된다.

```text
선수 과목 순서
작업 스케줄링
빌드 순서
패키지 의존성 설치 순서
프로젝트 작업 순서
```

예를 들어 어떤 패키지를 설치하기 전에 의존 패키지를 먼저 설치해야 한다면, 위상 정렬로 설치 순서를 구할 수 있다.

## 시간 복잡도

정점 수를 `V`, 간선 수를 `E`라고 하자.

진입 차수를 계산할 때 모든 간선을 확인한다.

그 뒤 각 정점과 간선을 한 번씩 처리한다.

```text
시간 복잡도: O(V + E)
```

## 공간 복잡도

진입 차수 배열, 큐, 결과 리스트가 필요하다.

```text
indegree: O(V)
queue: O(V)
result: O(V)
```

그래프 저장 공간을 제외한 기본 공간 복잡도는 다음과 같다.

```text
공간 복잡도: O(V)
```

그래프 자체를 인접 리스트로 저장하면 `O(V + E)`가 필요하다.

## 장점

```text
선후 관계가 있는 작업 순서를 구할 수 있다.
사이클이 있는지도 확인할 수 있다.
시간 복잡도가 O(V + E)로 효율적이다.
구현이 비교적 단순하다.
```

## 단점

```text
방향 그래프에서만 의미가 있다.
사이클이 있으면 위상 정렬 결과를 만들 수 없다.
정답이 여러 개일 수 있어 출력 순서가 구현에 따라 달라질 수 있다.
```

## 언제 사용하면 좋은가

위상 정렬은 다음 상황에서 좋다.

```text
어떤 작업보다 먼저 해야 하는 작업이 있을 때
의존성 순서를 계산해야 할 때
DAG에서 정점의 처리 순서를 정해야 할 때
사이클이 있는지 확인해야 할 때
```

## 주의할 점

위상 정렬은 일반 그래프가 아니라 DAG에서만 가능하다.

```text
방향 그래프여야 한다.
사이클이 없어야 한다.
```

또한 정답이 여러 개일 수 있으므로, 특정 순서가 필요하다면 큐 대신 우선순위 큐를 사용하는 경우도 있다.

```text
가능한 정점 중 번호가 작은 것부터 처리
```

## 한 문장 요약

위상 정렬은 방향과 선후 관계를 지키면서 DAG의 정점들을 한 줄로 나열하는 알고리즘이다.

  v

  |
  v

## 더 짧게 외우는 흐름

```text
진입 차수 계산
0인 정점 큐에 넣기
하나 꺼내 결과에 넣기
나가는 간선 제거처럼 처리
새로 0이 된 정점 큐에 넣기
```

## 머릿속 결정 순서

```text
1. 방향 그래프인가?
2. 사이클이 없어야 하는 문제인가?
3. 진입 차수를 계산했는가?
4. 진입 차수 0인 정점을 큐에 넣었는가?
5. 정점을 꺼낼 때 나가는 간선을 반영했는가?
6. 결과 길이로 사이클 여부를 확인했는가?
```
